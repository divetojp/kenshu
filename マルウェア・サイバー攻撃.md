# マルウェア・サイバー攻撃

**マルウェア（Malicious Software）**は、悪意ある目的で作られたソフトウェアの総称です。「ウイルスに感染した」という言葉は日常的に使われますが、実際にはウイルス・ワーム・ランサムウェアなど多くの種類があり、それぞれ動作と対策が異なります。

---

## はじめて読む人へ

「セキュリティソフトを入れていれば安全」は誤解です。マルウェアの種類・感染経路・攻撃者の目的を知ることで、どの状況でどんな対策が有効かを判断できるようになります。

### 読む前に押さえること

特に前提知識は不要です。1年生から読めます。

### 読み終えたら説明できること

- マルウェアの主な種類とそれぞれの動作を説明できる
- 感染経路ごとの対策を説明できる
- DDoS 攻撃の仕組みと対策を説明できる

---

## マルウェアの種類

### ウイルス（Virus）

他のファイルに寄生して増殖します。感染ファイルを実行したり開いたりすることで広がります。

!!! info ""
    **感染の仕組み**

    正常なファイル.exe + ウイルスコード
    → 実行すると他のファイルにコピー
    → ネットワーク経由で他の PC に拡散
### ワーム（Worm）

**ファイルに寄生せず**、ネットワークを通じて自己複製します。OS やアプリの脆弱性を突いて、ユーザーの操作なしに感染します。

!!! info ""
    代表例: WannaCry（2017年）
    Windows の脆弱性（EternalBlue）を利用
    世界 150 か国 23 万台以上のコンピュータに感染
    NHS（英国国民保健サービス）の病院システムが停止
### トロイの木馬（Trojan）

有用なソフトウェアに見せかけて、実際は悪意ある動作をするプログラムです。

!!! info ""
    **例**

    「無料ゲームアプリ」をインストール
    → 裏でキーロガーが動作しパスワードを盗む
    → C&C サーバーに情報を送信
### ランサムウェア（Ransomware）

ファイルを暗号化して「復号したければ金を払え」と脅迫します。近年の最大の脅威の一つです。

!!! info ""
    **攻撃の流れ**

    1. メールの添付ファイルやリンクから感染
    2. PC 内のファイルを AES などで暗号化
    3. 「○○ Bitcoin 払えば復号鍵を渡す」と表示
    4. バックアップも暗号化されることが多い
**対策：**
- 重要データは**オフラインバックアップ**を取る（クラウドも同期されてしまう場合がある）
- OS・アプリを最新にして脆弱性を減らす
- 不審なメールの添付ファイルを開かない

### スパイウェア・キーロガー

ユーザーの操作・入力を盗み見ます。

!!! info ""
    **キーロガーの例**

    キーボードの入力を記録 → ID・パスワード・クレジットカード番号を盗む

    **スパイウェアの例**

    ブラウザの履歴・ブックマーク・クッキーを収集
    → 広告業者や犯罪者に売却
### ボットネット

多数の感染 PC（ボット）を遠隔操作するネットワークです。

!!! info ""
    C&C（Command & Control）サーバー
    ↓ 命令
    Bot1  Bot2  Bot3 ... Bot10万
    （感染PC群 = ゾンビ PC）

    **利用例**

    - DDoS 攻撃の踏み台
    - スパムメールの大量送信
    - 仮想通貨のマイニング
    - パスワードのブルートフォース

## 主な感染経路

### メールの添付ファイル・リンク

最も多い感染経路です。

!!! warning ""
    **危険なパターン**

    「請求書 .pdf.exe」← 拡張子が偽装されている
    「Dropbox でファイルを共有しました」← 偽サイトへ誘導
    「荷物のお届けができませんでした」← フィッシング
**対策：**
- 送信者アドレスのドメインを確認する（@amaz0n.co.jp など）
- 知らない人からの添付ファイルは開かない
- リンクはホバーして実際の URL を確認してから開く

### ソフトウェアの脆弱性

OS・ブラウザ・アプリの脆弱性（セキュリティホール）を突かれます。

!!! info ""
    **代表的な脆弱性**

    ゼロデイ脆弱性: パッチが存在しない状態の脆弱性
    既知脆弱性:    パッチが出ているのに適用されていない
**対策：**
- OS・ブラウザ・ソフトウェアを**常に最新に保つ**
- サポート切れの OS は使わない（Windows 7・XP など）

### 不正サイト・ドライブバイダウンロード

正規サイトに見せかけたサイト、または改ざんされた正規サイトにアクセスするだけで感染します。

**対策：**
- ブラウザのセキュリティアップデートを維持する
- 信頼できないサイトではスクリプトをブロックする

### USB メモリ・外部デバイス

!!! info ""
    代表例: Stuxnet（2010年）
    核施設の制御システムを狙った国家レベルのマルウェア
    インターネット非接続の PC にも USB 経由で感染
**対策：**
- 出所不明の USB を接続しない
- 自動実行（AutoRun）を無効にする

## DoS 攻撃と DDoS 攻撃

### DoS 攻撃（Denial of Service）

サーバーに大量のリクエストを送り、**正常なユーザーが使えない状態**にします。

!!! info ""
    **通常のアクセス**

    ユーザー → Webサーバー（余裕を持って処理）

    **DoS 攻撃**

    攻撃者 → Webサーバー（100万回/秒のリクエスト）
    → サーバーが処理しきれずダウン
    → 正規ユーザーにアクセス不能
### DDoS 攻撃（Distributed DoS）

**複数の踏み台（ボットネット）から同時攻撃**します。

!!! info ""
    ```text
    ボット1                 ┐
    ボット2                 │ → Webサーバー（ダウン）
    ボット3                 │
    ...                    ┘
    ボット10万（世界中に分散）
    ```
### DDoS の種類

| 種類 | 仕組み | 代表例 |
|------|--------|-------|
| **帯域幅消費型** | 大量のパケットで帯域を使い切る | UDP Flood |
| **プロトコル攻撃** | TCP の SYN を大量送信して接続待ち状態を作る | SYN Flood |
| **アプリケーション層** | HTTP リクエストを大量に送る | HTTP Flood |
| **リフレクション攻撃** | 増幅器（DNS・NTP）を悪用して攻撃量を増やす | DNS Amplification |

### DDoS 対策

```python
# アプリケーション側での Rate Limiting（レート制限）の実装例
from fastapi import FastAPI, Request, HTTPException
from slowapi import Limiter, _rate_limit_exceeded_handler
from slowapi.util import get_remote_address

limiter = Limiter(key_func=get_remote_address)
app = FastAPI()
app.state.limiter = limiter

@app.get("/api/data")
@limiter.limit("10/minute")   # 1分間に10回まで
async def get_data(request: Request):
    return {"data": "..."}
```

**インフラレベルの対策：**
- CDN（Cloudflare・AWS Shield）による DDoS 防御
- WAF（Web Application Firewall）
- Anycast ネットワークで攻撃を分散吸収

## マルウェア対策のチェックリスト

!!! info ""
    【個人向け】
    □ OS・アプリを最新の状態に保つ
    □ ウイルス対策ソフトを入れて定義ファイルを最新にする
    □ 不審なメールの添付ファイル・リンクを開かない
    □ 重要ファイルは定期的にオフラインバックアップ
    □ 出所不明の USB を接続しない
    □ 公衆 Wi-Fi では VPN を使う

    【開発者向け】
    □ 使用するライブラリの脆弱性を定期的にチェック（npm audit / pip-audit）
    □ ユーザー入力のバリデーション（SQLインジェクション対策）
    □ 依存パッケージをロックファイルで固定
    □ シークレット情報をコードに直接書かない
    □ HTTPS を強制する
---

## 確認問題

1. ウイルスとワームの最大の違いを、「自己複製の方法」の観点から説明してください。
2. ランサムウェア対策として「クラウドバックアップだけでは不十分」な理由を説明してください。
3. DDoS 攻撃が DoS 攻撃より防御が難しい理由を説明してください。
