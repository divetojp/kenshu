実験管理（MLflow / wandb）¶

機械学習の実験は「パラメータを変えながら何度も試す」作業です。管理しないと「どの条件が一番良かったか」がすぐわからなくなります。実験管理ツールは パラメータ・メトリクス・成果物を記録・比較 します。

はじめて読む人へ¶

実験管理は、機械学習で試した条件と結果を記録する作業です。モデルを何度も試すと、どのデータ・パラメータ・コードで良い結果が出たのか分からなくなりがちです。

読む前に押さえること¶

  • パラメータ、メトリクス、モデルファイルをセットで記録します。
  • MLflow や wandb は、実験結果を後から比較するために使います。
  • モデルレジストリは、本番に使うモデルを管理する仕組みです。

読み終えたら説明できること¶

  • 実験条件を記録する理由を説明できる。
  • MLflow と wandb の役割を理解できる。
  • モデルのバージョン管理が必要な理由を説明できる。

MLflow¶

MLflow は、機械学習の実験条件と結果を記録するためのツールです。学習率、モデルの種類、特徴量、評価指標、学習済みモデルなどを1つの実験として残せます。

実験が増えると、どの設定が良かったのかを記憶だけで追うのは難しくなります。MLflow に記録しておくと、後から条件を並べて比較し、再現したいモデルを探しやすくなります。

OSS の実験管理ツールです。ローカルで動かすことができ、クラウド不要で始められます。

pip install mlflow

まずはローカル環境にMLflowを入れます。MLflowは、実験結果をファイルやローカルサーバーに保存し、あとでブラウザから比較できるようにします。

基本的な記録¶

次のコードは、ランダムフォレストを学習しながら、パラメータ、評価指標、モデル、混同行列の画像を1つの実験として保存します。実験管理で大事なのは、精度だけでなく「どの条件でその精度が出たか」を同時に残すことです。

import mlflow
import mlflow.sklearn
from sklearn.ensemble import RandomForestClassifier
from sklearn.metrics import accuracy_score, f1_score
from sklearn.model_selection import train_test_split

# 実験の作成（なければ自動で作成）
mlflow.set_experiment("heart-disease-prediction")

with mlflow.start_run(run_name="random_forest_v1"):
    # ハイパーパラメータを記録
    params = {"n_estimators": 100, "max_depth": 5, "random_state": 42}
    mlflow.log_params(params)

    # モデル訓練
    model = RandomForestClassifier(**params)
    model.fit(X_train, y_train)
    y_pred = model.predict(X_test)

    # メトリクスを記録
    mlflow.log_metrics({
        "accuracy": accuracy_score(y_test, y_pred),
        "f1": f1_score(y_test, y_pred, average="weighted"),
    })

    # モデルを保存（scikit-learn 形式）
    mlflow.sklearn.log_model(model, "model")

    # 任意のファイルも保存可能
    import matplotlib.pyplot as plt
    from sklearn.metrics import ConfusionMatrixDisplay
    fig, ax = plt.subplots()
    ConfusionMatrixDisplay.from_predictions(y_test, y_pred, ax=ax)
    plt.savefig("confusion_matrix.png")
    mlflow.log_artifact("confusion_matrix.png")

# UI で確認
# mlflow ui  → http://localhost:5000

mlflow.start_run(...) の中に書いた記録は、1回の実験としてまとまります。log_params は設定値、log_metrics は評価結果、log_model は学習済みモデル、log_artifact は画像やレポートなどのファイルを保存します。あとからUIで「max_depth=5 のときと10のときでF1はどう違ったか」を比較できます。

学習曲線のログ¶

深層学習では、最終精度だけでなく、エポックごとの損失や精度の変化を見ることが重要です。学習曲線を残すと、過学習しているのか、学習率が高すぎて不安定なのかを判断しやすくなります。

with mlflow.start_run():
    for epoch in range(50):
        # エポックごとのメトリクスを step で記録
        mlflow.log_metric("train_loss", train_loss, step=epoch)
        mlflow.log_metric("val_loss", val_loss, step=epoch)
        mlflow.log_metric("val_accuracy", val_acc, step=epoch)

step=epoch を指定すると、同じメトリクス名でも時系列として記録されます。train_loss は下がっているのに val_loss が上がる場合、訓練データに過剰適合している可能性があります。

PyTorch との連携¶

PyTorch では学習ループを自分で書くことが多いため、どのタイミングで何をログするかも自分で決めます。各エポックの訓練損失と検証精度を残しておくと、後から学習の進み方を比較できます。

import mlflow.pytorch

with mlflow.start_run():
    mlflow.log_params({
        "lr": 1e-3, "batch_size": 32, "epochs": 50,
        "hidden_dim": 128, "dropout": 0.3
    })

    # 学習ループ
    for epoch in range(50):
        train_loss = train_epoch(model, loader, optimizer, criterion)
        val_loss, val_acc = evaluate(model, val_loader, criterion)
        mlflow.log_metrics(
            {"train_loss": train_loss, "val_loss": val_loss, "val_acc": val_acc},
            step=epoch
        )

    mlflow.pytorch.log_model(model, "pytorch_model")

mlflow.pytorch.log_model は、PyTorchモデルをMLflowの成果物として保存します。パラメータやメトリクスと同じrunに紐づくため、「この精度を出したモデル本体はどれか」を後から取り出せます。

wandb（Weights & Biases）¶

クラウドベースの実験管理ツールです。チームでの共有・比較・ダッシュボードが充実しています。

pip install wandb
wandb login  # APIキーを入力

wandb はクラウド上のプロジェクトに実験を送信します。wandb login で自分のアカウントとローカル環境を紐づけると、学習中のメトリクスをWeb画面でリアルタイムに確認できます。

import wandb

wandb.init(
    project="heart-disease",
    name="rf_100trees",
    config={
        "n_estimators": 100,
        "max_depth": 5,
        "learning_rate": 0.05,
    }
)

config = wandb.config

# モデル訓練（省略）
for epoch in range(50):
    train_loss = train_epoch(...)
    val_acc = evaluate(...)

    wandb.log({
        "train_loss": train_loss,
        "val_accuracy": val_acc,
        "epoch": epoch
    })

# モデルの保存
wandb.save("model.pkl")
wandb.finish()

wandb.init の config には、実験条件を入れます。学習中は wandb.log で損失や精度を送ります。wandb.finish() は、その実験が終わったことを明示する処理です。ノートブックやスクリプトで複数実験を回すときに、runの区切りをはっきりさせます。

wandb の便利な機能¶

Sweep は、ハイパーパラメータの候補を自動で試す仕組みです。人間が手で learning_rate や batch_size を変えて何十回も実行する代わりに、探索範囲と評価指標を指定しておきます。

# ハイパーパラメータ自動探索（wandb sweep）
sweep_config = {
    "method": "bayes",           # ベイズ最適化
    "metric": {"name": "val_accuracy", "goal": "maximize"},
    "parameters": {
        "learning_rate": {"min": 0.0001, "max": 0.1, "distribution": "log_uniform_values"},
        "batch_size": {"values": [16, 32, 64, 128]},
        "hidden_dim": {"values": [64, 128, 256]},
    }
}

def train_sweep():
    with wandb.init() as run:
        cfg = run.config
        model = MLP(hidden_dim=cfg.hidden_dim)
        optimizer = torch.optim.Adam(model.parameters(), lr=cfg.learning_rate)
        # 学習...
        wandb.log({"val_accuracy": val_acc})

sweep_id = wandb.sweep(sweep_config, project="my-project")
wandb.agent(sweep_id, train_sweep, count=30)

sweep_config は探索のルールです。method: "bayes" は、過去の結果を見ながら有望な設定を選ぶベイズ最適化を表します。wandb.agent は、その設定に従って train_sweep を複数回実行します。

MLflow でモデルレジストリ（本番管理）¶

モデルレジストリは、実験で生まれた多数のモデルの中から「本番で使うモデル」を管理する場所です。精度が高いモデルを見つけても、どのファイルを本番APIに載せたのかが曖昧だと運用できません。

# モデルを「本番用」として登録
with mlflow.start_run() as run:
    mlflow.sklearn.log_model(model, "model",
                              registered_model_name="production-classifier")

# 特定バージョンを「Production」に昇格
from mlflow.tracking import MlflowClient
client = MlflowClient()
client.transition_model_version_stage(
    name="production-classifier",
    version=3,
    stage="Production"
)

# Production モデルを取得して推論
production_model = mlflow.sklearn.load_model(
    "models:/production-classifier/Production"
)

このコードでは、モデルを production-classifier という名前で登録し、特定バージョンを Production ステージへ昇格しています。本番API側は models:/production-classifier/Production を読み込めば、現在の本番モデルを取得できます。モデルを差し替えるときも、APIコードを書き換えずにレジストリ側のステージを変更できます。
