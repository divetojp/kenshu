# ソーシャルエンジニアリング・詐欺対策

**技術的な脆弱性ではなく、「人間の心理」を突く攻撃**です。どんなに堅牢なシステムを作っても、人間がだまされてパスワードを教えてしまえば意味がありません。フィッシング詐欺・サポート詐欺・なりすましはすべてここに分類されます。

---

## はじめて読む人へ

「自分はだまされない」と思っている人が最もだまされやすいです。ソーシャルエンジニアリングは、人間の「権威への服従」「緊急時の判断力低下」「親切心」などの心理を組織的に悪用します。攻撃の手口を知ることが最大の対策です。

### 読む前に押さえること

特に前提知識は不要です。

### 読み終えたら説明できること

- フィッシング詐欺の主な手口と見分け方を説明できる
- サポート詐欺・遠隔操作詐欺の流れを説明できる
- 無線 LAN・スマートフォン利用時のセキュリティリスクを説明できる

---

## ソーシャルエンジニアリングとは

!!! info ""
    技術的攻撃: システムの脆弱性を突く → セキュリティパッチで対策できる

    **ソーシャルエンジニアリング**

    人間の心理・行動パターンを利用してセキュリティを突破する
    → 人間がいる限り完全には防げない
### 攻撃者が利用する心理

| 心理 | 利用例 |
|------|--------|
| **権威への服従** | 「警察庁からです。ご協力ください」 |
| **緊急性・焦り** | 「今すぐ対応しないとアカウントが削除されます」 |
| **親切心・助けたい** | 「困っているんですが、パスワードを...」 |
| **希少性・お得感** | 「今だけ無料！期間限定」 |
| **社会的証明** | 「多くのユーザーがこの方法で解決しました」 |

## フィッシング詐欺

### 仕組み

正規サービスを装った偽のメール・サイトに誘導して、ID・パスワード・クレジットカード情報を盗みます。

!!! info ""
    【典型的なフィッシングの流れ】
    1. 偽メールが届く
    「【重要】Amazon アカウントに不審なアクセスがありました」
    「今すぐ確認してください」→ リンクをクリック

    2. 本物そっくりの偽サイトに誘導
    URL: amazon-secure-login.com ← 本物は amazon.co.jp

    3. ログインフォームに ID・パスワードを入力
    → 攻撃者のサーバーに送信される

    4. 攻撃者がそのID・パスワードでログイン
    → 買い物・個人情報の盗取・2次被害
### フィッシングの見分け方

!!! warning ""
    □ 送信者アドレスを確認する
    × amazon-support@amaz0n.co.jp  ← 0 が数字
    × noreply@amazon.verify-account.net  ← ドメインが違う
    ○ noreply@amazon.co.jp

    □ リンクの URL を確認する（クリック前にホバー）
    × https://amazon-jp.secure-login.net/
    ○ https://www.amazon.co.jp/

    □ HTTPS でも安全とは限らない
    フィッシングサイトも HTTPS を取得できる
    → 錠前マークだけで安全と判断しない

    □ 緊急を煽る文言に注意
    「今すぐ」「24時間以内に」「アカウントが停止されます」

    □ 公式サイトから直接ログインする
    メールのリンクをたどらず、ブックマークや直接入力でアクセス
### スピアフィッシング

特定の個人・組織を狙った精度の高いフィッシングです。

!!! info ""
    **通常のフィッシング**

    「お客様へ」（不特定多数に送信）

    **スピアフィッシング**

    「田中さん、先日の○○プロジェクトについてです。
    添付の資料を確認してください。 — 鈴木部長より」
    → 名前・職場・状況を事前調査して信頼性を高める
    → SNS の公開情報を使って攻撃が精巧化
**対策：**
- SNS に職場・役職・プロジェクト名を公開しすぎない
- 「上司からのメール」でも添付ファイルを疑う（なりすましの可能性）

## サポート詐欺

### 手口

!!! warning ""
    1. 突然「ウイルスを検出しました」という警告画面が表示
    大きな警告音と共に「Microsoft のサポートに電話してください」

    2. 電話すると「サポート担当者」（偽）が対応
    「遠隔操作で修復します。このソフトをインストールしてください」

    3. 遠隔操作ソフト（TeamViewer 等）をインストールさせる
    → PC を完全に操作される

    4. 「修復費用として 10 万円かかります」
    → 電子マネー・ギフトカードで支払わされる
    または：バックドアを仕込んで後日再侵入
### 見分け方と対処法

!!! warning ""
    □ 正規の Microsoft・Apple・Google は突然電話を要求しない
    □ ウイルス警告が出ても、ブラウザを強制終了（タスクマネージャー）
    → ブラウザの警告音・警告画面はマルウェアではない場合が多い
    □ 知らない人に遠隔操作ソフトをインストールさせない
    □ 電子マネー・ギフトカードで代金を要求してきたら詐欺
    **□ すでに遠隔操作させてしまった場合**

    → インターネットを切断、パスワードをすべて変更、専門家に相談

## 無線 LAN のセキュリティ

### 公衆 Wi-Fi のリスク

!!! info ""
    攻撃: 中間者攻撃（Man-in-the-Middle Attack）

    ユーザー → [攻撃者の偽 Wi-Fi ← 同じ SSID を使って偽造] → インターネット

    **全通信を傍受できる**

    ID・パスワード（HTTP の場合）
    ブラウザ履歴
    送受信するデータ
**リスクが高い公衆 Wi-Fi の特徴：**

!!! info ""
    × 「フリー Wi-Fi」「FREE」「Guest」など認証なしで接続できる
    × パスワードが掲示されていてだれでも知っている
    × HTTPS 以外の通信が平文で流れる
### 安全な利用方法

!!! warning ""
    □ HTTPS サイトのみ使用する（URL バーに鍵マーク）
    □ VPN を使う（すべての通信を暗号化）
    □ 公衆 Wi-Fi でのネットバンキング・機密情報は避ける
    □ 自動接続を無効にする（フリー Wi-Fi への自動接続を防ぐ）
    □ テザリングを使う（スマートフォンの回線を使う）
### WPA2 と WPA3

| 規格 | 状態 | 特徴 |
|------|------|------|
| WEP | 非推奨（解読済み） | 最古の規格、使用禁止 |
| WPA | 非推奨 | WEP の後継、脆弱性あり |
| WPA2 | 現在の標準 | AES 暗号化、現在でも広く使用 |
| WPA3 | 最新 | より強固な認証、解読困難 |

## スマートフォンのセキュリティ

### 主なリスク

!!! info ""
    ① 不正アプリ（偽アプリ・マルウェア入りアプリ）
    → 公式ストア（App Store / Google Play）以外からインストールしない
    → インストール時の「権限」を確認する

    ② 紛失・盗難
    → スクリーンロック（PIN・指紋・顔認証）を必ず設定
    → リモートワイプ機能を有効にする（iCloud / Find My Device）

    ③ SMS フィッシング（スミッシング）
    「【佐川急便】お荷物を保管しています。以下でご確認ください」

    ④ QR コード詐欺
    偽の QR コードを貼って偽サイトへ誘導
    → QR を読んだ URL を開く前に確認する
### スマートフォンの設定チェックリスト

!!! info ""
    □ OS を最新に保つ（セキュリティパッチ）
    □ スクリーンロックを設定する
    □ 公式ストア以外からのインストールを禁止する
    □ 不要なアプリの権限（位置情報・マイク・カメラ）を確認する
    □ 公衆 Wi-Fi に自動接続しない設定にする
    □ Bluetooth を使わないときはオフにする
    □ バックアップを定期的に取る

## テレワーク・リモートワークのセキュリティ

### リスク

!!! info ""
    **オフィス環境**

    管理されたネットワーク・物理的なセキュリティ

    **自宅・カフェ**

    家族がいる・画面を見られるリスク
    ホームルーターのセキュリティが低い
    公衆 Wi-Fi を使うリスク
### 対策

| リスク | 対策 |
|--------|------|
| 通信の盗聴 | 会社支給の VPN を使う |
| 画面の盗み見 | プライバシーフィルターを使う |
| 端末の紛失 | フルディスク暗号化（BitLocker / FileVault）を有効にする |
| ルーターの脆弱性 | ルーターのファームウェアを最新に保つ・デフォルトパスワードを変更する |
| 影・肩越しの覗き見（ショルダーハック） | 公共の場でパスワードを入力しない |

## 確認問題

1. フィッシングサイトが HTTPS（鍵マーク）を表示している場合でも詐欺の可能性があるのはなぜですか？
2. サポート詐欺の被害を受けないために、「突然の警告画面」が出たとき最初にすべき行動を説明してください。
3. 公衆 Wi-Fi で VPN を使う理由を、「中間者攻撃」の概念を使って説明してください。
