# Markdown 記法

## Markdown とは

**シンプルな記号を使って文章を装飾できる書き方のルール** です。

`#` を付けると見出し、`**` で囲むと太字、など直感的に書けます。  
このテックガイドサイト自体も Markdown で書かれています。

このプロジェクトでは主に 2 つの場面で使います：
- **このテックガイド** のページを編集するとき
- **Astro のコンテンツ** （お知らせ・ブログ記事など）を書くとき

---

## はじめて読む人へ

Markdown は、文章に少しだけ記号を足して構造を表す記法です。HTML より簡単に、見出し・リスト・リンク・コードブロックを書けます。


### 読む前に押さえること

- Markdown は見た目ではなく、文章の構造を表すために使います。
- コードブロックには言語名を付けると読みやすくなります。
- リンクや見出しを正しく書くと、サイト全体が移動しやすくなります。

### 読み終えたら説明できること

- 見出し、リスト、リンク、コードブロックを書ける。
- 技術メモをMarkdownで整理できる。
- 読みやすいページの基本構造を作れる。

---

## 見出し

見出しは、文章の目次になる部分です。見た目を大きくするためだけではなく、内容の階層を表すために使います。

`#` はページ全体のタイトル、`##` は大きな章、`###` は章の中の小見出し、というように順番を守ると読みやすくなります。見出しの階層が飛ぶと、読み手が文書の構造を追いにくくなります。

`#` の数で見出しのレベルが変わります。

次のコードブロックは、Markdown で見出しを書くときの入力例です。`#` の数が増えるほど、見出しの階層が深くなります。

```markdown
# 一番大きな見出し（h1）
## 二番目の見出し（h2）
### 三番目の見出し（h3）
#### 四番目の見出し（h4）
```

表示例：

# 一番大きな見出し
## 二番目の見出し
### 三番目の見出し

1 つのページでは、通常 `#` はページタイトルとして 1 回だけ使います。その下に `##`、さらに細かい項目に `###` を置くと、文章の構造が自然に伝わります。

---

## テキストの装飾

文章の一部を強調したいときは、太字や斜体を使います。ただし、装飾を増やしすぎると逆に読みにくくなるため、重要語やコード名に絞って使います。

```markdown
**太字**
*斜体（イタリック）*
~~取り消し線~~
`インラインコード`
```

表示例：

**太字** ／ *斜体* ／ ~~取り消し線~~ ／ `インラインコード`

`インラインコード` は、文章の中でコマンド名、ファイル名、変数名などを示すときに使います。普通の強調ではなく「これは文字として正確に読むものです」という意味を持たせられます。

---

## 段落と改行

空白行を挟むと段落が分かれます。

Markdown では、ただ改行しただけでは同じ段落として扱われることがあります。話題を分けたいときは、空白行を 1 行入れて段落を分けます。

```markdown
これは1つ目の段落です。

これは2つ目の段落です。
```

段落を分けると、読み手が文章のまとまりを追いやすくなります。技術文書では、1 段落に 1 つの主張を置くと読みやすくなります。

## リスト

### 箇条書き（順番なし）

箇条書きは、順番が重要ではない項目を並べるときに使います。機能一覧、必要な道具、注意点などに向いています。

```markdown
- Git
- GitHub
- Astro
```

表示例：
- Git
- GitHub
- Astro

### 番号付きリスト

番号付きリストは、順番が重要な手順に使います。インストール手順や提出手順では、番号があると読み手がどこまで進んだか確認しやすくなります。

```markdown
1. リポジトリをクローン
2. ブランチを作成
3. コミットしてプッシュ
```

表示例：
1. リポジトリをクローン
2. ブランチを作成
3. コミットしてプッシュ

### ネスト（入れ子）

項目の中にさらに細かい項目を入れたいときは、インデントしてネストします。深くしすぎると読みにくくなるため、2 階層程度に抑えるとよいです。

```markdown
- 大項目
  - 小項目（スペース2つかTab）
  - 小項目
```

## リンク

リンクは、文章同士をつなぐための重要な要素です。リンクテキストには「こちら」ではなく、移動先の内容が分かる言葉を書くと親切です。

```markdown
[リンクのテキスト](URL)

[近江テック・アカデミー](https://example.com)
[Git ページへ](Git)
```

サイト内の別ページへのリンクは、ページ名だけで OK です（URLは不要）。

外部サイトへリンクする場合は URL を書き、サイト内のページへ移動する場合はページ名を書きます。内部リンクが整っていると、教材全体を横断して学びやすくなります。

---

## 画像

画像は、スクリーンショット、図解、ロゴなどを本文に埋め込むために使います。`[]` の中の代替テキストは、画像が見えない環境でも内容が分かるように書きます。

```markdown
![代替テキスト](画像のURL)

![ロゴ](https://example.com/logo.png)
```

代替テキストはアクセシビリティにも重要です。「画像」ではなく、「ログイン画面のスクリーンショット」のように、画像が何を表しているかを書きます。

---

## コードブロック

バッククォート3つ（` ``` `）で囲みます。言語名を指定するとシンタックスハイライトが適用されます。

コードブロックは、複数行のコマンドやプログラムを本文から分けて見せるために使います。開始側のバッククォートに `bash` や `python` などの言語名を付けると、読みやすく色分けされます。

````markdown
```bash
git commit -m "変更内容"
```

```html
<h1>タイトル</h1>
```
````

コードブロックの前後には、「何の例なのか」「実行すると何が起きるのか」を文章で添えると、読み手が写すだけでなく意味を理解できます。

---

## 引用

引用は、補足、注意、外部文献からの短い引用などを本文と区別して示すために使います。この教材では、注意点やメモを目立たせる用途にも使えます。

```markdown
> これは引用文です。
> 複数行にまたがることもできます。
```

表示例：

> これは引用文です。
> 複数行にまたがることもできます。

---

## 水平線（区切り線）

水平線は、大きな話題の区切りを視覚的に示すために使います。ページ内で多用すると分断されすぎるため、章の切れ目などに限定します。

```markdown
---
```

---

## 表（テーブル）

表は、複数の項目を比較するときに向いています。コマンド一覧、用語比較、設定値の説明などは、文章だけで並べるより表にすると読みやすくなります。

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| A   | B   | C   |
| D   | E   | F   |
```

表示例：

| 列1 | 列2 | 列3 |
|-----|-----|-----|
| A   | B   | C   |
| D   | E   | F   |

`:` を使って列の文字揃えを指定できます：

列の区切り線に `:` を付けると、左揃え・中央揃え・右揃えを指定できます。数値は右揃え、短いラベルは中央揃えにすると見やすいことがあります。

```markdown
| 左揃え | 中央揃え | 右揃え |
|:-------|:-------:|------:|
| 値     | 値      |     値 |
```

---

## チェックリスト

チェックリストは、作業の進捗管理に使えます。課題、レビュー項目、公開前確認など、完了・未完了を明確にしたいときに便利です。

```markdown
- [x] 完了した項目
- [ ] まだの項目
- [ ] まだの項目
```

表示例：
- [x] 完了した項目
- [ ] まだの項目

## まとめ：よく使う記法の早見表

| 書き方 | 見た目 |
|--------|--------|
| `# テキスト` | 大見出し |
| `## テキスト` | 中見出し |
| `**テキスト**` | **太字** |
| `*テキスト*` | *斜体* |
| `- テキスト` | 箇条書き |
| `[テキスト](URL)` | リンク |
| `` `コード` `` | `インラインコード` |
| `> テキスト` | 引用 |
| `---` | 区切り線 |

## 練習してみよう

テックガイドの `.md` ファイルをローカルで編集して、Markdown を実際に試してみましょう。

1. `git clone https://github.com/divetojp/kenshu.git` でリポジトリを取得
2. VS Code で任意の `.md` ファイルを開く
3. 自由にテキストを追加して **Ctrl+Shift+V**（Mac: ⌘+Shift+V）でプレビュー確認
4. `git add / commit / push` で変更を反映（GitHub Actions が自動でサイトをビルド・デプロイ）
