# ターミナル

## ターミナルとは

**文字を入力してコンピューターに命令を出す画面** のことです。

普段は「クリック」や「タップ」でPCを操作しますが、開発の世界では文字による命令（ **コマンド** ）を使うことが多いです。慣れると、マウスより素早く正確に作業できます。

> ターミナルは「黒い画面」「CUI」「コマンドライン」などとも呼ばれます。すべて同じものを指しています。

---

## はじめて読む人へ

ターミナルは、コンピュータに文字で命令するための入口です。最初は黒い画面に見えて怖く感じますが、実際には「今いる場所を確認する」「フォルダへ移動する」「コマンドを実行する」という基本操作の組み合わせです。


### 読む前に押さえること

- コマンドは、コンピュータに出す短い指示です。
- パスは、ファイルやフォルダの住所です。
- エラー文は失敗の通知ではなく、次に確認すべき場所を教えるヒントです。

### 読み終えたら説明できること

- `pwd`、`ls`、`cd` の役割を説明できる。
- 自分が今どのフォルダにいるか確認できる。
- ターミナルでプロジェクトフォルダへ移動できる。

## ターミナルの開き方

### Windows の場合

**PowerShell または Windows Terminal を使います。**

1. スタートメニューを開く（または `Win + S`）
2. 「Windows Terminal」または「PowerShell」と検索
3. クリックして開く

> **Windows Terminal** がない場合は Microsoft Store からインストールできます。PowerShell でも代用できます。

### Mac の場合

**方法①：Spotlight 検索**
1. `Command（⌘）+ Space` を押す
2. 「ターミナル」と入力
3. Enter を押す

**方法②：Finder から**
1. Finder を開く
2. アプリケーション → ユーティリティ → ターミナル

## コマンドの基本的な使い方

コマンドは「プログラム名 + オプション + 対象」という形で読むと理解しやすくなります。たとえば `ls -la` なら、`ls` がプログラム名、`-la` が表示方法を変えるオプションです。

最初はコマンドを暗記するより、実行前に「何を対象に、何をしたいのか」を言葉にしてください。ファイルを見たいのか、移動したいのか、作りたいのか、消したいのかで使うコマンドが決まります。

> **情報工学メモ：stdin/stdout/stderr、パイプ（`|`）とリダイレクト（`>`）**  
> すべてのコマンドは3つの「通り道」を持っています。 **stdin** （標準入力）はキーボードなどからの入力、 **stdout** （標準出力）は正常な結果の出力先（通常は画面）、 **stderr** （標準エラー出力）はエラーメッセージの出力先です。 **パイプ（`|`）** は「左のコマンドの stdout を右のコマンドの stdin につなぐ」仕組みです（例：`ps aux | grep node` は `ps aux` の結果を `grep` に渡す）。 **リダイレクト（`>`）** は stdout をファイルに書き出します（例：`ls > files.txt`）。これらを組み合わせることで、複雑な処理を短いコマンドで表現できます。

ターミナルに **コマンド（命令文）** を入力して、`Enter` キーを押すと実行されます。

コマンド例では、よく行頭に `$` が書かれます。これは実際に入力する文字ではなく、「ここからコマンドを入力する」という目印です。

!!! info ""
    ```
    $ コマンド名 オプション 対象
    ```

> 行頭の `$` は「ここに入力してください」というサインです。 **入力するときは `$` を含めないでください。**

## まず覚えるコマンド

### 今どこにいるか確認する

ターミナルでは、常に「今どのフォルダにいるか」が重要です。コマンドは現在地を基準に動くことが多いため、迷ったらまず `pwd` を実行します。

```bash
pwd
```

> **P** rint **W** orking **D** irectory の略。現在いるフォルダの場所（パス）を表示します。

```
/Users/yamada/Documents/ota_hp
```

この出力は、現在の作業ディレクトリの絶対パスです。`ota_hp` の中にいるなら、この場所を基準に `ls` や `cd` が動きます。

### フォルダの中を見る

現在地にどんなファイルやフォルダがあるかを確認するときは `ls` を使います。移動する前、ファイルを開く前、コマンドを実行する前に中身を確認する習慣をつけると安全です。

```bash
ls
```

> **L** i **s** t の略。今いるフォルダのファイル・フォルダ一覧を表示します。

```
README.md   src/   public/   package.json
```

末尾に `/` が付いているものはフォルダとして表示されることがあります。`package.json` のように拡張子があるものはファイルです。

### フォルダを移動する

`cd` は、作業する場所を変えるコマンドです。ターミナル操作では「移動してから実行する」ことが多いため、最初に覚えるべき重要コマンドです。

```bash
cd フォルダ名
```

> **C** hange **D** irectory の略。指定したフォルダに移動します。

```bash
cd Documents          # Documents フォルダに移動
cd ota_hp             # ota_hp フォルダに移動
cd ..                 # 一つ上のフォルダに戻る
cd                    # ホームディレクトリ（/Users/自分の名前）に移動
```

`cd ..` は 1 つ上のフォルダへ戻ります。`cd` だけを実行するとホームディレクトリへ戻るため、迷子になったときの復帰にも使えます。

### よく使う組み合わせ

実際の作業では、現在地を確認し、目的のフォルダへ移動し、中身を見る、という流れを何度も繰り返します。

```bash
# 今どこにいるか確認して、フォルダの中を見る
pwd
ls

# ota_hp に移動して中を確認する
cd ~/Documents/ota_hp
ls
```

この組み合わせを読めれば、ほとんどの開発手順書の最初の部分が理解できます。コマンドを覚えるより先に、「現在地を確認して、移動して、中を見る」という流れを身体で覚えると楽になります。

## パスとは

**ファイルやフォルダの住所** のことです。

パスは、ファイルやフォルダがどこにあるかを表す文字列です。ターミナルでは、クリックで場所を選ぶ代わりに、この住所を文字で指定します。

!!! info ""
    ```
    /Users/yamada/Documents/ota_hp/src/pages/index.astro
     └── ルート
             └── ユーザー名
                      └── フォルダ
                               └── プロジェクト
                                        └── ファイル
    ```

この例では、`/Users/yamada` がユーザーのホーム周辺、`Documents/ota_hp` がプロジェクト、`src/pages/index.astro` がプロジェクト内のファイルを表しています。

| 記号 | 意味 |
|------|------|
| `/`（先頭） | ルート（一番上） |
| `~` | ホームディレクトリ（`/Users/自分の名前`）の省略形 |
| `..` | 一つ上のフォルダ |
| `.` | 今いるフォルダ |

## タブ補完を使おう

フォルダ名やファイル名の途中まで入力して **`Tab`** キーを押すと、自動で補完されます。

タブ補完は、入力を速くするだけでなく、存在しないファイル名を打ってしまうミスも減らします。長いフォルダ名ほど積極的に使ってください。

```bash
cd Do  → Tab キーを押す → cd Documents/  と補完される
```

スペルを間違えにくくなるので、積極的に使いましょう。

## 困ったときの対処法

### コマンドが終わらない

長い処理が走っているか、何かを待っている状態です。

→ `Ctrl + C` を押すと処理を中断できます。

### 文字を打っても何も表示されない

パスワード入力待ちのことがあります（セキュリティのため文字が表示されない）。  
→ パスワードを入力して Enter を押してみてください。

### ターミナルの画面をきれいにしたい

画面にログや出力が増えて見づらいときは、表示だけを消して整理できます。実行結果の履歴が消えるわけではなく、画面が上に流れるだけです。

```bash
clear
```

## VS Code のターミナル

**Visual Studio Code** （VS Code）というエディタには、ターミナルが内蔵されています。エディタとターミナルを切り替えなくて済むので便利です。

`Ctrl + `` ` （バッククォート）で開閉できます。
