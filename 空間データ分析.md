## 地理データの基本

### 座標系（CRS: Coordinate Reference System）

!!! warning ""
    WGS84 (EPSG:4326):
      世界標準の緯度・経度（GPS で使われる）
      例: 東京駅 → 緯度 35.6812, 経度 139.7671
    
    JGD2011 (EPSG:6668):
      日本の測量基準（国土地理院が定めた日本専用）
    
    Web Mercator (EPSG:3857):
      Google Maps・OpenStreetMap で使われる平面座標
      距離・面積の計算には不向き
    
    注意: 距離や面積を計算するときは平面直角座標系（メートル単位）に変換する

### 主なデータ形式

| 形式 | 特徴 | 用途 |
|------|------|------|
| **GeoJSON** | JSON ベース・Web 標準 | Web マップ・API |
| **Shapefile (.shp)** | 業界標準・複数ファイル | GIS ソフト・政府データ |
| **GeoParquet** | 列指向・高速 | 大規模データ |
| **CSV + 緯度経度列** | シンプル | ポイントデータ |

## ポイントデータの可視化

```python
import geopandas as gpd
from shapely.geometry import Point
import pandas as pd
import matplotlib.pyplot as plt
import contextily as ctx   # 背景地図タイル

# 架空の店舗データ（東京都内）
stores = pd.DataFrame({
    "name":    ["渋谷店", "新宿店", "池袋店", "品川店", "上野店"],
    "lon":     [139.7018, 139.6994, 139.7100, 139.7398, 139.7741],
    "lat":     [ 35.6580,  35.6896,  35.7299,  35.6285,  35.7146],
    "sales":   [1200, 980, 1500, 750, 890],   # 万円/月
})

# GeoDataFrame に変換
gdf = gpd.GeoDataFrame(
    stores,
    geometry=gpd.points_from_xy(stores["lon"], stores["lat"]),
    crs="EPSG:4326"
)

# 売上規模を点のサイズに反映
fig, ax = plt.subplots(figsize=(8, 8))
gdf.to_crs("EPSG:3857").plot(
    ax=ax,
    markersize=gdf["sales"] / 30,
    column="sales",
    cmap="YlOrRd",
    legend=True,
    alpha=0.8,
)
ctx.add_basemap(ax, source=ctx.providers.OpenStreetMap.Mapnik)
ax.set_title("東京都内の店舗売上（サイズ = 売上規模）")
plt.show()
```

## 空間結合（Spatial Join）

「ポイントデータをポリゴン（地域）に紐付ける」操作です。

```python
import geopandas as gpd
from shapely.geometry import Point
import pandas as pd

# 国土数値情報（行政区域データ）を使う例
# 実際のデータは https://nlftp.mlit.go.jp/ からダウンロード
# ここでは架空の区データを作成

from shapely.geometry import Polygon

# 架空の区ポリゴン
wards = gpd.GeoDataFrame({
    "ward": ["渋谷区", "新宿区", "豊島区"],
    "geometry": [
        Polygon([(139.68, 35.64), (139.73, 35.64), (139.73, 35.67), (139.68, 35.67)]),
        Polygon([(139.68, 35.67), (139.73, 35.67), (139.73, 35.71), (139.68, 35.71)]),
        Polygon([(139.68, 35.71), (139.73, 35.71), (139.73, 35.75), (139.68, 35.75)]),
    ]
}, crs="EPSG:4326")

# 空間結合: 各店舗が属する区を特定
joined = gpd.sjoin(gdf, wards, how="left", predicate="within")
print(joined[["name", "sales", "ward"]])

# 区ごとに集計
ward_sales = joined.groupby("ward")["sales"].agg(["sum", "count", "mean"])
print("\n区別の売上集計:")
print(ward_sales)
```

import folium
from folium.plugins import HeatMap
import pandas as pd

# インタラクティブ地図の作成
m = folium.Map(location=[35.68, 139.70], zoom_start=11)

# マーカーを追加
for _, row in gdf.iterrows():
    folium.CircleMarker(
        location=[row["lat"], row["lon"]],
        radius=row["sales"] / 100,
        popup=f"{row['name']}: {row['sales']}万円",
        color="crimson",
        fill=True,
        fill_color="crimson",
        fill_opacity=0.7,
    ).add_to(m)

# ヒートマップ
heat_data = [[row["lat"], row["lon"], row["sales"]] for _, row in gdf.iterrows()]
HeatMap(heat_data, radius=40).add_to(m)

m.save("store_map.html")   # HTML として保存、ブラウザで開ける

## 空間的自己相関（Moran's I）

「近い場所ほど似た値を持つか」を測る統計指標です。感染症の拡散・不動産価格・所得分布などの分析に使います。

$$
I = \frac{n \sum_i \sum_j w_{ij}(y_i - \bar{y})(y_j - \bar{y})}{W \sum_i (y_i - \bar{y})^2}
$$

- $I > 0$：空間的正の自己相関（近隣が似た値）
- $I \approx 0$：空間的無相関（ランダム配置）
- $I < 0$：空間的負の自己相関（チェッカーボード状）

```python
# pip install pysal libpysal esda
import libpysal
from esda.moran import Moran
import geopandas as gpd

# サンプルデータ（米国の失業率）
guerry = libpysal.examples.load_example('Guerry')
path = guerry.get_path('guerry.shp')
gdf_guerry = gpd.read_file(path)

# 空間重み行列（Queen 接続 = 辺または頂点を共有する隣接）
w = libpysal.weights.Queen.from_dataframe(gdf_guerry)
w.transform = 'R'   # 行標準化

# Moran's I を計算
y = gdf_guerry['Donatns']   # 寄付額（例）
moran = Moran(y, w)

print(f"Moran's I: {moran.I:.4f}")
print(f"p 値: {moran.p_sim:.4f}")
# I > 0 かつ p < 0.05 なら「地理的なクラスタリングが存在する」
```

## 実践：日本の人口密度分析

```python
# e-Stat（政府統計）から都道府県別データを取得
import pandas as pd
import geopandas as gpd
import requests

# 都道府県 GeoJSON（GitHub の公開データを使う）
url = "https://raw.githubusercontent.com/dataofjapan/land/master/japan.geojson"
japan_prefectures = gpd.read_file(url)
print(japan_prefectures.head())
print(japan_prefectures.columns.tolist())

# 人口データを結合して可視化
# （実際は e-Stat API や CSV から取得）
population_data = pd.DataFrame({
    "nam_ja": ["東京都", "大阪府", "愛知県"],  # 都道府県名
    "population": [14000000, 8800000, 7500000],
})
merged = japan_prefectures.merge(population_data, on="nam_ja", how="left")

fig, ax = plt.subplots(figsize=(8, 10))
merged.plot(column="population", cmap="YlOrRd", legend=True,
            missing_kwds={"color": "lightgray"}, ax=ax)
ax.set_title("都道府県別人口")
ax.axis("off")
plt.show()
```
