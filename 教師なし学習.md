## クラスタリング

クラスタリングは、正解ラベルがないデータを似たもの同士のグループに分ける方法です。顧客の購買傾向、文章の内容、センサーデータのパターンなどを探索するときに使えます。

ただし、クラスタ番号そのものに最初から意味があるわけではありません。分かれたグループの特徴を人間が確認し、「高頻度購入者」「価格重視」「外れ値に近い」などの解釈を後から与えます。

### k-means

最も有名なクラスタリング手法です。k 個のクラスタ中心（セントロイド）に近いデータ点を割り当てます。

#### アルゴリズムを手計算で追う

4 点のデータを 2 つのクラスタに分ける例です。

```
データ点（2 次元）:
  A = (1, 1)
  B = (1, 2)
  C = (5, 1)
  D = (6, 2)

目標: k=2 でクラスタリングする
```

**■ STEP 1: クラスタ中心を初期化（ランダムに 2 点を選ぶ）**

```
中心 C1 = (1, 1) ← 点 A を初期中心とする
中心 C2 = (5, 1) ← 点 C を初期中心とする
```

**■ STEP 2: 各点を「最も近い中心」に割り当て**

距離の計算（ユークリッド距離: $\sqrt{(x_2-x_1)^2 + (y_2-y_1)^2}$）:

```
点 A=(1,1):  C1 まで = 0.0,   C2 まで = 4.0   → C1 に割り当て
点 B=(1,2):  C1 まで = 1.0,   C2 まで ≈ 4.1   → C1 に割り当て
点 C=(5,1):  C1 まで = 4.0,   C2 まで = 0.0   → C2 に割り当て
点 D=(6,2):  C1 まで ≈ 5.1,   C2 まで = 1.4   → C2 に割り当て

グループ1: {A, B}  グループ2: {C, D}
```

**■ STEP 3: 各グループの平均で中心を更新**

「グループが決まったら、その中心を再計算する」という発想です。最初に設定した中心は仮のものでした。実際にグループ内のデータの平均点が「真の中心」に近いはずです。ですから平均で中心を更新します。

```
新しい C1 = A と B の平均 = ((1+1)/2, (1+2)/2) = (1.0, 1.5)
新しい C2 = C と D の平均 = ((5+6)/2, (1+2)/2) = (5.5, 1.5)
```

**■ STEP 4: 割り当てを再計算 → 変化がなければ終了**

```
C1=(1.0,1.5), C2=(5.5,1.5) で再割り当て：

点 A=(1,1): C1まで 0.5, C2まで 4.5 → C1
点 B=(1,2): C1まで 0.5, C2まで 4.5 → C1
点 C=(5,1): C1まで 4.1, C2まで 0.7 → C2
点 D=(6,2): C1まで 5.1, C2まで 0.7 → C2

→ 変化なし → 収束！
グループ1: {A, B}   グループ2: {C, D}
```

この「割り当て → 中心更新 → 再割り当て」を繰り返すのが k-means です。なぜこれが収束するのか直感的に考えると——中心を更新するたびに「各点が最も近い中心」という条件を満たすよう割り当てが改善され、「割り当てに対して最適な中心（平均）」という条件も満たされます。この2つの最適化を交互に行うことで、クラスタ内の距離の総和が単調に減少し、必ず有限ステップで収束することが証明されています。

```python
import numpy as np

# 上の例を NumPy で確認
X_example = np.array([[1,1],[1,2],[5,1],[6,2]], dtype=float)
centers   = np.array([[1,1],[5,1]], dtype=float)  # 初期中心

for iteration in range(5):
    # 各点と各中心の距離を計算
    dists = np.array([[np.linalg.norm(x - c) for c in centers] for x in X_example])
    labels = dists.argmin(axis=1)  # 最も近い中心のインデックス

    # 中心を更新
    new_centers = np.array([X_example[labels==k].mean(axis=0) for k in range(2)])

    print(f"iter {iteration}: labels={labels}  centers={new_centers.round(2)}")
    if np.allclose(centers, new_centers):
        print("収束！")
        break
    centers = new_centers
```

#### スケーリングが必須な理由

```
身長（cm）と体重（kg）で人をクラスタリングする場合：
  身長: 170cm vs 175cm → 差 = 5
  体重:  60kg vs  75kg → 差 = 15

スケーリングなし → 体重の差（大きい値）が距離を支配
スケーリング後  → 身長も体重も同じ尺度で比較できる
```

次のコードでは、まずスケーリングを行い、次にエルボー法でクラスタ数の候補を調べ、最後に k=3 でクラスタリングします。k-means は距離に基づくため、スケーリングが特に重要です。

```python
from sklearn.cluster import KMeans
from sklearn.preprocessing import StandardScaler
import matplotlib.pyplot as plt
import numpy as np

# データの準備（スケーリングは必須）
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# エルボー法で最適な k を探す
inertias = []
for k in range(1, 11):
    km = KMeans(n_clusters=k, random_state=42, n_init=10)
    km.fit(X_scaled)
    inertias.append(km.inertia_)

plt.plot(range(1, 11), inertias, marker="o")
plt.xlabel("クラスタ数 k")
plt.ylabel("慣性（Inertia）")
plt.title("エルボー法")
plt.savefig("elbow.png", dpi=150)
# → 急激に減少が鈍る「肘」の部分が最適な k

# クラスタリング実行
km = KMeans(n_clusters=3, random_state=42, n_init=10)
labels = km.fit_predict(X_scaled)
```

`inertia_` は、各点が自分のクラスタ中心からどれくらい離れているかの合計です。クラスタ数を増やすほど小さくなりますが、ある地点から改善が鈍くなります。その曲がり角を「肘」と見て、k の候補にします。

> **情報工学メモ：k-means の限界**  
> k-means は球形のクラスタを想定しており、クラスタの形が複雑（三日月形・リング形）な場合はうまく機能しません。また k を事前に指定する必要があります。複雑な形状には DBSCAN、階層的クラスタリング（Ward 法）が有効です。

### DBSCAN（密度ベース）

k を指定せず、密度が高い領域を自動でクラスタとして検出します。外れ値も自動検出します。

DBSCAN は、「近くに十分な数の点が集まっている領域」をクラスタとみなします。丸いクラスタだけでなく、複雑な形のかたまりにも対応しやすい一方、`eps` と `min_samples` の調整が重要です。

```python
from sklearn.cluster import DBSCAN

dbscan = DBSCAN(eps=0.5, min_samples=5)
labels = dbscan.fit_predict(X_scaled)

# ラベル -1 が外れ値
print(f"クラスタ数: {len(set(labels)) - (1 if -1 in labels else 0)}")
print(f"外れ値数: {(labels == -1).sum()}")
```

`eps` は近傍とみなす距離、`min_samples` は密な領域と判断するために必要な点の数です。ラベル `-1` は、どのクラスタにも属さない外れ値を表します。

### ガウス混合モデル（GMM）

k-means がクラスタを「中心からの距離」で硬く割り当てるのに対し、GMM は「各クラスタに属する確率」を返します。楕円形のクラスタや、境界があいまいなデータに向いています。

```python
from sklearn.mixture import GaussianMixture
import matplotlib.pyplot as plt

# GMM：各サンプルがどのガウス分布に属するかを確率で返す
gmm = GaussianMixture(n_components=3, covariance_type="full", random_state=42)
gmm.fit(X_scaled)

labels = gmm.predict(X_scaled)            # 最尤のクラスタ番号
proba  = gmm.predict_proba(X_scaled)      # 各クラスタへの所属確率
bic    = gmm.bic(X_scaled)               # BIC でクラスタ数を選ぶ指標

print(f"BIC: {bic:.1f}")
print(f"サンプル0の所属確率: {proba[0].round(3)}")
```

`covariance_type` を変えると楕円の形を制限できます（`"diag"` で軸平行の楕円、`"spherical"` で球形）。クラスタ数は AIC / BIC が最小になる k を探します。

| 手法 | クラスタの形 | 確率出力 | ノイズ処理 |
|------|------------|---------|-----------|
| k-means | 球形 | なし | なし |
| GMM | 任意の楕円 | あり | なし |
| DBSCAN | 任意の形 | なし | あり（-1） |

### 階層的クラスタリング（Ward 法）

k を事前に決めずに、似たデータを段階的に結合していく手法です。デンドログラム（樹形図）を見て適切なクラスタ数を後から決められます。

```python
from scipy.cluster.hierarchy import linkage, dendrogram, fcluster
import matplotlib.pyplot as plt

# Ward 法：クラスタ内分散が最小になるように結合
Z = linkage(X_scaled, method="ward")

# デンドログラムで切る高さを決める
plt.figure(figsize=(12, 5))
dendrogram(Z, truncate_mode="lastp", p=20, show_leaf_counts=True)
plt.title("デンドログラム（Ward 法）")
plt.xlabel("サンプル")
plt.ylabel("距離")
plt.savefig("dendrogram.png", dpi=150)

# 切断高さからクラスタ数を決定（distance_threshold で指定）
labels = fcluster(Z, t=3, criterion="maxclust")  # クラスタ数を 3 に指定
```

デンドログラムで縦方向に長い水平線（分岐間の距離が大きいところ）を探し、そこを「切断高さ」とすると自然なクラスタ数が見えます。

## 次元削減

### PCA（主成分分析）

データの分散が最大になる方向に新しい軸（主成分）を作ります。次元を圧縮しながら情報をできるだけ保持します。

PCA は、高次元データを少ない軸で表現するための線形変換です。たくさんの特徴量を 2 次元に落として可視化したり、ノイズを減らしたりする目的で使います。

```python
from sklearn.decomposition import PCA

pca = PCA(n_components=2)   # 2 次元に圧縮
X_2d = pca.fit_transform(X_scaled)

# 各主成分が説明する分散の割合
print(f"寄与率: {pca.explained_variance_ratio_}")
print(f"累積寄与率: {pca.explained_variance_ratio_.cumsum()}")

# 2D 散布図でクラスタを可視化
plt.scatter(X_2d[:, 0], X_2d[:, 1], c=labels, cmap="tab10", s=20)
plt.xlabel("第1主成分")
plt.ylabel("第2主成分")
plt.savefig("pca_2d.png", dpi=150)
```

`explained_variance_ratio_` は、各主成分が元データの分散をどれくらい説明しているかを表します。2 次元に落とした結果を散布図にすると、クラスタの分かれ方や外れ値を確認できます。

### 何次元まで落とすか

次元削減では、どこまで情報を残すかを決める必要があります。累積寄与率を見ると、元データの分散を何% 保てるかが分かります。

```python
pca_full = PCA()
pca_full.fit(X_scaled)

# 累積寄与率 95% を達成する次元数を確認
cumsum = np.cumsum(pca_full.explained_variance_ratio_)
n_components = np.argmax(cumsum >= 0.95) + 1
print(f"95% を説明するのに必要な次元数: {n_components}")
```

この例では、累積寄与率が 95% に到達する最小の次元数を求めています。可視化なら 2〜3 次元、前処理なら 90〜99% など、目的に応じて基準を選びます。

### t-SNE（非線形次元削減）

PCA より複雑な構造を 2D/3D で可視化できます。データの「かたまり」の確認に使います（距離の絶対値に意味はない）。

t-SNE は、近い点同士を低次元でも近くに置くことを重視する可視化手法です。クラスタの「見た目」を確認するには便利ですが、軸の値やクラスタ間の距離を厳密に解釈しすぎないようにします。

```python
from sklearn.manifold import TSNE

tsne = TSNE(n_components=2, perplexity=30, random_state=42, n_iter=1000)
X_tsne = tsne.fit_transform(X_scaled)

plt.scatter(X_tsne[:, 0], X_tsne[:, 1], c=y, cmap="tab10", s=10, alpha=0.7)
plt.colorbar()
plt.title("t-SNE 可視化")
plt.savefig("tsne.png", dpi=150)
```

`perplexity` は、近傍の広さに関わるパラメータです。値によって見え方が変わるため、複数の値で確認することがあります。

### UMAP（高速な非線形次元削減）

t-SNE より高速で大規模データにも使えます。

UMAP は、局所構造を保ちながら高速に低次元へ写像する手法です。大きめのデータを可視化したいときに使われます。

```python
import umap  # pip install umap-learn

reducer = umap.UMAP(n_components=2, random_state=42)
X_umap = reducer.fit_transform(X_scaled)
```

## 異常検知

正常なデータで学習し、外れたデータを異常として検出します。

異常検知では、正解ラベルが十分にないことが多いため、「通常のデータから見て孤立しているか」を基準にします。Isolation Forest は、少ない分割で孤立するデータを異常とみなします。

```python
from sklearn.ensemble import IsolationForest

iso = IsolationForest(contamination=0.05, random_state=42)  # 5% を外れ値と想定
labels = iso.fit_predict(X_scaled)
# -1: 異常  1: 正常

anomalies = X[labels == -1]
print(f"検出した異常件数: {len(anomalies)}")
```

`contamination=0.05` は、全体の 5% 程度を外れ値とみなす想定です。この値は業務上の許容範囲や実際の異常率に合わせて調整します。
