# 認証・認可

Web アプリケーションで「誰がアクセスしているか」を確認し（認証）、「何をしてよいか」を制御する（認可）仕組みです。ログイン機能・API の保護・ロール管理の基本を扱います。

---

## はじめて読む人へ

「ログインしているかどうかの確認」と「管理者だけが使える機能の制限」は別の問題です。どちらも大切ですが、混在させると設計が崩れます。

### 読む前に押さえること

- HTTP のリクエスト・レスポンスの基本（[ネットワーク基礎](ネットワーク基礎.md) 参照）
- [FastAPI](FastAPI.md) の基本的なルーティングを知っていること

### 読み終えたら説明できること

- 認証と認可の違いを説明できる
- JWT の構造（header.payload.signature）を説明できる
- FastAPI で JWT を発行・検証する実装の流れを説明できる

---

## 認証と認可の違い

| | 認証（Authentication） | 認可（Authorization） |
|--|----------------------|----------------------|
| 問い | 「あなたは誰ですか？」 | 「あなたは何をしてよいですか？」 |
| 例 | ログイン・パスワード検証 | 管理者のみ削除可・自分の投稿のみ編集可 |
| 英略 | AuthN | AuthZ |
| 順序 | 先に行う | 認証の後に行う |

認証なしに認可はできません。認証が通った後、そのユーザーの権限を確認するのが認可です。

## セッション認証とトークン認証

### セッション認証

サーバー側がセッション情報を保持し、クライアントには Cookie で ID のみ渡します。

1. ログイン → サーバーがセッションをDBに保存 → session_id を Cookie に
2. リクエスト → Cookie の session_id をサーバーが照合 → 誰かを特定
3. ログアウト → サーバーがセッションを削除
**特徴：**
- サーバーが状態を持つ（スケールアウトに工夫が必要）
- Cookie を使うので CSRF 対策が必要
- セッションを即座に無効化できる

### トークン認証（JWT）

サーバーが状態を持たず、署名付きトークンをクライアントに渡します。

1. ログイン → サーバーがJWTを発行 → クライアントがローカルに保存
2. リクエスト → Authorization ヘッダーにJWTを付ける
3. サーバーが署名を検証 → 誰かを特定（DBを見ない）
**特徴：**
- サーバーがステートレス（スケールアウトしやすい）
- API・マイクロサービスとの相性が良い
- トークンを即座に無効化できない（有効期限まで有効）

## JWT（JSON Web Token）

JWT は `header.payload.signature` の 3 つをドットで繋いだ文字列です。

```
eyJhbGciOiJIUzI1NiJ9.eyJzdWIiOiJ1c2VyMTIzIn0.SflKxwRJSMeKKF2QT4fwpMeJf36POk6yJV_adQssw5c
```

### header（アルゴリズム情報）

```json
{
  "alg": "HS256",
  "typ": "JWT"
}
```

### payload（クレーム：運ぶデータ）

```json
{
  "sub": "user123",
  "email": "user@example.com",
  "role": "editor",
  "exp": 1735689600
}
```

`sub`（subject）はユーザー ID、`exp` は有効期限（Unix タイムスタンプ）です。payload は Base64 エンコードされているだけで**暗号化されていません**。秘密情報（パスワード等）を入れてはいけません。

### signature（署名）

HMACSHA256(
base64(header) + "." + base64(payload),
秘密鍵
)
サーバーだけが知る秘密鍵で署名します。これにより、クライアントが payload を改ざんしても署名が一致せず、サーバーが検知できます。

## FastAPI での JWT 実装

```bash
pip install python-jose[cryptography] passlib[bcrypt]
```

### トークン発行

```python
from datetime import datetime, timedelta
from jose import jwt
from passlib.context import CryptContext
from fastapi import FastAPI, HTTPException
from pydantic import BaseModel

SECRET_KEY = "your-secret-key-here"  # 本番では環境変数に
ALGORITHM = "HS256"
ACCESS_TOKEN_EXPIRE_MINUTES = 30

app = FastAPI()
pwd_context = CryptContext(schemes=["bcrypt"])

# 仮のユーザーDB
USERS_DB = {
    "alice@example.com": {
        "hashed_password": pwd_context.hash("password123"),
        "role": "editor"
    }
}

class LoginRequest(BaseModel):
    email: str
    password: str

def create_token(data: dict, expires_delta: timedelta) -> str:
    payload = data.copy()
    payload["exp"] = datetime.utcnow() + expires_delta
    return jwt.encode(payload, SECRET_KEY, algorithm=ALGORITHM)

@app.post("/login")
def login(req: LoginRequest):
    user = USERS_DB.get(req.email)
    if not user or not pwd_context.verify(req.password, user["hashed_password"]):
        raise HTTPException(status_code=401, detail="メールアドレスまたはパスワードが違います")

    token = create_token(
        {"sub": req.email, "role": user["role"]},
        timedelta(minutes=ACCESS_TOKEN_EXPIRE_MINUTES)
    )
    return {"access_token": token, "token_type": "bearer"}
```

### トークン検証（依存性注入）

```python
from fastapi import Depends
from fastapi.security import HTTPBearer, HTTPAuthorizationCredentials
from jose import JWTError

security = HTTPBearer()

def get_current_user(credentials: HTTPAuthorizationCredentials = Depends(security)):
    try:
        payload = jwt.decode(credentials.credentials, SECRET_KEY, algorithms=[ALGORITHM])
        email: str = payload.get("sub")
        role: str = payload.get("role")
        if email is None:
            raise HTTPException(status_code=401, detail="不正なトークンです")
        return {"email": email, "role": role}
    except JWTError:
        raise HTTPException(status_code=401, detail="トークンが無効です")

# 認証が必要なエンドポイント
@app.get("/me")
def get_me(current_user = Depends(get_current_user)):
    return current_user

# 認可：ロールチェック
@app.delete("/posts/{post_id}")
def delete_post(post_id: int, current_user = Depends(get_current_user)):
    if current_user["role"] != "admin":
        raise HTTPException(status_code=403, detail="管理者のみ削除できます")
    return {"deleted": post_id}
```

## OAuth2（外部サービスによる認証）

「Google / GitHub でログイン」の仕組みです。自前でパスワード管理をせずに済みます。

1. ユーザーが「Google でログイン」をクリック
2. Google の認証画面にリダイレクト
3. ユーザーが Google で認証 → 認可コードが返る
4. サーバーが認可コードと access_token を交換（Google API を呼ぶ）
5. access_token でユーザー情報（メール等）を取得
6. 自前 JWT を発行してログイン完了
```python
# authlib を使った例（FastAPI + Google OAuth）
pip install authlib httpx

from authlib.integrations.starlette_client import OAuth

oauth = OAuth()
oauth.register(
    name="google",
    client_id="YOUR_CLIENT_ID",
    client_secret="YOUR_CLIENT_SECRET",  # 環境変数に
    server_metadata_url="https://accounts.google.com/.well-known/openid-configuration",
    client_kwargs={"scope": "openid email profile"},
)
```

## セキュリティの注意点

| 注意点 | 理由 |
|--------|------|
| `SECRET_KEY` は環境変数に | コードに書くと漏洩する |
| トークンの有効期限を短くする | 漏洩時の被害を限定する（30分〜1時間が目安）|
| HTTPS 必須 | トークンを平文で送らない |
| payload に機密情報を入れない | Base64 は暗号化ではない |
| リフレッシュトークンを使う | access_token を短命に、refresh_token で更新 |
