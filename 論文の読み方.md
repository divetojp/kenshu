# 論文・技術文書の読み方

**論文を読む力は、学び続ける力**です。授業や教科書の内容はすでに「枯れた知識」ですが、論文は最新の知見に直接アクセスする手段です。最初は難しく感じますが、読み方のコツを知れば効率よく情報を取り出せるようになります。

---

## はじめて読む人へ

「論文を読もうとしたけど数式で詰まった」「英語が難しくて挫折した」——よくある経験です。でも、論文を**全部精読する必要はありません**。目的に合わせて読む場所を選ぶことが大切です。

### 読む前に押さえること

特に前提知識は不要です。3・4年生向けですが、2年生でも読めます。

### 読み終えたら説明できること

- 論文の各セクションが何のためにあるか説明できる
- 論文の効率的な読み順を説明できる
- arXiv・Google Scholar で論文を検索・管理できる

---

## 論文の構造

ほとんどの論文は同じ構造を持っています。最初にこの「地図」を頭に入れておくと、どこを読めばよいかわかります。

| セクション | 書いてあること | 最初に読む優先度 |
|-----------|--------------|---------------|
| **Abstract（概要）** | 問い・方法・結果・意義を 150〜250 語で要約 | ★★★（必ず読む） |
| **Introduction（序論）** | 背景・先行研究・本論文の貢献 | ★★★ |
| **Related Work（関連研究）** | 先行研究との比較 | ★☆☆（後で読む） |
| **Method（方法）** | 提案手法の詳細・数式 | ★★☆ |
| **Experiments / Results（実験・結果）** | データセット・評価指標・性能比較 | ★★★ |
| **Discussion（考察）** | 結果の解釈・限界 | ★★☆ |
| **Conclusion（結論）** | まとめと今後の課題 | ★★★（必ず読む） |

---

## 論文の効率的な読み順

### 初めて読むとき：「スキャン読み」

全部を読もうとせず、まず**15〜20 分で全体像を掴む**ことを目標にします。

!!! info ""
    ステップ 1（3分）: タイトルと Abstract を読む
    → 何について・何を提案して・どんな結果か を把握

    ステップ 2（5分）: Introduction の最初と最後の段落を読む
    → 背景と「本論文の貢献（Our contributions）」を確認

    ステップ 3（5分）: 図・表をすべて眺める
    → キャプションを読んで、主な結果を把握

    ステップ 4（5分）: Conclusion を読む
    → 何が示されたか・今後の課題は何か

    → この時点で「この論文は自分に必要か」を判断する
### 必要と判断したら：「精読」

!!! info ""
    ステップ 5: Introduction 全体を読む
    ステップ 6: Method を読む（数式が難しければ最初はスキップ）
    ステップ 7: Experiments・Results を丁寧に読む
    ステップ 8: Discussion を読む
    ステップ 9: 数式が必要なら Method に戻る

## 論文を探す方法

### Google Scholar

最も広く使われる学術検索エンジンです。

!!! info ""
    ```text
    https://scholar.google.com/

    検索のコツ:
      キーワード + site:arxiv.org  → arXiv の論文に絞る
      "exact phrase"               → 完全一致で検索
      author:Goodfellow            → 著者で絞る
      after:2022                   → 2022年以降に絞る
    ```
### arXiv（アーカイブ）

機械学習・AI・数学・物理など理系分野の**プレプリント**（査読前の論文）サーバーです。最新の研究に無料でアクセスできます。

!!! info ""
    https://arxiv.org/

    **カテゴリ**

    cs.LG  → 機械学習
    cs.AI  → 人工知能
    cs.CV  → コンピュータビジョン
    cs.CL  → 自然言語処理
    stat.ML → 統計的機械学習

    arXiv 番号の形式: 2301.12345（年・月・連番）
### Semantic Scholar / Paperswithcode

!!! info ""
    https://www.semanticscholar.org/
    → 引用関係の可視化・関連論文の推薦が優秀

    https://paperswithcode.com/
    → 論文と実装コード（GitHub）がセットで検索できる
    → State of the Art（最新の最高性能）が一覧できる

## 論文管理ツール

### Zotero（無料・推奨）

!!! info ""
    https://www.zotero.org/

    - ブラウザ拡張機能で論文を 1 クリックで保存
    - PDF・メモ・タグを一元管理
    - 参考文献リストを自動生成（卒論に便利）
    - チームで共有も可能
### Notion / Obsidian でメモを取る

論文ごとにメモページを作る習慣をつけましょう。

```markdown
# [論文タイトル]
- **著者**: Goodfellow et al.
- **年**: 2014
- **URL**: https://arxiv.org/abs/...

## 問いと貢献
GAN（生成的敵対ネットワーク）の提案。
生成器と識別器を競わせることでリアルな画像を生成する。

## 手法の要点
- Generator G: ランダムノイズ → 偽画像
- Discriminator D: 本物か偽物かを判定
- min-max ゲーム: min_G max_D V(D,G)

## 実験結果
MNIST・CIFAR-10 で先行手法を上回る画像品質

## 自分の感想・疑問
- GAN の学習不安定問題（mode collapse）はどう対処する？
- Wasserstein GAN（後の論文）で解決される

## 引用したい箇所
"Generative adversarial nets... p.2"
```

## 英語論文を読むコツ

### よく出る表現パターン

!!! info ""
    【Introduction でよく出る構文】
    X is important because ...  → X が重要な理由
    However, existing methods have ...  → 先行研究の課題
    In this paper, we propose ...  → 本論文の提案
    Our main contributions are as follows:  → 貢献の列挙

    【Results でよく出る表現】
    Table 1 shows that ...  → 表の説明
    outperforms / achieves better performance than  → 上回る
    state-of-the-art  → 最新最高の手法

    【Conclusion でよく出る表現】
    In this work, we have demonstrated that ...  → まとめ
    A limitation of our approach is ...  → 限界
    Future work includes ...  → 今後の課題
### 辞書・翻訳ツールの使い方

!!! info ""
    DeepL（https://www.deepl.com）
    → 段落単位でコピペ翻訳。精度が高い

    Google 翻訳
    → PDF の翻訳機能が便利（ファイルをアップロードすると丸ごと翻訳）

    ChatGPT / Claude
    → 「この文章の意味を日本語で説明してください」と聞くと
    文脈を考慮した解釈が得られる
**注意：** 機密情報・未公開データを AI に貼り付けない（→ [データ倫理](データ倫理.md)）

## 数式で詰まったとき

数式を全部理解する必要はありません。まずは**何をしている式か**を把握することを目標にしましょう。

!!! info ""
    ```text
    【読み方の戦略】
    1. 式の左辺（= の左）が何を表しているかを把握する
    2. 右辺の各記号の定義を本文で探す
    3. 式の「形」（log・exp・Σ・argmax）からざっくりした動作を推測
    4. コードや図に対応させて解釈する

    【よく出る記号の意味】
    argmax_θ L(θ)  → L を最大にする θ を求める（最適化）
    ∇_θ L          → L の θ に関する勾配（方向）
    𝔼[X]           → X の期待値（平均）
    KL(P||Q)       → P と Q の分布の違い（KL ダイバージェンス）
    ```

## サーベイ（文献調査）の進め方

卒業研究では、先行研究を体系的に調査する「サーベイ」が必要です。

!!! info ""
    ステップ 1: キーワードで 30〜50 本を検索
    ステップ 2: Abstract だけ読んで、10〜15 本に絞る
    ステップ 3: スキャン読みして、精読すべき 5〜8 本を選ぶ
    ステップ 4: 精読した論文の「Related Work」を読む
    → そこで引用されている論文も重要な論文
    ステップ 5: 引用を追いながら古い論文に遡る
    （重要な論文は多くの論文から引用されている）
**被引用数のチェック：** Google Scholar では被引用数が確認できます。被引用数が多いほど影響力の高い論文です（ただし古い論文ほど自然に高くなる）。

## 確認問題

1. 初めて読む論文で最初に読むべき 3 つのセクションを優先度順に挙げてください。
2. arXiv の論文は査読前の場合がありますが、それでも読む価値がある理由を説明してください。
3. 「相関係数 r=0.65、p<0.01」という結果が書いてあった論文を読んで、どんな点を批判的に確認しますか？
