# 因子分析

**観測できない潜在的な共通因子** を発見するための統計モデルです。PCA が次元削減の道具であるのに対し、因子分析は「なぜ変数間に相関があるのか」という構造の解釈が目的です。アンケート分析・心理尺度・社会科学データでよく使われます。

---

## はじめて読む人へ

「数学・理科・技術の点数は相関している → 背後に『理系的能力』という共通因子があるのでは？」——因子分析はこのように、直接は測れない潜在的な構造を推定します。PCA と見た目が似ていますが目的が違います。PCA は「圧縮」、因子分析は「解釈」です。

- [確率・統計基礎](確率・統計基礎.md) の相関・共分散の概念

### 読み終えたら説明できること

- 因子分析と PCA の「目的」の違いを説明できる
- 因子負荷量・共通性・独自性が何を表すかを説明できる
- バリマックス回転がなぜ解釈性を上げるかを説明できる
- スクリープロットで因子数を決める方法を説明できる

## モデルの導出

### 基本仮定

変数 $x_i$（$i = 1, \ldots, p$）が $m$ 個の共通因子 $F_1, \ldots, F_m$ と独自因子 $\varepsilon_i$ の線形結合で表されると仮定します。

$$
x_i = \lambda_{i1} F_1 + \lambda_{i2} F_2 + \cdots + \lambda_{im} F_m + \varepsilon_i
$$

- $F_j$：潜在因子（直接観測できない）、$E[F_j]=0$、$\text{Var}(F_j)=1$
- $\lambda_{ij}$：因子負荷量（変数 $i$ と因子 $j$ の関係の強さ）
- $\varepsilon_i$：独自因子（$E[\varepsilon_i]=0$、異なる変数間で無相関）

行列形式で書くと：

$$
\mathbf{x} = \Lambda \mathbf{F} + \boldsymbol{\varepsilon}
$$

### 共分散構造から負荷量を求める

モデルの仮定から、$\mathbf{x}$ の共分散行列 $\Sigma$ がどう分解されるかを導きます。

$$
\Sigma = \text{Cov}(\mathbf{x}) = \text{Cov}(\Lambda\mathbf{F} + \boldsymbol{\varepsilon})
= \Lambda \text{Cov}(\mathbf{F}) \Lambda^\top + \text{Cov}(\boldsymbol{\varepsilon})
$$

因子が互いに無相関（$\text{Cov}(\mathbf{F}) = I$）で、独自因子も互いに無相関（$\text{Cov}(\boldsymbol{\varepsilon}) = \Psi$、対角行列）という仮定のもとで：

$$
\boxed{\Sigma = \Lambda \Lambda^\top + \Psi}
$$

この分解が因子分析の核心です。$\Lambda\Lambda^\top$ の部分が共通因子で説明される「共通分散」、$\Psi$ の対角成分 $\psi_i$ が独自因子による「独自分散」を表します。

---

## 共通性・独自性・因子負荷量の意味

変数 $x_i$ の全分散 $\sigma_i^2$ は次のように分解されます。

$$
\sigma_i^2 = \underbrace{\sum_{j=1}^{m} \lambda_{ij}^2}_{h_i^2 \text{（共通性）}} + \underbrace{\psi_i}_{\text{独自分散}}
$$

| 量 | 記号 | 意味 |
|----|------|------|
| 共通性（communality） | $h_i^2 = \sum_j \lambda_{ij}^2$ | 共通因子で説明できる分散の割合 |
| 独自性（uniqueness） | $\psi_i = \sigma_i^2 - h_i^2$ | その変数固有の分散 |
| 因子負荷量 | $\lambda_{ij}$ | 変数 $i$ と因子 $j$ の相関係数（標準化後） |

**例：5 教科の学力テスト**

国語・英語 → 共通因子「言語能力」に高い負荷量
数学・理科・技術 → 共通因子「理系能力」に高い負荷量
体育 → どの共通因子にも低い負荷量 → 独自性が高い
---

## 実装

```python
from sklearn.decomposition import FactorAnalysis
from sklearn.preprocessing import StandardScaler
from sklearn.datasets import load_iris
import pandas as pd
import numpy as np

data = load_iris()
X_scaled = StandardScaler().fit_transform(data.data)

fa = FactorAnalysis(n_components=2, random_state=42)
fa.fit(X_scaled)

# 因子負荷量行列
loadings_fa = pd.DataFrame(
    fa.components_.T,
    index=data.feature_names,
    columns=["Factor1", "Factor2"]
)
print("=== 因子負荷量 ===")
print(loadings_fa.round(3))

# 共通性（各変数の因子負荷量の二乗和）
communality = (fa.components_.T ** 2).sum(axis=1)
print("\n=== 共通性 ===")
for name, h2 in zip(data.feature_names, communality):
    print(f"{name}: {h2:.3f}")

# 独自分散
print("\n=== 独自分散 ===")
for name, psi in zip(data.feature_names, fa.noise_variance_):
    print(f"{name}: {psi:.3f}")
```

## 因子回転（バリマックス法）

### なぜ回転が必要か

初期解は数学的には正しいですが、「どの因子がどの変数を説明するか」が不明瞭になりがちです。例えば、ある変数が Factor1 に 0.6、Factor2 に 0.6 と両方に中程度の負荷を持つと解釈が難しくなります。

因子回転は **因子軸を回転させて、各変数が特定の因子に高く・他には低く負荷するよう** 単純構造（simple structure）を作ります。

### バリマックス回転の考え方

各因子の負荷量の二乗の分散を最大化することで、負荷量が「0 か大きいか」の二極化した構造を得ます。

$$
V = \sum_{j=1}^{m} \left[ \frac{1}{p} \sum_{i=1}^{p} \lambda_{ij}^4 - \left( \frac{1}{p} \sum_{i=1}^{p} \lambda_{ij}^2 \right)^2 \right] \to \text{maximize}
$$

回転は直交変換なので、**共通性は変わらず、因子の解釈性が向上** します。

```python
# バリマックス回転の手動実装（参考）
from scipy.stats import ortho_group
import numpy as np

def varimax(loadings, max_iter=1000, tol=1e-6):
    """バリマックス回転"""
    p, m = loadings.shape
    rotation = np.eye(m)
    
    for _ in range(max_iter):
        old_rotation = rotation.copy()
        for i in range(m):
            for j in range(i + 1, m):
                # Jacobi 回転で 2 因子ずつ最適化
                u = loadings[:, i] ** 2 - loadings[:, j] ** 2
                v = 2 * loadings[:, i] * loadings[:, j]
                A = np.sum(u)
                B = np.sum(v)
                C = np.sum(u ** 2 - v ** 2)
                D = 2 * np.sum(u * v)
                num = D - 2 * A * B / p
                den = C - (A ** 2 - B ** 2) / p
                theta = 0.25 * np.arctan2(num, den)
                c, s = np.cos(theta), np.sin(theta)
                R = np.eye(m)
                R[i, i] =  c; R[i, j] = -s
                R[j, i] =  s; R[j, j] =  c
                loadings = loadings @ R
                rotation = rotation @ R
        if np.max(np.abs(rotation - old_rotation)) < tol:
            break
    return loadings, rotation
```

## 因子数の決め方

### 1. スクリープロット（固有値プロット）

固有値が「急激に落ちた直後」のところで切ります。

```python
from sklearn.preprocessing import StandardScaler
from sklearn.datasets import load_iris
import matplotlib.pyplot as plt
import numpy as np

data = load_iris()
X_scaled = StandardScaler().fit_transform(data.data)
cov_matrix = np.cov(X_scaled.T)
eigenvalues = np.linalg.eigvalsh(cov_matrix)[::-1]

plt.figure(figsize=(6, 4))
plt.plot(range(1, len(eigenvalues) + 1), eigenvalues, 'o-')
plt.axhline(y=1, color='r', linestyle='--', label='固有値 = 1 の基準（Kaiser）')
plt.xlabel("因子番号")
plt.ylabel("固有値")
plt.title("スクリープロット")
plt.legend()
plt.show()

print("固有値:", eigenvalues.round(3))
```

### 2. Kaiser 基準

固有値 ≥ 1 の因子を採用します（標準化データでは 1 変数分の情報量を上回る因子のみ保持）。

### 3. 適合度（AIC・BIC・RMSEA）

統計的な適合度で選ぶ方法です。固有値基準より客観的ですが、計算が複雑です。

| 基準 | 考え方 | 注意 |
|------|--------|------|
| スクリープロット | 折れ曲がりの前 | 視覚的判断に依存 |
| Kaiser（固有値≥1） | 情報量基準 | 因子数を過大評価しがち |
| 平行分析 | ランダムデータと比較 | より信頼できる基準 |

## PCA vs 因子分析

| 観点 | PCA | 因子分析 |
|------|-----|---------|
| 目的 | 次元削減・圧縮 | 潜在構造の発見・解釈 |
| モデル | 統計モデルなし（行列演算） | 確率モデルあり（$\Sigma = \Lambda\Lambda^\top + \Psi$） |
| 誤差の扱い | 全分散を説明しようとする | 共通分散のみ説明する |
| 因子の数 | 寄与率で決める | 適合度・解釈性で決める |
| 回転 | なし（主成分は直交固定） | バリマックスなどで解釈性を向上 |
| 典型的な使用場面 | ML の前処理・可視化 | 心理尺度・アンケート分析 |
