# 特徴量エンジニアリング

**生データをモデルが学習しやすい形に変換する作業** です。「データ前処理」とも呼ばれ、精度向上に最も時間がかかる工程の一つです。良いモデルより良いデータの方が重要、とも言われます。

## はじめて読む人へ

特徴量エンジニアリングは、モデルに渡すデータを学習しやすい形へ整える作業です。良いモデルを選ぶ前に、欠損値、カテゴリ、スケール、日時などを適切に扱う必要があります。


### 読む前に押さえること

- モデルは生データではなく、数値化された特徴量から学習します。
- 欠損値や外れ値を放置すると、評価が不安定になります。
- 前処理はPipeline化して、学習時と推論時で同じ処理を使います。

### 読み終えたら説明できること

- 欠損補完、エンコーディング、スケーリングの目的を説明できる。
- 日時や対数変換などの特徴量生成を理解できる。
- ColumnTransformer を使う理由を説明できる。

## 欠損値の処理

欠損値は、データが記録されていない状態です。モデルによっては欠損をそのまま扱えないため、削除するか、何らかの値で補完する必要があります。補完方法は、データの意味と欠損の理由に合わせて選びます。

```python
import pandas as pd
import numpy as np
from sklearn.impute import SimpleImputer, KNNImputer

df = pd.read_csv("data.csv")

# 欠損の確認
print(df.isnull().sum())
print(df.isnull().mean() * 100)  # 欠損率（%）

# 単純補完
df["age"].fillna(df["age"].median(), inplace=True)   # 中央値
df["city"].fillna(df["city"].mode()[0], inplace=True)  # 最頻値

# sklearn による補完
imputer = SimpleImputer(strategy="mean")
df[["age", "income"]] = imputer.fit_transform(df[["age", "income"]])

# KNN 補完（近傍のデータを使って推測 — より精度が高い）
knn_imputer = KNNImputer(n_neighbors=5)
X_imputed = knn_imputer.fit_transform(df.select_dtypes("number"))
```

`isnull().sum()` は欠損の個数、`isnull().mean()` は欠損率を確認します。数値列は中央値や平均、カテゴリ列は最頻値で補完することが多いですが、欠損そのものに意味がある場合は「欠損フラグ」を特徴量として追加することもあります。

`KNNImputer` は、似たデータ点を使って欠損値を推定します。単純補完より自然な値になることがありますが、計算コストが高く、スケーリングや外れ値の影響も受けます。

## カテゴリ変数のエンコーディング

カテゴリ変数は、「東京」「大阪」「京都」のように数値ではない値を持つ列です。多くの機械学習モデルは文字列をそのまま扱えないため、数値に変換する必要があります。

One-Hot Encoding は、カテゴリごとに0/1の列を作る方法です。順序のないカテゴリに対して、モデルが勝手に大小関係を学習してしまうのを防げます。

### One-Hot エンコーディング

One-Hot Encoding は、カテゴリの種類ごとに列を作り、該当するカテゴリだけを 1 にする方法です。カテゴリに大小関係がない場合に使います。

```python
from sklearn.preprocessing import OneHotEncoder
import pandas as pd

df = pd.DataFrame({"color": ["red", "blue", "red", "green"]})

# pandas の方法（小規模向け）
dummies = pd.get_dummies(df["color"], prefix="color")
# → color_blue, color_green, color_red の 3 列

# sklearn の方法（パイプライン組み込み向け）
enc = OneHotEncoder(sparse_output=False, drop="first")  # 多重共線性を避けて 1 列削減
enc.fit_transform(df[["color"]])
```

`pd.get_dummies` は手軽ですが、学習時と推論時の列ずれを自分で管理する必要があります。`OneHotEncoder` は Pipeline に組み込めるため、本番運用ではこちらが安全です。

### ターゲットエンコーディング（高カーディナリティ向け）

カテゴリの種類が多い（都道府県・職業など）場合、One-Hot は列数が爆発します。

ターゲットエンコーディングは、カテゴリを目的変数の平均などで置き換える方法です。ただし、目的変数の情報を使うため、データリークに特に注意が必要です。

```python
from sklearn.model_selection import KFold

def target_encode(df, col, target, n_splits=5):
    """交差バリデーション付きターゲットエンコーディング（データリーク防止）"""
    encoded = pd.Series(index=df.index, dtype=float)
    kf = KFold(n_splits=n_splits, shuffle=True, random_state=42)
    for train_idx, val_idx in kf.split(df):
        mean_map = df.iloc[train_idx].groupby(col)[target].mean()
        encoded.iloc[val_idx] = df.iloc[val_idx][col].map(mean_map)
    return encoded.fillna(df[target].mean())
```

この実装では、交差検証の fold ごとに訓練側だけでカテゴリ平均を作り、検証側へ適用しています。自分自身の正解ラベルを使ってエンコードしないようにすることで、データリークを避けます。

## 数値特徴量のスケーリング

スケーリングは、数値特徴量の尺度をそろえる処理です。身長のように 100 台の値と、年収のように数百万の値が混ざると、距離や係数に基づくモデルでは大きい尺度の特徴量が過剰に効いてしまいます。

```python
from sklearn.preprocessing import StandardScaler, MinMaxScaler, RobustScaler

X = df[["age", "income", "score"]].to_numpy()

# 標準化：平均 0、標準偏差 1（外れ値に弱い）
std_scaler = StandardScaler()
X_std = std_scaler.fit_transform(X)

# 正規化：0〜1 の範囲（外れ値に弱い）
min_max = MinMaxScaler()
X_norm = min_max.fit_transform(X)

# ロバストスケーリング：中央値と IQR で変換（外れ値に強い）
robust = RobustScaler()
X_robust = robust.fit_transform(X)
```

`StandardScaler` は平均 0・標準偏差 1 に変換します。`MinMaxScaler` は 0〜1 に収めます。`RobustScaler` は中央値と四分位範囲を使うため、外れ値が多いデータに向いています。

> **どのスケーリングを選ぶか：** 外れ値が少ないなら StandardScaler。外れ値が多いデータ（収入・取引金額など）は RobustScaler が安全です。ツリー系モデル（決定木・ランダムフォレスト・XGBoost）はスケーリング不要です。

## 特徴量の生成

特徴量生成は、生データからモデルが学びやすい情報を作る作業です。元の列をそのまま使うだけでは表れにくい関係を、日時、比率、対数、交互作用などで表現します。

```python
# 多項式特徴量（非線形な関係を線形モデルで捉える）
from sklearn.preprocessing import PolynomialFeatures

poly = PolynomialFeatures(degree=2, include_bias=False)
X_poly = poly.fit_transform(X[:, :2])  # x1, x2 → x1, x2, x1², x1×x2, x2²

# 日時特徴量の展開
df["datetime"] = pd.to_datetime(df["datetime"])
df["hour"] = df["datetime"].dt.hour
df["day_of_week"] = df["datetime"].dt.dayofweek  # 月=0, 日=6
df["is_weekend"] = df["day_of_week"].isin([5, 6]).astype(int)
df["month"] = df["datetime"].dt.month

# 対数変換（右裾が長い分布を正規化）
df["log_income"] = np.log1p(df["income"])  # log(1 + x)：ゼロ対応
```

多項式特徴量は、線形モデルでも曲線的な関係を表せるようにします。日時特徴量は、時刻・曜日・月などの周期的な傾向をモデルに渡すために使います。対数変換は、収入や売上のように一部の大きな値が目立つ分布を扱いやすくします。

## 時系列特徴量

時系列データ（株価・アクセス数・センサー値など）では、過去の値そのものを特徴量にすることで、モデルが時間的なパターンを学習しやすくなります。

```python
import pandas as pd
import numpy as np

# センサーログ（1 分ごとの計測値）
df = pd.DataFrame({
    "ts": pd.date_range("2024-01-01", periods=1000, freq="min"),
    "value": np.random.randn(1000).cumsum() + 50
}).set_index("ts")

# ── ラグ特徴量（過去 n ステップの値） ─────────────────────
for lag in [1, 5, 10, 30]:
    df[f"lag_{lag}"] = df["value"].shift(lag)

# ── ローリング統計量（直近 n ステップの集計） ────────────────
for window in [5, 15, 60]:
    df[f"roll_mean_{window}"]   = df["value"].rolling(window).mean()
    df[f"roll_std_{window}"]    = df["value"].rolling(window).std()
    df[f"roll_max_{window}"]    = df["value"].rolling(window).max()

# ── 差分特徴量（変化量・トレンド除去） ───────────────────────
df["diff_1"]  = df["value"].diff(1)   # 1 ステップ差分
df["diff_5"]  = df["value"].diff(5)   # 5 ステップ差分

# ── 時刻特徴量 ─────────────────────────────────────────────
df = df.reset_index()
df["hour"]       = df["ts"].dt.hour
df["minute"]     = df["ts"].dt.minute
df["dayofweek"]  = df["ts"].dt.dayofweek
df["is_weekend"] = df["dayofweek"].isin([5, 6]).astype(int)

# ラグを使う場合、先頭 n 行は NaN になるので dropna 必須
df_clean = df.dropna()
print(f"元: {len(df)} 行  → 前処理後: {len(df_clean)} 行")
```

| 特徴量の種類 | 目的 |
|------------|------|
| ラグ特徴量 | 前の値との依存関係を学習 |
| ローリング平均 | 局所的なトレンドを捉える |
| ローリング標準偏差 | 変動の大きさを特徴化 |
| 差分 | トレンドを除去・変化量を特徴化 |
| 時刻・曜日 | 周期的なパターンを明示 |

> **時系列の交差検証は特別**：通常の k-fold は未来のデータを訓練に使ってしまう（データリーク）。`TimeSeriesSplit` を使って必ず「過去 → 未来」の順で分割します。

## 特徴量選択

不要な特徴量はモデルをノイズで汚染します。削減することで精度・解釈性・速度が向上します。

特徴量が多ければ多いほど良いとは限りません。関係の薄い特徴量やノイズの多い特徴量が増えると、モデルが偶然のパターンを覚えやすくなります。

```python
from sklearn.feature_selection import SelectKBest, f_classif, RFE
from sklearn.ensemble import RandomForestClassifier

X, y = df.drop("target", axis=1), df["target"]

# 統計的検定による選択
selector = SelectKBest(score_func=f_classif, k=10)
X_selected = selector.fit_transform(X, y)
selected_cols = X.columns[selector.get_support()]
print(f"選択された特徴量：{list(selected_cols)}")

# 重要度による選択（ツリーモデル）
rf = RandomForestClassifier(n_estimators=100, random_state=42)
rf.fit(X, y)
importances = pd.Series(rf.feature_importances_, index=X.columns)
print(importances.sort_values(ascending=False).head(10))
```

`SelectKBest` は統計的なスコアに基づいて上位 k 個の特徴量を選びます。ツリーモデルの重要度は、モデルがどの特徴量を分岐に使ったかをもとにします。どちらも絶対的な正解ではなく、特徴量を絞るための手がかりです。

## sklearn Pipeline で一括管理

前処理ステップをパイプラインにまとめると、データリークを防ぎ、コードも整理できます。

Pipeline 化の目的は、学習時と推論時で同じ前処理を必ず通すことです。Notebook 上で手作業で前処理すると、あとから同じ手順を再現しにくくなります。

```python
from sklearn.pipeline import Pipeline
from sklearn.compose import ColumnTransformer
from sklearn.ensemble import RandomForestClassifier

# 列の種類ごとに前処理を定義
numeric_features = ["age", "income"]
categorical_features = ["city", "color"]

preprocessor = ColumnTransformer([
    ("num", StandardScaler(), numeric_features),
    ("cat", OneHotEncoder(drop="first"), categorical_features),
])

# 前処理 + モデルを一本化
pipeline = Pipeline([
    ("preprocessor", preprocessor),
    ("model", RandomForestClassifier(n_estimators=100, random_state=42)),
])

pipeline.fit(X_train, y_train)
y_pred = pipeline.predict(X_test)

# パイプラインごと保存（本番環境に持ち込む）
import joblib
joblib.dump(pipeline, "pipeline.pkl")
```

`ColumnTransformer` は、数値列とカテゴリ列に別々の前処理を割り当てます。`Pipeline` は、その前処理の後にモデルを実行する順序をまとめます。保存するときは、モデルだけでなくパイプライン全体を保存するのが安全です。
