# データ可視化

**数値を図にする** ことでデータのパターン・異常・関係性を直感的に把握できます。matplotlib は静的グラフ、seaborn は統計的なグラフ、plotly はインタラクティブなグラフに向いています。

データ分析では、平均値や相関係数だけを見ても全体像を誤解することがあります。外れ値が 1 つだけ混ざっている、2 つのグループが重なっている、時間とともに傾向が変わっている、といった情報は図にすると一気に見つけやすくなります。

この章では「きれいな図を作る」より先に、**目的に合った図を選び、データの性質を読み取る** ことを重視します。レポートや発表で使う図だけでなく、分析の途中で仮説を立てるための図も扱います。

## この章で学ぶこと

- 折れ線・棒・散布図・ヒストグラムを目的に応じて使い分ける
- matplotlib の `fig` / `ax` の考え方を理解する
- seaborn で分布・外れ値・相関を確認する
- plotly でブラウザ上のインタラクティブな図を作る
- Streamlit で分析結果を Web アプリとして見せる

---

## はじめて読む人へ

データ可視化は、数字の表を人間が読み取りやすい形に変える作業です。図は飾りではなく、データの分布・関係・外れ値・時間変化を見つけるための分析道具です。

コードやコマンドが出てきたら、最初から全部を覚えようとしなくて大丈夫です。まずは「何を入力し、何が処理され、何が出力されるのか」を文章で説明できるように読むと、手を動かす前の理解が安定します。

### 読む前に押さえること

- グラフは目的に合わせて選びます。
- 軸・単位・凡例がない図は、読み手に誤解を与えやすくなります。
- 平均値だけでなく、分布や外れ値も確認します。

### 読み終えたら説明できること

- 目的に合ったグラフを選べる。
- matplotlib、seaborn、plotly の使い分けを説明できる。
- 図から読み取れることを文章で説明できる。

## グラフ選びの基本

最初に考えるべきことは「どのライブラリを使うか」ではなく、**何を知りたいのか** です。目的が決まると、使うグラフはかなり絞れます。

| 知りたいこと | よく使うグラフ | 例 |
|--------------|----------------|----|
| 時間とともにどう変わるか | 折れ線グラフ | 月別売上、気温の推移、学習 loss |
| カテゴリごとの大きさを比べる | 棒グラフ | 学科別人数、曜日別売上 |
| 2 つの数値の関係を見る | 散布図 | 勉強時間と点数、広告費と売上 |
| 1 つの数値の分布を見る | ヒストグラム | テスト点数、年齢、処理時間 |
| グループごとの分布を比べる | 箱ひげ図・バイオリンプロット | クラス別点数、性別ごとの購入金額 |
| 複数の数値列の関係をまとめて見る | ヒートマップ・pairplot | 特徴量同士の相関 |

良い可視化は、見た瞬間に「何を比較しているか」が分かります。軸ラベル、単位、タイトル、凡例が不足している図は、どれだけ見た目が整っていても読み手に負担をかけます。

## matplotlib 基礎

matplotlib では、図全体を `Figure`、実際にグラフを描く領域を `Axes` と呼びます。最初は `plt.plot()` だけでも描けますが、複数の図を並べたり細かく調整したりするには `fig, ax = plt.subplots()` の形を理解すると便利です。

`fig` はキャンバス全体、`ax` はその中の1つのグラフ領域です。タイトル、軸ラベル、凡例などは `ax` に対して設定すると、どのグラフに何を設定しているかが分かりやすくなります。

matplotlib は Python の標準的な可視化ライブラリです。細かい調整がしやすく、論文・レポート・保存用画像に向いています。

matplotlib では、図全体を `fig`、その中の 1 つひとつの描画領域を `ax` と考えます。`plt.subplots(2, 2)` は「2 行 2 列の図を入れる枠を作る」という意味です。

### 折れ線・棒・散布図・ヒストグラム

次のコードは、同じデータ分析でよく使う 4 種類の図を 1 枚に並べる例です。ここで見てほしいのは、グラフの種類によって「何を比較しているか」が違うという点です。折れ線は変化、棒はカテゴリ比較、散布図は 2 変数の関係、ヒストグラムは分布を表します。

```python
import matplotlib.pyplot as plt
import numpy as np

# 日本語フォント設定（環境に合わせて変更）
plt.rcParams["font.family"] = "Hiragino Sans"   # Mac
# plt.rcParams["font.family"] = "MS Gothic"     # Windows

x = np.linspace(0, 2 * np.pi, 100)

# 折れ線グラフ
fig, axes = plt.subplots(2, 2, figsize=(12, 8))

axes[0, 0].plot(x, np.sin(x), label="sin(x)")
axes[0, 0].plot(x, np.cos(x), label="cos(x)")
axes[0, 0].legend()
axes[0, 0].set_title("折れ線グラフ")

# 棒グラフ
categories = ["A", "B", "C", "D"]
values = [3, 7, 2, 5]
axes[0, 1].bar(categories, values, color="steelblue")
axes[0, 1].set_title("棒グラフ")

# 散布図
rng = np.random.default_rng(42)
x_scatter = rng.normal(0, 1, 100)
y_scatter = 2 * x_scatter + rng.normal(0, 0.5, 100)
axes[1, 0].scatter(x_scatter, y_scatter, alpha=0.6)
axes[1, 0].set_title("散布図")

# ヒストグラム
data = rng.normal(70, 15, 1000)
axes[1, 1].hist(data, bins=30, edgecolor="black")
axes[1, 1].set_title("ヒストグラム")

plt.tight_layout()
plt.savefig("plots.png", dpi=150)  # ファイルに保存
plt.show()
```

このコードでは、同じ画面に 4 種類のグラフを並べています。`axes[0, 0]` のように場所を指定して描くことで、複数の観点を 1 枚の図で比較できます。

- `plot()` は連続的な変化を見るために使います
- `bar()` はカテゴリ間の大小比較に使います
- `scatter()` は 2 変数の関係や外れ値を見るために使います
- `hist()` は値がどの範囲に集中しているかを見るために使います

分析の初期段階では、1 つのグラフだけで結論を出さないことが大切です。たとえば散布図で関係がありそうに見えても、ヒストグラムを見ると外れ値が原因だった、ということがあります。

また、グラフを描いたら「タイトルがあるか」「軸の意味が分かるか」「単位が分かるか」を確認します。可視化は自分が理解するためだけでなく、他の人にデータの見方を伝えるための道具でもあります。

### ファイル保存とフォーマット

分析中は画面に表示するだけでも十分ですが、レポート・発表・Web ページで使うには画像ファイルとして保存します。保存形式によって、見た目のきれいさや編集のしやすさが変わります。

```python
# 高解像度で PNG 保存
plt.savefig("output.png", dpi=150, bbox_inches="tight")

# PDF 保存（論文・レポート向け）
plt.savefig("output.pdf", bbox_inches="tight")

# SVG 保存（Web 埋め込み向け）
plt.savefig("output.svg", bbox_inches="tight")
```

保存形式は提出先に合わせて選びます。画像として貼るだけなら PNG、印刷や論文向けなら PDF、Web ページで拡大しても荒れない図にしたいなら SVG が便利です。`bbox_inches="tight"` を付けると、軸ラベルやタイトルが画像の外にはみ出しにくくなります。

`plt.savefig()` は、現在の図をファイルとして保存します。`plt.show()` の後に環境によって図が閉じられることがあるため、確実に保存したい場合は表示より前に保存する、と覚えておくと安全です。

## seaborn — 統計的なグラフ

pandas の DataFrame を直接渡せるため、データサイエンスの分析に便利です。

seaborn は matplotlib の上に作られたライブラリで、統計的な可視化を少ないコードで書けます。DataFrame の列名をそのまま `x=` や `y=` に指定できるため、pandas で前処理したデータをそのまま分析できます。

次の例で使う `tips` は、レストランの会計金額やチップ額を含むサンプルデータです。seaborn では、`data=df` で表全体を渡し、`x="day"` や `y="total_bill"` のように列名を指定して図を作ります。

```python
import seaborn as sns
import pandas as pd

df = sns.load_dataset("tips")   # サンプルデータ（チップ額）

# 箱ひげ図（分布と外れ値の可視化）
fig, axes = plt.subplots(1, 3, figsize=(15, 5))

sns.boxplot(data=df, x="day", y="total_bill", ax=axes[0])
axes[0].set_title("曜日別の合計金額")

# バイオリンプロット（分布形状まで見える）
sns.violinplot(data=df, x="day", y="total_bill", ax=axes[1])
axes[1].set_title("バイオリンプロット")

# 相関ヒートマップ
corr = df.select_dtypes("number").corr()
sns.heatmap(corr, annot=True, fmt=".2f", cmap="coolwarm", ax=axes[2])
axes[2].set_title("相関ヒートマップ")

plt.tight_layout()
plt.savefig("seaborn_plots.png", dpi=150)
```

箱ひげ図は、中央値・四分位範囲・外れ値を一度に見られる図です。平均値だけでは見えない「ばらつき」や「極端な値」を確認できます。

バイオリンプロットは箱ひげ図に分布の形を加えたものです。山が太い部分ほどデータが多く集まっています。グループごとの分布の形が違うかを見るときに便利です。

相関ヒートマップは、数値列同士の相関係数を色で表示します。ただし、相関が強いことは因果関係を意味しません。相関が見つかったら、散布図で形を確認し、必要なら背景にある要因を考えます。

ここで重要なのは、seaborn のコードが「統計量を図に変換している」ということです。箱ひげ図は四分位数、ヒートマップは相関係数のように、背後には必ず数値的な要約があります。図を読むときは、見た目だけでなく、その図がどの統計量を表しているかを意識します。

### pairplot — 特徴量の関係を一覧

`pairplot` は、数値列のすべての組み合わせを散布図として並べる機能です。列が多いデータでは図が大きくなりますが、特徴量同士の関係を最初に眺めるには便利です。

```python
# 数値列ペアの散布図を一括表示（特徴量エンジニアリング前の探索に便利）
sns.pairplot(df, hue="sex", corner=True)
plt.savefig("pairplot.png", dpi=150)
```

`pairplot` は数値列の組み合わせを一気に確認するための図です。機械学習の前に、目的変数と関係が強そうな特徴量、外れ値が多い特徴量、ほとんど情報を持たない特徴量を探すときに役立ちます。

`hue` を指定すると、カテゴリごとに色分けできます。色分けによって、同じ数値の関係でもグループごとに傾向が違うかを確認できます。

## Plotly — インタラクティブなグラフ

ブラウザで動かせる・拡大縮小できる・Streamlit との相性が抜群です。

Plotly はマウスで拡大・縮小・ホバー確認ができるため、探索的な分析や Web アプリでの可視化に向いています。発表資料やダッシュボードでは、聞き手が細部を確認しやすいという利点があります。

静的な図では、点が重なっていると個々のデータを確認しにくいことがあります。Plotly では点にマウスを乗せると詳細情報を表示できるため、データ点を「一覧する」だけでなく「調べる」ための図を作れます。

```python
import plotly.express as px
import plotly.graph_objects as go

df = px.data.tips()

# インタラクティブな散布図
fig = px.scatter(df, x="total_bill", y="tip", color="sex",
                 size="size", hover_data=["day"],
                 title="合計金額とチップの関係")
fig.show()   # ブラウザで開く

# HTML ファイルとして保存（Webサイトに埋め込める）
fig.write_html("scatter.html")

# FastAPI から JSON として返すことも可能
fig_json = fig.to_json()
```

`plotly.express` は短いコードでよく使うグラフを作るための API です。`hover_data` に列名を指定すると、点にマウスを乗せたときに追加情報を表示できます。データ点が多い場合は、静的な散布図よりも Plotly のほうが読み取りやすいことがあります。

`fig.show()` はブラウザや Notebook 上で図を表示し、`fig.write_html()` は動くグラフを HTML ファイルとして保存します。レポートに静止画を貼るのではなく、相手が自分で拡大・確認できる図を渡したいときに便利です。

### Plotly で地図・時系列

時系列データは、時間の順序が意味を持つデータです。年、月、日、時刻などを横軸に置き、値がどう変化したかを線で追います。

```python
# 時系列グラフ
df_time = px.data.gapminder().query("country == 'Japan'")
fig = px.line(df_time, x="year", y="gdpPercap", title="日本の GDP 推移")
fig.show()
```

時系列グラフでは、横軸を時間、縦軸を観測値にします。日付の間隔が不規則なデータでは、欠損している期間がないかを先に確認してください。欠損をそのまま線でつなぐと、実際には観測していない変化を見せてしまうことがあります。

時系列では、単に右上がりか右下がりかを見るだけでなく、季節性、急な変化、長期トレンドを分けて考えます。折れ線は点と点をつなぐため、観測していない期間にも連続的な変化があったように見える点に注意してください。

## Streamlit との連携

Streamlit では matplotlib・seaborn・plotly のグラフをそのまま表示できます。

Streamlit は、Python だけで簡単な分析アプリを作れるライブラリです。Notebook の中だけで完結していた図を、ブラウザで他の人にも触ってもらえる形にできます。

次のコードは、CSV を読み込み、matplotlib の静的グラフと Plotly のインタラクティブグラフを同じ Streamlit アプリに表示する例です。可視化を「自分の画面で見る図」から「他人が操作できる画面」に変える発想を押さえてください。

```python
import streamlit as st
import pandas as pd
import matplotlib.pyplot as plt
import plotly.express as px

df = pd.read_csv("data.csv")

# matplotlib グラフを表示
fig, ax = plt.subplots()
ax.hist(df["score"], bins=20)
st.pyplot(fig)

# plotly グラフを表示（インタラクティブ）
fig2 = px.scatter(df, x="age", y="score", color="group")
st.plotly_chart(fig2)
```

matplotlib の図は `st.pyplot(fig)`、Plotly の図は `st.plotly_chart(fig)` で表示します。ユーザーが条件を変えながら見る画面では、拡大やホバーができる Plotly が特に使いやすいです。

Streamlit では、このコードを `app.py` のようなファイルに保存して実行すると、ブラウザ上にアプリが立ち上がります。授業やチーム開発では、Notebook の結果をそのまま見せるより、入力やフィルタを用意したアプリにすると、相手が自分で仮説を確認しやすくなります。

## どのライブラリを使うべきか

| 用途 | 推奨 |
|------|------|
| 論文・レポート・静的画像 | matplotlib |
| 統計的な探索分析（EDA） | seaborn |
| Web アプリ・発表資料（動く） | plotly |
| Jupyter Notebook での分析 | seaborn + plotly |
| Streamlit アプリ | plotly（`st.plotly_chart`） |

迷ったら、まず seaborn でデータの傾向を確認し、レポート用に matplotlib で整え、アプリ化するときに Plotly を使う、という流れで十分です。最初からすべてをきれいに作ろうとすると、分析そのものが遅くなります。

## よくある失敗

### 軸の範囲で印象が変わる

縦軸の範囲を極端に狭くすると、小さな差が大きな差に見えます。差を強調したいときほど、軸の範囲を明示し、読み手が誤解しないようにします。

### 平均だけを見て分布を見ない

平均が同じでも、分布の形がまったく違うことがあります。ヒストグラムや箱ひげ図を使って、ばらつき・外れ値・偏りを確認してください。

### 色を使いすぎる

色が多すぎると、どこを見ればよいか分かりにくくなります。色は「グループの違い」や「強調したい値」に限定して使います。

### 先に結論を決めて図を選ぶ

可視化は証拠を探す作業ではなく、データを理解する作業です。都合のよいグラフだけを選ぶと、分析の信頼性が下がります。
