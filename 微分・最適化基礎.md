# 微分・最適化基礎

機械学習モデルを「学習させる」とは、損失関数を最小化するパラメータを見つけることです。そのために使うのが微分（勾配）と最適化アルゴリズムです。バックプロパゲーションを理解するための数学的土台を整理します。

---

## はじめて読む人へ

微分は「どの方向に動けば損失が減るか」を教えてくれます。最適化は「その方向に少しずつ動く」繰り返しです。数式を完璧に追うよりも、「勾配が正なら右から下がる方向、負なら左から下がる方向」という直感から入ると理解が安定します。

### 読む前に押さえること

- 微分は「変化率」です。f'(x) は x が少し増えたとき f(x) がどれだけ変わるかを表します。
- 多変数関数では、各変数について偏微分を求め、それをまとめたものが勾配（gradient）です。
- 最適化は、勾配の逆方向に少しずつパラメータを動かして最小値を探すプロセスです。

### 読み終えたら説明できること

- 微分・偏微分・勾配の関係を説明できる。
- 勾配降下法がなぜ損失を減らせるか説明できる。
- 連鎖律（Chain Rule）がバックプロパゲーションにどう使われるか説明できる。

---

## 微分の基礎

関数 f(x) の微分 f'(x) は、x における「傾き（瞬間変化率）」です。

$$
f'(x) = \lim_{h \to 0} \frac{f(x + h) - f(x)}{h}
$$

```python
import numpy as np
import matplotlib.pyplot as plt

def f(x):
    return x**2  # f(x) = x²

# 数値微分（有限差分近似）
def numerical_derivative(f, x, h=1e-5):
    return (f(x + h) - f(x - h)) / (2 * h)

x = np.linspace(-3, 3, 100)
y = f(x)
dy = numerical_derivative(f, x)  # f'(x) = 2x を近似

# x=1 での接線
# f'(1) = 2 → x=1 で傾き 2
```

### よく使う微分の公式

| 関数 | 微分 |
|------|------|
| $f(x) = c$（定数） | $f'(x) = 0$ |
| $f(x) = x^n$ | $f'(x) = nx^{n-1}$ |
| $f(x) = e^x$ | $f'(x) = e^x$ |
| $f(x) = \ln(x)$ | $f'(x) = 1/x$ |
| $f(x) = \text{sigmoid}(x)$ | $f'(x) = \text{sigmoid}(x)(1 - \text{sigmoid}(x))$ |
| $f(x) = \text{ReLU}(x)$ | $f'(x) = 1\;(x > 0),\; 0\;(x \leq 0)$ |

```python
def sigmoid(x):
    return 1 / (1 + np.exp(-x))

def sigmoid_derivative(x):
    s = sigmoid(x)
    return s * (1 - s)

def relu(x):
    return np.maximum(0, x)

def relu_derivative(x):
    return (x > 0).astype(float)
```

## 偏微分と勾配

パラメータが複数あるとき（$w_1, w_2, w_3, \ldots$）、各パラメータについての「その変数だけで微分したもの」が **偏微分**、それをまとめたベクトルが **勾配（gradient）** です。

$$
\nabla f = \left[\frac{\partial f}{\partial w_1},\; \frac{\partial f}{\partial w_2},\; \ldots,\; \frac{\partial f}{\partial w_n}\right]
$$

勾配は「最も急激に損失が増える方向」を向きます。最小化したいので、**勾配の逆方向** に動けば損失が減ります。

```python
def loss(w1, w2):
    # 2 変数の損失関数の例
    return (w1 - 2)**2 + (w2 + 1)**2

# 数値偏微分
def partial_derivative(f, w1, w2, which='w1', h=1e-5):
    if which == 'w1':
        return (f(w1 + h, w2) - f(w1 - h, w2)) / (2 * h)
    else:
        return (f(w1, w2 + h) - f(w1, w2 - h)) / (2 * h)

w1, w2 = 0.0, 0.0
grad_w1 = partial_derivative(loss, w1, w2, 'w1')  # ≈ -4（w1=2 に向かう）
grad_w2 = partial_derivative(loss, w1, w2, 'w2')  # ≈  2（w2=-1 に向かう）
```

## 勾配降下法

損失を最小化するための基本アルゴリズムです。

勾配 $\nabla L(w)$ は「現在の $w$ の位置で損失が最も急激に増える方向」を指します。ということは、損失を減らしたいなら**その逆方向**に動けば良いわけです。逆方向に動くには「マイナスをかける（$-\nabla L$）」だけで済みます。

どれだけ移動するかは、一歩の大きさを決める係数 $\eta$（学習率）で調整します。これをまとめると更新則になります。

$$
w \leftarrow w - \eta \nabla L(w)
$$

$\eta$（イータ）は **学習率（learning rate）**：一度に動く歩幅です。大きすぎると最小点を飛び越え、小さすぎると収束に時間がかかります。

```python
import numpy as np

# 1 次元の損失関数: L(w) = (w - 3)² の最小化
def L(w):
    return (w - 3) ** 2

def dL(w):          # 解析的微分: L'(w) = 2(w - 3)
    return 2 * (w - 3)

w = 0.0             # 初期値
lr = 0.1            # 学習率
history = [w]

for step in range(30):
    grad = dL(w)
    w = w - lr * grad
    history.append(w)

print(f"最終 w: {w:.4f}")  # ≈ 3.0（最小点）
```

### 学習率の影響

| 学習率 | 挙動 |
|--------|------|
| 小さすぎる | 収束が遅い |
| 適切 | 安定して最小点へ |
| 大きすぎる | 発散・振動する |

```python
# 学習率が大きすぎると発散する例
w_bad = 0.0
lr_bad = 1.5

for step in range(10):
    w_bad = w_bad - lr_bad * dL(w_bad)
    print(f"step {step}: w = {w_bad:.2f}")
# 最小点 3.0 を飛び越えて振動する
```

## 確率的勾配降下法（SGD）と派生手法

全データで勾配計算すると計算量が大きいため、**ミニバッチ（小さなサブセット）** で近似します。

### 各手法の比較

| 手法 | 更新式 | 特徴 |
|------|--------|------|
| SGD | $w \leftarrow w - \eta g$ | 最も単純。学習率の設定が重要 |
| Momentum | $v \leftarrow \beta v + g;\; w \leftarrow w - \eta v$ | 前の更新方向に慣性を持たせる |
| AdaGrad | $w \leftarrow w - \eta g / \sqrt{\sum g^2 + \varepsilon}$ | パラメータごとに学習率を調整。頻出パラメータは小さく |
| RMSprop | $v \leftarrow \rho v + (1-\rho)g^2;\; w \leftarrow w - \eta g/\sqrt{v+\varepsilon}$ | AdaGrad の学習率減衰を修正 |
| Adam | Momentum + RMSprop | 最もよく使われる。多くの問題に安定して機能する |

### Adam の仕組み（手動実装）

Adam（Adaptive Moment Estimation）は、勾配の1次モーメント（方向）と2次モーメント（大きさ）を両方追跡して学習率を自動調整します。

```python
import numpy as np

def adam_update(params, grads, m, v, t,
                lr=1e-3, beta1=0.9, beta2=0.999, eps=1e-8):
    """
    params: パラメータ（dict）
    grads:  勾配（dict）
    m:      1次モーメント推定値（平均的な勾配方向）
    v:      2次モーメント推定値（勾配の分散）
    t:      ステップ数
    """
    t += 1
    for key in params:
        g = grads[key]

        # 1次・2次モーメントの指数移動平均
        m[key] = beta1 * m[key] + (1 - beta1) * g
        v[key] = beta2 * v[key] + (1 - beta2) * g**2

        # バイアス補正（初期ステップで 0 に引っ張られるのを修正）
        m_hat = m[key] / (1 - beta1**t)
        v_hat = v[key] / (1 - beta2**t)

        # パラメータ更新
        params[key] -= lr * m_hat / (np.sqrt(v_hat) + eps)

    return params, m, v, t

# β₁=0.9: 勾配方向の履歴を 90% 残す（慣性）
# β₂=0.999: 勾配の大きさの履歴を 99.9% 残す（スケール調整）
# ε=1e-8: ゼロ除算防止の微小値
```

### PyTorch での使い方

```python
import torch
import torch.nn as nn

model = nn.Linear(10, 1)

# よく使われる最適化アルゴリズム
optimizer_sgd      = torch.optim.SGD(model.parameters(), lr=0.01, momentum=0.9)
optimizer_adam     = torch.optim.Adam(model.parameters(), lr=1e-3, betas=(0.9, 0.999))
optimizer_adamw    = torch.optim.AdamW(model.parameters(), lr=1e-3, weight_decay=1e-4)
optimizer_rmsprop  = torch.optim.RMSprop(model.parameters(), lr=1e-3)

# AdamW（Adam + 重み減衰）は現在の深層学習のデファクトスタンダード
```

| 状況 | おすすめ |
|------|---------|
| まず試す | AdamW（weight_decay=1e-4） |
| 学習が安定しない | 学習率を 10 分の 1 にする |
| 大規模データ・高速化 | SGD + Momentum（学習率スケジューラ必須） |
| 正則化も合わせて行う | AdamW |

## 連鎖律（Chain Rule）とバックプロパゲーション

### なぜ連鎖律が必要か

ニューラルネットワークは「いくつもの変換を積み重ねた合成関数」です。入力が層を通るたびに変換されて最終的に損失が計算されます。

```mermaid
flowchart LR
    X["入力 x"] --> F1["層1: f₁"] --> F2["層2: f₂"] --> F3["層3: f₃"] --> L["損失 L"]
```

学習するには：「重み $w_1$ を少し動かしたら、損失 $L$ はどれだけ変わる？」を知る必要があります（$\partial L / \partial w_1$ を計算したい）

w₁ は層1 の中にあります。でも損失は層3 の後で計算されます。直接は分かりません。そこで **連鎖律** を使って間接的に求めます。

---

### STEP 1：連鎖律の直感 ─ 「影響の連鎖」

```
工場のラインで考えてみましょう：

  原料 x → 機械A → 中間品 g → 機械B → 製品 y

「原料が 1 kg 増えたとき、製品は何 kg 増える？」

  機械A の変換率（原料→中間品）: 0.8 kg/kg
  機械B の変換率（中間品→製品）: 0.5 kg/kg

  全体の変換率 = 0.8 × 0.5 = 0.4 kg/kg
```

これが連鎖律の本質です。**「影響が連鎖するとき、変化率は掛け算でつながる」**

数式で書くと：

$$
y = f(g(x)) \text{ のとき} \quad \frac{dy}{dx} = \frac{dy}{dg} \times \frac{dg}{dx}
$$

---

### STEP 2：手計算で確認する

`y = (2x + 1)²` の `x=3` での微分を、連鎖律で計算してみましょう。

```
内側の関数 g(x) = 2x + 1   → g'(x) = 2
外側の関数 f(g) = g²        → f'(g) = 2g

x=3 では：
  g(3) = 2*3 + 1 = 7
  dy/dx = f'(g(3)) × g'(3)
        = 2*7      × 2
        = 14       × 2
        = 28
```

直接計算で確認：`y = (2*3+1)² = 49`、`(2*3.001+1)² = 49.0280...` → 差 ÷ 0.001 ≈ 28 ✓

---

### STEP 3：ニューラルネットへの応用

1 ニューロンの順伝播と逆伝播を完全に手で追います。

```
設定：
  入力 x = 2.0
  重み w = 0.5
  バイアス b = 0.1
  正解 y_true = 1.0
```

**■ 順伝播（前向き計算）**

```
u    = w * x + b
     = 0.5 * 2.0 + 0.1
     = 1.1

out  = sigmoid(u)
     = 1 / (1 + e^(-1.1))
     ≈ 0.750

損失 L = (out - y_true)²
       = (0.750 - 1.0)²
       = (-0.250)²
       = 0.0625
```

**■ 逆伝播（連鎖律を逆順に）**

求めたいのは「w を少し動かしたら損失 L はどれだけ変わるか（∂L/∂w）」です。経路を逆にたどります。

```
L → out → u → w  という経路を逆向きに

∂L/∂out = 2 * (out - y_true)
         = 2 * (0.750 - 1.0)
         = -0.500   ← out が増えると損失は 0.5 の速さで変化

∂out/∂u = sigmoid(u) * (1 - sigmoid(u))
         = 0.750 * (1 - 0.750)
         = 0.188    ← sigmoid の微分（u が増えると out がこの速さで変化）

∂u/∂w   = x = 2.0  ← w が増えると u は 2 倍の速さで増える

───────────────────────────────────
∂L/∂w = ∂L/∂out × ∂out/∂u × ∂u/∂w
       = (-0.500) × 0.188  × 2.0
       = -0.188

∂L/∂b = ∂L/∂out × ∂out/∂u × ∂u/∂b
       = (-0.500) × 0.188  × 1.0
       = -0.094
```

**∂L/∂w = -0.188** が意味すること：「w を少し増やすと損失が減る方向」→ w を増やせば良い。

**■ 重みの更新**

```
w_new = w - lr * ∂L/∂w
      = 0.5 - 0.1 * (-0.188)
      = 0.5 + 0.019
      = 0.519

b_new = b - lr * ∂L/∂b
      = 0.1 - 0.1 * (-0.094)
      = 0.109
```

→ 更新後 w=0.519, b=0.109 で同じ入力を流すと、out が 1.0 に少し近づきます。これを何百・何千回と繰り返すのが学習です。

### STEP 4：Python で実装・確認

```python
import numpy as np

def sigmoid(x):
    return 1 / (1 + np.exp(-x))

def sigmoid_derivative(x):
    s = sigmoid(x)
    return s * (1 - s)

# 設定
x = 2.0
w = 0.5
b = 0.1
y_true = 1.0
lr = 0.1

# ─── 順伝播 ───────────────────
u   = w * x + b          # 1.1
out = sigmoid(u)          # 0.750
L   = (out - y_true) ** 2 # 0.0625

print(f"順伝播: u={u:.3f}  out={out:.3f}  L={L:.4f}")

# ─── 逆伝播（連鎖律） ─────────
dL_dout = 2 * (out - y_true)    # -0.500
dout_du = sigmoid_derivative(u) # 0.188
du_dw   = x                     # 2.0
du_db   = 1.0                   # 1.0

dL_dw = dL_dout * dout_du * du_dw  # -0.188
dL_db = dL_dout * dout_du * du_db  # -0.094

print(f"勾配: ∂L/∂w={dL_dw:.3f}  ∂L/∂b={dL_db:.3f}")

# ─── 重みの更新 ───────────────
w_new = w - lr * dL_dw
b_new = b - lr * dL_db

print(f"更新後: w={w_new:.3f}  b={b_new:.3f}")

# ─── 更新後で再度順伝播して改善を確認 ─
out_new = sigmoid(w_new * x + b_new)
L_new   = (out_new - y_true) ** 2
print(f"更新後の損失: {L_new:.4f}  (元: {L:.4f}) → 損失が減った！")
```

PyTorch では `loss.backward()` がこの計算をすべてのパラメータについて自動で行います（自動微分 / autograd）。手動でやったことを完全に代替しています。

### 多層への拡張

層が増えても、連鎖律を繰り返し適用するだけです。

$$
\frac{\partial L}{\partial w_1} = \frac{\partial L}{\partial a_3} \times \frac{\partial a_3}{\partial a_2} \times \frac{\partial a_2}{\partial a_1} \times \frac{\partial a_1}{\partial w_1}
$$

層が深くなるほど掛け算が増えます。途中に 0 に近い値が入ると全体が 0 に近づき、勾配が消えてしまいます（**勾配消失問題**）。ReLU や残差接続がこの問題への対処です。

## ヤコビアン行列（発展）

多変数 → 多変数の写像 $f: \mathbb{R}^n \to \mathbb{R}^m$ の微分は、偏微分を並べた $m \times n$ 行列（ヤコビアン）で表します。バッチ処理や多出力ネットワークの勾配計算に登場します。

```python
import torch

x = torch.tensor([1.0, 2.0, 3.0], requires_grad=True)
y = torch.stack([x[0]**2 + x[1], x[1]*x[2]])  # R³ → R²

# ヤコビアン行列
J = torch.autograd.functional.jacobian(
    lambda v: torch.stack([v[0]**2 + v[1], v[1]*v[2]]),
    x
)
print(J)  # shape: (2, 3)
```

## 数学的導出

### 連鎖律の厳密な証明

$y = f(g(x))$ で $g$ が $x_0$ で微分可能、$f$ が $g(x_0)$ で微分可能とします。

微分の定義から：

$$
\frac{dy}{dx}\bigg|_{x=x_0} = \lim_{h \to 0} \frac{f(g(x_0 + h)) - f(g(x_0))}{h}
$$

$\Delta g = g(x_0 + h) - g(x_0)$ とおくと、$h \to 0$ のとき $\Delta g \to 0$ なので：

$$
= \lim_{h \to 0} \frac{f(g(x_0) + \Delta g) - f(g(x_0))}{\Delta g} \cdot \frac{\Delta g}{h}
$$

$\Delta g \neq 0$ のとき（ほとんどの場合）：

$$
= \lim_{\Delta g \to 0} \frac{f(g(x_0) + \Delta g) - f(g(x_0))}{\Delta g} \cdot \lim_{h \to 0} \frac{g(x_0+h) - g(x_0)}{h} = f'(g(x_0)) \cdot g'(x_0)
$$

$$
\boxed{\frac{dy}{dx} = f'(g(x)) \cdot g'(x)}
$$

---

### sigmoid 微分 $\sigma'(x) = \sigma(x)(1 - \sigma(x))$ の導出

$\sigma(x) = \frac{1}{1 + e^{-x}}$ を微分します。

商の微分規則 $\left(\frac{u}{v}\right)' = \frac{u'v - uv'}{v^2}$ を適用します（$u = 1$、$v = 1 + e^{-x}$）：

$$
\sigma'(x) = \frac{0 \cdot (1 + e^{-x}) - 1 \cdot (-e^{-x})}{(1 + e^{-x})^2} = \frac{e^{-x}}{(1 + e^{-x})^2}
$$

これを $\sigma(x)$ で表すために変形します：

$$
\sigma'(x) = \frac{1}{1 + e^{-x}} \cdot \frac{e^{-x}}{1 + e^{-x}} = \sigma(x) \cdot \frac{e^{-x}}{1 + e^{-x}}
$$

$\frac{e^{-x}}{1 + e^{-x}} = 1 - \frac{1}{1 + e^{-x}} = 1 - \sigma(x)$ を代入して：

$$
\boxed{\sigma'(x) = \sigma(x)(1 - \sigma(x))}
$$

この形が「sigmoid の値だけから微分が計算できる」ため、バックプロパゲーションで効率的に使えます。

---

### 勾配降下法の収束条件

**1 次元の凸関数の場合：**  
$L(w) = (w - w^*)^2$ のような二次関数では、更新則 $w \leftarrow w - \eta \cdot 2(w - w^*)$ から：

$$
w_{t+1} - w^* = (1 - 2\eta)(w_t - w^*)
$$

収束（誤差が減る）条件は $|1 - 2\eta| < 1$、つまり：

$$
\boxed{0 < \eta < 1}
$$

$\eta = 1$ のとき $w_{t+1} = w^*$ で 1 ステップで収束、$\eta > 1$ のとき発散します。一般のリプシッツ連続な勾配（勾配の変化が $L$-リプシッツ）を持つ凸関数では $\eta < 1/L$ が収束条件です。
