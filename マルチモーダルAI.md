# マルチモーダル AI

テキスト・画像・音声・動画など複数のモダリティ（データ形式）を統合して推論する AI 技術です。CLIP による画像-テキスト対応・LLaVA や GPT-4V による視覚言語モデル・Whisper による音声理解——単一モダリティを超えた統合的な知覚と推論を扱います。

---

## はじめて読む人へ

「この画像を説明して」「このグラフから何が読み取れますか？」——GPT-4V や Claude がこれに答えられるのは、視覚と言語を統合して処理するマルチモーダルモデルがあるからです。CLIP は「イヌの写真」と「dog」というテキストが意味的に近いことを学習しており、これが画像検索・ゼロショット分類の基盤になっています。

### 読む前に押さえること

- [自己教師あり学習](自己教師あり学習) — コントラスト学習の概念
- [Transformer・Attention](Transformer-Attention) — 言語モデルの構造
- [CNN（画像認識）](CNN) — 画像特徴抽出の基礎

### 読み終えたら説明できること

- CLIP がテキストと画像を同じ空間に埋め込む仕組みを説明できる
- VLM（視覚言語モデル）が画像とテキストを統合する 3 つのアーキテクチャを比較できる
- マルチモーダル学習の主な応用（画像検索・VQA・OCR）を説明できる

## モダリティとは

| モダリティ | データ形式 | 特徴 |
|----------|---------|------|
| テキスト | トークン列 | 記号的・構造的 |
| 画像 | ピクセル行列 | 連続的・空間的 |
| 音声 | 波形・スペクトログラム | 連続的・時系列 |
| 動画 | フレーム系列 | 空間 × 時間 |
| 点群 | 3D 座標集合 | 空間的 |

**マルチモーダルの課題：** 各モダリティは全く異なる表現形式を持ちます。テキストはトークンの離散列、画像はピクセルの連続行列——これらを「共通の意味空間」に変換することが核心技術です。

## CLIP（Contrastive Language-Image Pre-training）

### 基本アイデア

4 億組の（テキスト, 画像）ペアを使い、「テキストと対応する画像の埋め込みを近く、対応しない画像は遠く」なるよう学習します。

!!! info ""
    テキストエンコーダ → t₁ t₂ t₃ t₄  (行ベクトル: N 個のテキスト)
                         ┌─────────────┐
    イメージエンコーダ → │i₁ i₂ i₃ i₄ │ (列ベクトル: N 個の画像)
                         └─────────────┘
    コサイン類似度行列:
       ↑正例（対角成分）を高く
       ↓負例（非対角）を低く
    → N² - N 個の負例を使う（大バッチが重要な理由）

**損失関数：**

$$
\mathcal{L} = -\frac{1}{N}\sum_i \left[\log \frac{e^{s_{i,i}/\tau}}{\sum_j e^{s_{i,j}/\tau}} + \log \frac{e^{s_{i,i}/\tau}}{\sum_j e^{s_{j,i}/\tau}}\right]
$$

$s_{i,j}$：$i$ 番目のテキストと $j$ 番目の画像のコサイン類似度。対角（$i=j$）が正例。

### ゼロショット分類

CLIP の強力な応用の一つです。分類するラベルを「a photo of a {class}」のようなテンプレートに埋め込み、画像との類似度でクラスを決定します。

!!! info ""
    クエリ画像: 🐱 の写真
    
    テキスト候補:
      "a photo of a cat"    → 類似度: 0.94  ← 最大 = このクラスに分類
      "a photo of a dog"    → 類似度: 0.12
      "a photo of a car"    → 類似度: 0.05
    
    → Fine-tuning なしで分類できる（ゼロショット）

**ImageNet での精度：** CLIP の ViT-L モデルは教師あり学習なしで ImageNet Top-1 精度 76% を達成（ResNet-50 の監督あり 76% と同等）。

## 視覚言語モデル（VLM）

「画像を見て質問に答える（VQA）」「画像を説明する（Image Captioning）」「OCR」などのタスクを扱います。

### アーキテクチャ パターン 1：クロスアテンション融合

!!! info ""
    画像トークン → ビジョンエンコーダ → 視覚特徴 ──→ クロスアテンション
                                                           ↑
    テキストトークン → 言語モデル ──────────────────────────┘

Flamingo（DeepMind）が代表例。言語モデルの中間層に視覚特徴をクロスアテンションで注入します。

### アーキテクチャ パターン 2：プロジェクション層（LLaVA 型）

!!! info ""
    画像 → ビジョンエンコーダ（CLIP の ViT）→ 視覚トークン列
                                                  ↓ 線形プロジェクション
    テキストトークン ─────────────────────────→ 統合トークン列 → LLM

**LLaVA（Large Language and Vision Assistant）：** CLIP の ViT と Vicuna（LLaMA の派生）を線形変換で結合。シンプルな接続ながら強力な性能を示しました。学習は 2 段階：
1. 視覚-言語アライメント（プロジェクション層のみ学習）
2. End-to-end 指示チューニング（LLM + プロジェクション層）

### アーキテクチャ パターン 3：Q-Former（BLIP-2）

!!! info ""
    画像 → 凍結ビジョンエンコーダ → Q-Former（学習可能）→ 凍結 LLM

Q-Former は 32 個の学習可能なクエリトークンを持ち、視覚情報の要点を抽出して LLM に渡す「ボトルネック」として機能します。ビジョンエンコーダと LLM 両方を凍結し、Q-Former だけ学習するため非常に効率的です。

### GPT-4V / Claude 3 Sonnet

商用 VLM の最高峰。OCR・グラフ読み取り・複雑な視覚推論が可能です。アーキテクチャの詳細は非公開ですが、大規模なマルチモーダルデータでの事前学習 + RLHF/DPO によるアライメントが行われていると推測されます。

## マルチモーダルの応用

### 画像-テキスト検索

CLIP の埋め込みを使い、テキストで画像を検索（またはその逆）します。

!!! info ""
    クエリ: "赤いドレスを着た女性がパリで踊っている"
    → 意味的に近い画像を数百万枚の DB から高速検索
    → ストック写真サービス・Bing Image Search で活用

### VQA（Visual Question Answering）

画像に関する質問に自然言語で回答します。医療診断・衛星画像解析・製造品質管理に応用されています。

### マルチモーダル RAG

ベクトルデータベースに画像・テキスト・表の埋め込みを混在させ、クエリに応じて最適なモダリティから情報を取得します。

### OCR・文書理解

PDF・スキャン画像からテキストを抽出するだけでなく、表・グラフの意味を理解します。

## 音声-テキストモダリティ

### Whisper（OpenAI）

自己教師あり学習（音声 → テキストの弱教師）で学習した音声認識モデル。68 万時間の多言語音声で学習。

```python
import whisper
model = whisper.load_model("large")
result = model.transcribe("audio.mp3", language="ja")
print(result["text"])
```

### SpeechT5（Microsoft）

テキスト・音声・その他の音声タスク（TTS・ASR・声質変換）を統一したフレームワーク。

## 数学的導出

### CLIP 損失と相互情報量

CLIP の NT-Xent 損失は、テキストと画像の**相互情報量の下界を最大化**することに対応します（自己教師あり学習のページを参照）：

$$
I(\text{text}; \text{image}) \geq \log N - \mathcal{L}_{\text{CLIP}}
$$

$N$：バッチサイズ（≒負例の数）。大きいほど相互情報量の推定が正確になります。OpenAI は 32,768 のバッチサイズで学習しました。

### ゼロショット分類の確率解釈

クラス $c_k$ に対するテキストテンプレート $t_k$、入力画像 $v$ として：

$$
P(\text{class} = k \mid v) = \frac{\exp(\cos(\phi_v, \phi_{t_k})/\tau)}{\sum_j \exp(\cos(\phi_v, \phi_{t_j})/\tau)}
$$

これは Softmax 分類器で、特徴量が「埋め込み空間でのコサイン類似度」です。Fine-tuning なしで任意のクラスセットに適用できる理由は、テキストエンコーダが「クラス名の意味」を理解しているからです。

## 確認問題

1. CLIP の事前学習が「ゼロショット分類」を可能にする理由を、埋め込み空間の観点から説明してください。
2. LLaVA の 2 段階学習（アライメント → 指示チューニング）の各段階で何が学習されるかを説明してください。
3. Q-Former がビジョンエンコーダと LLM 両方を凍結できる理由を、効率的な学習の観点から説明してください。
4. マルチモーダル RAG がテキストのみの RAG より優れる場面を具体例で説明してください。
