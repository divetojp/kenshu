# サポートベクターマシン（SVM）

2 つのグループを分ける境界線を引くとき、「**できるだけ両方のグループから遠い位置に境界を引く**」のが SVM の考え方です。この「余裕（マージン）」を最大にすることで、見たことのないデータでも正確に分類できます。

---

## はじめて読む人へ

SVM は、決定木や Random Forest と並ぶ代表的な分類アルゴリズムです。理論的な背景（マージン最大化）が明確で、少ないデータでも安定して動きます。

「境界を引く」という直感から始め、徐々に数式に進みます。数式が難しく感じたら、**ASCII アートの図** だけ見て「なるほど、一番遠い位置に線を引くのか」という感覚を掴めば十分です。

### 読む前に押さえること

- [決定木](決定木.md) や [アンサンブル学習](アンサンブル学習.md) を先に読んでいること（比較のため）
- 2 次元の散布図（xy グラフ）に点を打つイメージを持てること
- SVC（分類）と SVR（回帰）がありますが、このページでは SVC（分類）を中心に扱います

### 読み終えたら説明できること

- 「マージン最大化」が何を意味するかを言葉で説明できる
- パラメータ C を大きくすると何が起きるか説明できる
- カーネルを使う理由を「直線では分けられないデータ」と結びつけて説明できる

## マージン最大化の原理

### なぜ「マージン最大化」なのか

2 クラスを分ける直線（決定境界）は無数に引けます。

!!! info ""
    ```text
    問い：どの直線が「最も良い」分類器か？

    データ点:
      クラス A: ● ● ●
      クラス B: ○ ○ ○

             【直線 1】        【直線 2】
       ●●●|○○○         ●●●  |○○○
                                  ← 余裕（マージン）が広い

    どちらも訓練データを正しく分類できるが…
      直線 1：境界が近すぎる → 未知のデータが少しずれただけで間違える
      直線 2：両クラスから最も遠い位置 → 未知データへの余裕が大きい
    ```
**SVM の考え方：「両クラスのデータから最も遠い位置に境界を引く」**

これが **マージン最大化** の直感です。境界からのマージン（余裕）が大きいほど、未知データへの汎化性能が高まります。

---

### 数学的な定式化

境界を「余裕を持って分ける」という要件を、数式で表現してみましょう。2 クラスをそれぞれ +1, -1 に割り当てると、**両クラスのデータが境界から少なくとも距離 1 以上離れている** 状態を次のように書けます。

$$
\mathbf{w}^\top \mathbf{x} + b \geq +1 \quad \text{（クラス A, +1）}
$$

$$
\mathbf{w}^\top \mathbf{x} + b \leq -1 \quad \text{（クラス B, −1）}
$$

この 2 つの境界面の間の距離（マージン幅）を計算すると $\frac{2}{\|\mathbf{w}\|}$ になります。

ここで「なぜ w が分母にあるのか？」と感じるかもしれません。直感的に考えてみましょう。w は境界面の向きと急さを決めるベクトルです。$\|\mathbf{w}\|$（w の大きさ）が大きいほど境界面が急峻になり、クラス A と B のデータが境界に近づきます——つまりマージンが狭まります。逆に $\|\mathbf{w}\|$ が小さいほど境界がなだらかになり、両クラスから遠くなります——マージンが広がります。ですから分母に $\|\mathbf{w}\|$ があるのは「傾きが急なほどマージンが狭い」という直感と一致しています。

$$
\text{マージン幅} = \frac{2}{\|\mathbf{w}\|}
$$

ここから「マージンを最大化したい」という目標を変換できます。$\frac{2}{\|\mathbf{w}\|}$ を最大化するには、分母の $\|\mathbf{w}\|$ を最小化すれば十分です。記号の形は変わりましたが、言いたいことは同じです：**境界の傾きをなるべく小さく（なだらかに）保つほど、余裕が生まれます**。

マージンを最大化したい $=$ $\frac{2}{\|\mathbf{w}\|}$ を最大化 $=$ $\|\mathbf{w}\|$ を最小化

!!! info ""
    ```text
         クラス A                クラス B
         ●  ●                    ○  ○
       ●   ●  ←マージン→  ○  ○
         ●                          ○
             ↑               ↑
        サポートベクター   サポートベクター
           （境界に最も近い点）
                ↕  決定境界  ↕
    ```
**サポートベクター**：決定境界の最も近くにあるデータ点です。このデータ点だけが境界の決定に影響します（他のデータ点をいくつ動かしても境界は変わりません）。

```python
import numpy as np
from sklearn.svm import SVC
from sklearn.datasets import make_classification
import matplotlib.pyplot as plt

# 線形に分離できる 2 クラスデータ
X, y = make_classification(n_samples=50, n_features=2, n_redundant=0,
                           n_clusters_per_class=1, random_state=42)

# SVM でフィット
model = SVC(kernel="linear", C=1.0)
model.fit(X, y)

print(f"サポートベクターの数: {model.support_vectors_.shape[0]}")
print(f"サポートベクターの座標:\n{model.support_vectors_.round(2)}")
```

## ハードマージンとソフトマージン

### ハードマージン SVM（線形分離可能な場合）

すべての点を正しく分類することを要求します。外れ値があると機能しません。

### ソフトマージン SVM（現実のデータ向け）

誤分類をある程度許容し、マージンと誤分類数のトレードオフをパラメータ **C** で制御します。

$$
\min \frac{1}{2}\|\mathbf{w}\|^2 + C \sum_i \xi_i
$$

| C の値 | 挙動 |
|--------|------|
| 大きい | 誤分類を許さない → マージンが狭い・過学習しやすい |
| 小さい | 誤分類を許容 → マージンが広い・汎化しやすい |

```python
from sklearn.svm import SVC
from sklearn.datasets import make_classification
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler
from sklearn.metrics import classification_report
import numpy as np
import matplotlib.pyplot as plt

X, y = make_classification(n_samples=200, n_features=2, n_redundant=0,
                            n_clusters_per_class=1, random_state=42)
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42)

scaler = StandardScaler()
X_train_s = scaler.fit_transform(X_train)
X_test_s  = scaler.transform(X_test)

# 線形 SVM
clf = SVC(kernel='linear', C=1.0)
clf.fit(X_train_s, y_train)

print(classification_report(y_test, clf.predict(X_test_s)))
print(f"サポートベクターの数: {clf.n_support_}")
```

## カーネルトリック

線形に分離できないデータを扱うために、特徴量を高次元空間に写像します。カーネル関数 $K(\mathbf{x}, \mathbf{z}) = \phi(\mathbf{x})^\top \phi(\mathbf{z})$ を使うことで、高次元空間での内積を元の空間での計算として行えます（φ を明示的に計算しない）。

```mermaid
flowchart LR
    A["2次元XOR\n（線形分離不可）"] --> B["カーネルで\n高次元に写像"]
    B --> C["3次元空間で\n線形分離可能"]
```

### 主なカーネル

| カーネル | 数式 | 使い場面 |
|---------|------|---------|
| 線形 | $\mathbf{x}^\top \mathbf{z}$ | データが線形分離可能・高次元テキスト |
| 多項式 | $(\gamma \mathbf{x}^\top \mathbf{z} + r)^d$ | 特徴量の組み合わせを考慮したい場合 |
| RBF（ガウス） | $\exp(-\gamma\|\mathbf{x}-\mathbf{z}\|^2)$ | 汎用。最もよく使われる |
| Sigmoid | $\tanh(\gamma \mathbf{x}^\top \mathbf{z} + r)$ | ニューラルネットとの類似性がある |

### RBF カーネルのパラメータ

```python
from sklearn.model_selection import GridSearchCV

# C と gamma のグリッドサーチ
param_grid = {
    'C':     [0.1, 1, 10, 100],
    'gamma': ['scale', 'auto', 0.01, 0.1, 1]
}

svc = SVC(kernel='rbf')
grid = GridSearchCV(svc, param_grid, cv=5, scoring='accuracy', n_jobs=-1)
grid.fit(X_train_s, y_train)

print(f"最適パラメータ: {grid.best_params_}")
print(f"交差検証精度:   {grid.best_score_:.3f}")
```

| γ の値 | 挙動 |
|--------|------|
| 小さい | 広いガウス → 滑らかな境界 → 汎化性高い |
| 大きい | 狭いガウス → 複雑な境界 → 過学習しやすい |

## 決定境界の可視化

```python
def plot_decision_boundary(clf, X, y, title="SVM 決定境界"):
    h = 0.02
    x_min, x_max = X[:, 0].min() - 0.5, X[:, 0].max() + 0.5
    y_min, y_max = X[:, 1].min() - 0.5, X[:, 1].max() + 0.5
    xx, yy = np.meshgrid(np.arange(x_min, x_max, h),
                          np.arange(y_min, y_max, h))
    Z = clf.predict(np.c_[xx.ravel(), yy.ravel()]).reshape(xx.shape)

    plt.contourf(xx, yy, Z, alpha=0.3, cmap='RdBu')
    plt.scatter(X[:, 0], X[:, 1], c=y, cmap='RdBu', edgecolors='k', s=40)

    # サポートベクターをハイライト
    sv = clf.support_vectors_
    plt.scatter(sv[:, 0], sv[:, 1], s=100, facecolors='none', edgecolors='k', linewidths=2)
    plt.title(title)
    plt.show()

# 線形 vs RBF の比較
fig, axes = plt.subplots(1, 2, figsize=(12, 5))
for ax, (kernel, C) in zip(axes, [('linear', 1), ('rbf', 10)]):
    clf = SVC(kernel=kernel, C=C, gamma='scale')
    clf.fit(X_train_s, y_train)
    plt.sca(ax)
    plot_decision_boundary(clf, X_train_s, y_train, title=f"kernel='{kernel}'")
plt.tight_layout()
```

## 多クラス分類

SVM はもともと 2 クラス分類器ですが、`SVC` は内部で自動的に多クラスに拡張します。

| 戦略 | 内容 |
|------|------|
| One-vs-Rest（OvR） | クラス数だけ 2 クラス分類器を作る |
| One-vs-One（OvO） | 全クラスペアに分類器を作る（`SVC` のデフォルト） |

```python
from sklearn.datasets import load_iris
from sklearn.svm import SVC
from sklearn.metrics import accuracy_score

data = load_iris()
X_train, X_test, y_train, y_test = train_test_split(
    StandardScaler().fit_transform(data.data), data.target,
    test_size=0.3, random_state=42
)

clf = SVC(kernel='rbf', C=10, gamma='scale', decision_function_shape='ovr')
clf.fit(X_train, y_train)
print(f"Iris 3クラス精度: {accuracy_score(y_test, clf.predict(X_test)):.3f}")
```

## SVR（サポートベクター回帰）

回帰問題にも SVM を適用できます。予測値の「ε-チューブ」の外に出た点だけペナルティを課します。

```python
from sklearn.svm import SVR
from sklearn.datasets import load_diabetes
from sklearn.metrics import r2_score

data = load_diabetes()
X_train, X_test, y_train, y_test = train_test_split(
    StandardScaler().fit_transform(data.data), data.target,
    test_size=0.2, random_state=42
)

svr = SVR(kernel='rbf', C=100, epsilon=10, gamma='scale')
svr.fit(X_train, y_train)
y_pred = svr.predict(X_test)

print(f"R²: {r2_score(y_test, y_pred):.3f}")
```

## SVM vs 他の分類手法

| 手法 | 強み | 弱み |
|------|------|------|
| SVM（RBF） | 高次元・小〜中規模で強い | 大規模データで遅い・確率出力がない |
| ロジスティック回帰 | 確率出力・解釈しやすい | 非線形境界には弱い |
| LDA | クラス間分散を最大化・解釈性高い | 正規分布仮定・共分散が同じ前提 |
| ランダムフォレスト | 大規模・外れ値に強い | 解釈性は低い |
| SVM（線形） | テキスト分類に強い | 特徴量が少ないと効果薄 |
