バイオインフォマティクス¶

生命科学のデータ（ゲノム・タンパク質・遺伝子発現）をコンピュータで解析する学際領域です。DNA 配列アライメント・RNA-seq 解析・タンパク質構造予測（AlphaFold）・単細胞 RNA シーケンシング（scRNA-seq）を扱います。データサイエンスの手法が生命科学の謎を解くフロンティア領域です。

はじめて読む人へ¶

「ChatGPT が言語を学習するように、タンパク質の『言語』を学習させたら構造予測ができた」——これが AlphaFold 2 の本質です。ゲノムデータは次世代シーケンサーで指数的に増加しており、機械学習・統計の知識を持つデータサイエンティストが生物学者と協力する場面が急増しています。

読む前に押さえること¶

  • 確率・統計基礎 — 統計的推定・仮説検定
  • 深層学習入門 — NN の基礎（AlphaFold の理解のため）
  • 自己教師あり学習 — 大規模事前学習の概念

読み終えたら説明できること¶

  • DNA・RNA・タンパク質の「セントラルドグマ」をデータの観点で説明できる
  • Smith-Waterman アルゴリズムの動的計画法を説明できる
  • AlphaFold 2 の MSA と Evoformer の役割を説明できる

セントラルドグマとデータ¶

生命情報の流れ¶

DNA （ゲノム） ↓ 転写 RNA （トランスクリプトーム） ↓ 翻訳 タンパク質 （プロテオーム） ↓ 機能・構造 | レベル | データ形式 | 代表的ツール | |--------|---------|------------| | ゲノム（DNA）| FASTQ・VCF | BWA・GATK | | トランスクリプトーム（RNA）| count matrix | DESeq2・edgeR | | タンパク質 | PDB（3D 座標）| AlphaFold・ESMFold | | 単細胞（scRNA-seq）| スパース行列 | Seurat・Scanpy |

配列と表現¶

DNA・RNA の表現¶

DNA 配列: ATCGATCG... (A, T, C, G の 4 文字) RNA 配列: AUCGAUCG... (A, U, C, G の 4 文字; T → U) タンパク質配列: MKVLLS... (20 種のアミノ酸の 1 文字コード) ヒトゲノムは約 30 億塩基対。1 人のゲノムシーケンシングのコストは 2001 年の 30 億ドルから 2024 年には 200 ドル以下になりました（ムーアの法則を超えるコスト低下）。

k-mer 表現¶

配列を長さ \(k\) の連続部分列に分割します。

配列 ATCGATCG を k=3 の k-mer に分割すると、ATC, TCG, CGA, GAT, ATC, TCG という連続部分列が得られます。k-mer の頻度ヒストグラムで配列を特徴量化できます。 \(k=4\) の場合 \(4^4 = 256\) 次元のベクトル表現になります。ゲノム分類・ウイルス検出に使われます。

配列アライメント¶

グローバルアライメント（Needleman-Wunsch）¶

2 つの配列全体を比較して最適な対応を見つけます。進化的関係の解析に使われます。

ローカルアライメント（Smith-Waterman）¶

配列の部分領域で最も類似した箇所を見つけます。データベース検索（BLAST）に応用されます。

動的計画法による最適アライメント：

\[ H(i, j) = \max\begin{cases} 0 \\ H(i-1, j-1) + s(a_i, b_j) \\ H(i-1, j) - d \\ H(i, j-1) - d \end{cases} \]
  • \(s(a_i, b_j)\)：置換スコア行列（BLOSUM62 など）
  • \(d\)：ギャップペナルティ
  • \(H(i,j)\)：配列 \(a\) の \(i\) 文字目と \(b\) の \(j\) 文字目までの最適スコア

BLAST（Basic Local Alignment Search Tool）¶

Smith-Waterman は \(O(mn)\) で遅いため、BLAST は heuristic で高速化しています。

  1. クエリ配列を \(k=11\) の k-mer に分割
  2. 類似した k-mer が DB に存在する箇所を探す（ハッシュ検索）
  3. その周辺をスコア拡張して全体アライメント

NCBI の GenBank（約 2,000 億塩基対）に対して秒単位で検索できます。

RNA-seq 解析（遺伝子発現解析）¶

実験の流れ¶

細胞・組織 → RNA 抽出 → 逆転写 → cDNA → シーケンサー → FASTQ（生リード）→ リファレンスゲノムへのマッピング → 遺伝子ごとの read count 行列 → 差次発現解析

read count 行列¶

サンプル 遺伝子1 遺伝子2 遺伝子3 ...
サンプル1 120 0 5 —
サンプル2 110 0 3 —
サンプル3 85 320 210 —
→ 「どの遺伝子がどの条件で多く発現しているか」を統計的に解析します。

差次発現解析（Differential Expression）¶

疾患 vs 正常、処理 vs 未処理などのグループ間で発現量が有意に異なる遺伝子を特定します。

\[ \text{log}_2\text{FC} = \log_2\frac{\text{条件 A の平均発現量}}{\text{条件 B の平均発現量}} \]

DESeq2（負の二項分布モデル）： RNA-seq データは過分散（分散 > 平均）なため、ポアソン分布ではなく負の二項分布を使います。多重比較補正（Benjamini-Hochberg）を適用して FDR を制御します。

タンパク質構造予測と AlphaFold¶

タンパク質折り畳み問題¶

アミノ酸配列（1 次構造）から立体構造（3 次構造）を予測します。これは 50 年来の生物学の最難問でした。

配列: M - K - V - L - L - S - A - E - ... (300 アミノ酸) ↓ ??? 3D 構造: 各原子の (x, y, z) 座標

AlphaFold 2 の核心アイデア¶

1. MSA（多重配列アライメント）：

進化的に類似した配列（ホモログ）を数千〜数万集めてアライメントします。「同じ位置が同時に変異している（共変化）= その 2 残基が空間的に近い」という情報を抽出します。

2. Evoformer：

MSA と残基間の距離・方向を「ペア特徴量行列」として Transformer 風に反復処理します。

MSA 表現（行：配列、列：残基位置） ↕ 行アテンション・列アテンション ペア表現（i行j列 = 残基 i と j の関係） ↕ triangle update 両者を交互に更新（Evoformer ブロック × 48 回） 3. Structure Module：

Evoformer の出力から各アミノ酸の座標（厳密には回転・平行移動）を予測します。

成果（CASP14, 2020）： 予測精度がほぼ実験精度（約 1 Å の誤差）に達し、「解決済みの問題」と宣言されました。AlphaFold DB には全既知タンパク質（2 億以上）の予測構造が公開されています。

単細胞 RNA-seq（scRNA-seq）¶

バルク RNA-seq との違い¶

手法 測定対象 分かること
バルク RNA-seq 組織全体の平均発現量 細胞の種類の混合が見えない
scRNA-seq 1 細胞ずつの発現量 細胞の多様性が分かる
1 回の実験で： 10,000〜100,000 個の細胞、各細胞で 20,000 遺伝子の発現量を測定 → 巨大なスパース行列

標準的な解析パイプライン（Scanpy）¶

import scanpy as sc

# データ読み込み
adata = sc.read_h5ad("data.h5ad")  # AnnData 形式

# 前処理
sc.pp.filter_cells(adata, min_genes=200)
sc.pp.filter_genes(adata, min_cells=3)
sc.pp.normalize_total(adata, target_sum=1e4)  # 正規化
sc.pp.log1p(adata)                             # 対数変換

# 次元削減
sc.tl.pca(adata, svd_solver='arpack')          # PCA
sc.pp.neighbors(adata, n_neighbors=10)
sc.tl.umap(adata)                              # UMAP 可視化

# クラスタリング（Leiden アルゴリズム）
sc.tl.leiden(adata, resolution=0.5)

# 可視化
sc.pl.umap(adata, color=['leiden', 'gene_name'])

UMAP（次元削減）で細胞を 2D に可視化すると、「細胞型」ごとにクラスターが形成されます。各クラスターのマーカー遺伝子から「このクラスターはT細胞」「これは線維芽細胞」などを同定します。

数学的導出¶

Smith-Waterman の最適性証明¶

動的計画法の最適部分構造：「最適なローカルアライメントの任意の部分アライメントも最適である」

\(H(i, j)\) の最大を与えた後、バックトラッキングで最適アライメントを再構成します。この過程は有向非巡回グラフ（DAG）の最長経路問題と等価で、\(O(mn)\) で解けることが示せます（分割統治は使えないため）。

RNA-seq の正規化：TPM の導出¶

生リード count（\(C_{ij}\)：サンプル \(j\) の遺伝子 \(i\) のリード数）から TPM（Transcripts Per Million）を計算します：

\[ \text{TPM}_{ij} = \frac{C_{ij}/L_i}{\sum_k C_{kj}/L_k} \times 10^6 \]

\(L_i\)：遺伝子 \(i\) の長さ。長い遺伝子ほど断片化されやすく count が多くなるバイアスを補正し、異なるサンプル間で比較可能にします。

確認問題¶

  1. Smith-Waterman と BLAST の違いを「精度と速度のトレードオフ」で説明してください。
  2. AlphaFold 2 で MSA（多重配列アライメント）を使う理由を「共変化情報」の観点から説明してください。
  3. scRNA-seq データが「スパース」である理由（多くの遺伝子で count = 0）を生物学的に説明してください。
  4. RNA-seq の TPM 正規化が必要な理由を 2 つ説明してください（遺伝子長・シーケンス深度）。
