# HTML / CSS

## Web ページはどうやって作られているか

ブラウザで Web ページを表示するとき、裏側では主に 3 種類のファイルが使われています。

| ファイル | 役割 | 例え |
|----------|------|------|
| **HTML** | ページの「構造」を定義する | 家の骨組み・間取り |
| **CSS** | ページの「見た目」を定義する | 壁の色・インテリア |
| **JavaScript** | ページの「動き」を定義する | 電気・自動ドア |

このページでは **HTML** と **CSS** を解説します。

---

## はじめて読む人へ

HTML はページの構造、CSS は見た目を担当します。家にたとえると、HTML は柱や部屋の配置、CSS は壁紙や家具の見え方です。

コードやコマンドが出てきたら、最初から全部を覚えようとしなくて大丈夫です。まずは「何を入力し、何が処理され、何が出力されるのか」を文章で説明できるように読むと、手を動かす前の理解が安定します。

### 読む前に押さえること

- HTML タグは、文章や画像の意味をブラウザに伝えます。
- CSS セレクターは、どの要素に見た目を適用するかを指定します。
- レイアウトは、余白・幅・並び方を少しずつ調整する作業です。

### 読み終えたら説明できること

- HTML と CSS の役割の違いを説明できる。
- 基本的なタグと CSS プロパティを読める。
- Flexbox で横並びレイアウトの考え方を理解できる。

## HTML とは

HTML は、ブラウザに「この文章は見出しです」「ここは段落です」「これはリンクです」と意味を伝えるための言語です。見た目を整えるためだけではなく、ページの構造を表す役割があります。

たとえば大きな文字にしたいから `h1` を使う、という考え方は危険です。`h1` はページの主題を表す見出しであり、見た目は CSS で調整します。HTML では意味、CSS では見た目を担当させると、保守しやすいページになります。

**HTML** （HyperText Markup Language）は、 **Web ページの構造を作るための言語** です。

「見出し」「段落」「画像」「リンク」などを **タグ** という記号で表現します。

### タグの書き方

HTML タグは、基本的に「開始タグ」「内容」「終了タグ」の 3 つで 1 セットです。次の例では、ブラウザに「この内容はこの種類の要素です」と伝えています。

```html
<タグ名>内容</タグ名>
```

開始タグは `<タグ名>`、終了タグは `</タグ名>` の形です。終了タグには `/` が付く、という違いをまず覚えてください。タグの中に書いた「内容」が、実際にページ上へ表示される文字になります。

例：

```html
<h1>近江テック・アカデミー</h1>
<p>滋賀県にあるプログラミングスクールです。</p>
```

ブラウザはこれを読み取って、`h1` は大きな見出し、`p` は段落として表示します。

ここで大切なのは、`h1` や `p` が「文字を大きくする命令」ではなく、「これは見出しです」「これは段落です」という意味を表すことです。見た目の大きさや色は、このあと学ぶ CSS で調整します。

## よく使う HTML タグ

### 見出し

見出しは `h1`〜`h6` まであります。数字が小さいほど大きな見出しです。

次のコードは、見出しの階層を HTML で表した例です。文章の章立てと同じように、`h1` を大見出し、`h2` を中見出し、`h3` を小見出しとして読むと分かりやすくなります。

```html
<h1>一番大きな見出し（ページタイトルなど）</h1>
<h2>二番目の見出し（セクションタイトルなど）</h2>
<h3>三番目の見出し</h3>
```

通常、`h1` はページ全体のタイトルとして 1 回だけ使い、その下に `h2`、さらに細かい話題に `h3` を使います。見出し番号を飛ばしすぎると、ページの構造が読みにくくなります。

### 段落・テキスト

本文のかたまりは `p`、強く伝えたい語句は `strong`、ニュアンスとして強調したい語句は `em` で表します。見た目が似ていても、HTML では「意味」を使い分けます。

```html
<p>段落のテキストです。</p>
<strong>太字にしたいテキスト</strong>
<em>斜体（強調）にしたいテキスト</em>
```

`strong` は多くのブラウザで太字、`em` は斜体として表示されます。ただし、見た目は CSS で変えられるため、「太字にしたいから strong」ではなく「重要だから strong」と考えるのが基本です。

### リンク

リンクは `a` タグで作ります。`href` 属性に移動先を書き、タグの中にクリックされる文字を書きます。

```html
<!-- 外部サイトへのリンク -->
<a href="https://example.com">リンクのテキスト</a>

<!-- サイト内の別ページへのリンク -->
<a href="/about">会社概要</a>

<!-- 別タブで開く場合 -->
<a href="https://example.com" target="_blank">外部リンク</a>
```

`href` は「このリンクがどこへ向かうか」を表す属性です。`target="_blank"` は別タブで開く指定ですが、外部サイトへ移動させるときだけ使うなど、ユーザーが予想しやすい動きにすることが大切です。

### 画像

画像は `img` タグで表示します。`img` は中身を囲むタグではないので、終了タグを書きません。

```html
<img src="/images/logo.png" alt="近江テック・アカデミーのロゴ" width="200">
```

- `src`：画像のファイルパスや URL
- `alt`：画像が表示できないときの代替テキスト（アクセシビリティのためにも必ず書く）

`alt` は、画像を見られない人や、画像の読み込みに失敗したときのための説明です。「画像」とだけ書くのではなく、画像が伝えている内容を短く書きます。

### リスト

複数の項目を並べるときは、ただ改行して書くよりもリストタグを使います。順番が重要でない場合は `ul`、手順のように順番が重要な場合は `ol` を使います。

```html
<!-- 箇条書き（順番なし） -->
<ul>
  <li>Git</li>
  <li>GitHub</li>
  <li>Astro</li>
</ul>

<!-- 番号付きリスト -->
<ol>
  <li>リポジトリをクローンする</li>
  <li>ブランチを作成する</li>
  <li>変更をコミットする</li>
</ol>
```

`li` はリストの 1 項目を表します。`ul` や `ol` の直下には基本的に `li` を置き、項目の中に必要ならリンクや文章を入れます。

### レイアウト用のタグ

ページの構造を表すためのタグです。見た目には直接影響しませんが、意味を明確にすることで、CSS でスタイルを当てやすくなります。

次の例は、ページ全体を「ヘッダー」「ナビゲーション」「メイン」「フッター」に分ける典型的な構造です。まずは、Web ページを大きな箱に分けて考える感覚をつかんでください。

```html
<header>ヘッダー（サイトのロゴ・ナビゲーションなど）</header>
<nav>ナビゲーションメニュー</nav>
<main>メインコンテンツ</main>
<section>ひとつのセクション</section>
<article>記事・お知らせなどの独立したコンテンツ</article>
<footer>フッター（コピーライトなど）</footer>

<!-- 意味を持たない汎用の箱 -->
<div>ブロック要素（縦に並ぶ）</div>
<span>インライン要素（横に並ぶ）</span>
```

`div` と `span` は便利ですが、意味を持たない汎用タグです。`header` や `main` など意味のあるタグで表せる場合は、そちらを優先すると読みやすい HTML になります。

## CSS とは

**CSS** （Cascading Style Sheets）は、 **HTML 要素に見た目を適用する言語** です。

HTML だけではすべての要素が上から下へ並んだシンプルな表示になります。CSS を加えることで、色・サイズ・配置などを細かく制御できます。

### CSS の基本的な書き方

CSS は「どの要素を」「どのように変えるか」を書きます。次のひな形では、`セレクター` が対象、`プロパティ` が変更したい項目、`値` が具体的な指定です。

```css
セレクター {
  プロパティ: 値;
}
```

- **セレクター** ：どの要素にスタイルを当てるかを指定する
- **プロパティ** ：何を変えるかを指定する（色・大きさ・余白など）
- **値** ：どう変えるかを指定する

例：

次の例では、ページ内のすべての `h1` 要素に同じスタイルが当たります。HTML 側に複数の `h1` があれば、そのすべてが対象になります。

```css
h1 {
  color: #1a1a2e;   /* 文字色を指定 */
  font-size: 2rem;  /* 文字サイズを指定 */
}
```

`color` や `font-size` の行は、必ず `プロパティ: 値;` の形で書きます。最後のセミコロン `;` を忘れると、次の指定が正しく読まれないことがあります。

## セレクターの種類

セレクターは、CSS が「どの HTML 要素を選ぶか」を決めるための指定です。タグ名・クラス・ID の 3 種類をまず使い分けられるようにしましょう。

```css
/* タグ名で指定（そのタグすべてに適用） */
p {
  line-height: 1.8;
}

/* クラス名で指定（class="card" のついた要素に適用） */
.card {
  background-color: white;
  border-radius: 8px;
}

/* ID で指定（id="header" のついた要素に適用） */
#header {
  position: fixed;
  top: 0;
}
```

タグ名セレクターは広く当たり、クラスセレクターは必要な部品に繰り返し使えます。ID セレクターはページ内で 1 つだけの要素に使うのが基本です。

HTML 側での使い方：

```html
<div class="card">カードの内容</div>
<header id="header">ヘッダー</header>
```

CSS の `.card` は HTML の `class="card"` に対応し、CSS の `#header` は HTML の `id="header"` に対応します。`.` や `#` は CSS 側だけに書き、HTML の属性値には付けません。

## よく使う CSS プロパティ

### テキスト

テキスト関連のプロパティは、文章の読みやすさに直接関わります。色やサイズだけでなく、行間や配置もまとめて調整します。

```css
color: #333333;          /* 文字色（16進数のカラーコード） */
font-size: 16px;         /* 文字サイズ */
font-weight: bold;       /* 太さ（bold で太字） */
line-height: 1.6;        /* 行間（数値が大きいほど広い） */
text-align: center;      /* 横位置（left / center / right） */
text-decoration: none;   /* 下線など（none で消す） */
```

`line-height` は初学者が見落としやすい指定ですが、本文の読みやすさに大きく影響します。長い文章では `1.5`〜`1.8` 程度にすると、行同士が詰まりすぎず読みやすくなります。

### ボックス（サイズ・余白・枠線）

HTML の多くの要素は、画面上では四角い箱として扱われます。箱の幅、高さ、余白、枠線、背景を制御するのが次のプロパティです。

```css
width: 100%;             /* 幅 */
max-width: 1200px;       /* 最大幅（これより広くならない） */
height: 200px;           /* 高さ */
padding: 16px;           /* 内側の余白（ボーダーの内側） */
margin: 0 auto;          /* 外側の余白（0 auto で左右中央寄せ） */
border: 1px solid #ccc; /* 枠線（太さ スタイル 色） */
border-radius: 8px;      /* 角丸 */
background-color: #fff; /* 背景色 */
```

`padding` は箱の内側を広げ、`margin` は箱の外側に距離を作ります。この違いを理解すると、レイアウトが崩れたときに「内側が狭いのか、外側の間隔が足りないのか」を切り分けやすくなります。

> **padding と margin の違い：**
> ```
> ┌─────────────────────────┐
> │  margin（外側の余白）    │
> │  ┌───────────────────┐  │
> │  │  border（枠線）    │  │
> │  │  ┌─────────────┐  │  │
> │  │  │  padding    │  │  │
> │  │  │  ┌───────┐  │  │  │
> │  │  │  │ 内容  │  │  │  │
> │  │  │  └───────┘  │  │  │
> │  │  └─────────────┘  │  │
> │  └───────────────────┘  │
> └─────────────────────────┘
> ```

## Flexbox（横並びレイアウト）

ナビゲーションメニューやカード並びなど、 **要素を横に並べたい** ときに使います。

Flexbox は、親要素に `display: flex` を指定することで、その直下の子要素の並び方を制御します。次の CSS では `.container` の中の要素が横方向に並びます。

```css
.container {
  display: flex;                   /* Flexbox を有効にする */
  justify-content: space-between;  /* 横方向：均等に配置 */
  align-items: center;             /* 縦方向：中央に揃える */
  gap: 16px;                       /* 要素間の隙間 */
}
```

`justify-content` は横方向、`align-items` は縦方向の揃え方です。`gap` を使うと、子要素同士の間隔をまとめて指定できます。

HTML 側：

```html
<div class="container">
  <div>アイテム1</div>
  <div>アイテム2</div>
  <div>アイテム3</div>
</div>
```

この HTML では、`.container` が親要素で、3 つの `div` が子要素です。Flexbox の指定は親に書き、並べられる対象は子要素になる、という関係を押さえてください。

`justify-content` の主な値：

| 値 | 意味 |
|---|---|
| `flex-start` | 左揃え（デフォルト） |
| `flex-end` | 右揃え |
| `center` | 中央揃え |
| `space-between` | 両端に配置して均等な間隔 |

## Astro での CSS の書き方

Astro ファイルでは `<style>` タグを使います。ここに書いたスタイルは **そのファイルの中だけ** に適用されます（他のページに影響しないので安全です）。

次の例では、HTML と CSS を同じ `.astro` ファイル内に書いています。上にページの構造、下の `<style>` の中に見た目の指定を書く、という読み方をしてください。

```astro
<h1>タイトル</h1>
<p>本文テキスト</p>

<style>
  h1 {
    color: #1a1a2e;
    font-size: 2.5rem;
  }

  p {
    color: #555;
    line-height: 1.8;
  }
</style>
```

Astro ではこの CSS が基本的に同じコンポーネント内に閉じるため、別ページの `h1` まで勝手に変えてしまう心配が少なくなります。大きなサイトでは、この「影響範囲が狭い」ことが保守のしやすさにつながります。

## よくある疑問

**Q. `px`・`rem`・`%` など単位が色々あるけど、何を使えばいい？**  
A. 基本的に以下を使い分けましょう：
- `px`：border の太さなど固定したい場合
- `rem`：フォントサイズや余白（ブラウザの文字サイズ設定に追従する）
- `%`：親要素に対する割合で幅を指定したい場合

**Q. 色の `#333333` って何？**  
A. 16進数のカラーコードです。`#RRGGBB`（赤・緑・青）の形式で色を指定します。  
色を視覚的に選びたい場合は「カラーピッカー」で検索するとブラウザ上で試せます。
