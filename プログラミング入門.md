# プログラミング入門

プログラミングとは、**コンピュータに仕事をお願いする方法**です。毎回手作業でやっていたことを「こういう手順でやっておいて」と書き残せば、何度でも同じ仕事をやってもらえます。このページでは、プログラミングを始める前に知っておくべき「考え方」を整理します。

---

## プログラムってなんだろう

スマホのアプリ、Webサイト、ゲーム——これらはすべて「プログラム」です。プログラムとは、**コンピュータへの指示書**のことです。

```
レシピ（料理の手順）      プログラム（コンピュータへの手順）
─────────────────────    ──────────────────────────────
1. 水を200ml沸かす      1. データを読み込む
2. 卵を割って入れる    2. 特定の条件でフィルタをかける
3. 3分待つ             3. 合計を計算して表示する
4. できあがり          4. 結果をファイルに保存する
```

レシピが「誰でも同じ料理を作れる手順書」であるように、プログラムは「コンピュータが同じ動作を何度でも正確に繰り返せる手順書」です。

---

## なぜプログラミングを学ぶのか

「1000人分のアンケートを集計してください」と言われたとき、どうしますか？

```
手作業でやる場合:
  1000行を1行ずつ確認 → 数時間〜数日かかる
  うっかりミスが混ざる可能性がある

プログラムを使う場合:
  20行ほどのコードを書く → 1秒以内に完了
  何度やっても同じ結果（ミスがない）
```

プログラミングは「繰り返し・大量・正確」な仕事をコンピュータに任せる技術です。

### 何が作れるようになるか

| 分野 | 例 |
|------|-----|
| データ分析 | アンケート集計・売上グラフ作成・予測モデル |
| Web アプリ | ログイン機能・フォーム・API |
| 自動化 | ファイル整理・定期レポート送信・スクレイピング |
| AI・機械学習 | 画像認識・自然言語処理・推薦システム |

## プログラムがコンピュータで動く仕組み

コンピュータは「0か1かしかわからない」機械です。でも人間は `print("こんにちは")` のような読みやすい言葉で書けます。この橋渡しをするのが **プログラミング言語** です。

```
人間が書くコード
  print("こんにちは")
        ↓ 翻訳（インタープリタ or コンパイラ）
コンピュータが実行
  01001000 01100101 01101100 01101100 01101111...
```

プログラミング言語には Python・JavaScript・Java・C++ などたくさんの種類があります。「どれが正しい」はなく、目的によって使い分けます。

| 言語 | 向いている用途 |
|------|-------------|
| **Python** | データ分析・機械学習・自動化・Web バックエンド |
| **JavaScript** | Web の見た目・ブラウザ上の動作 |
| **SQL** | データベースの検索・集計 |
| **Bash（シェル）** | ターミナルでの自動化 |

このサイトでは主に **Python** を使います。読みやすく、データサイエンスと Web 両方で使えるからです。

## 最初のコード

Python をインストールしたら、ターミナルで `python3` と入力して試してみましょう。

```python
# # から始まる行は「コメント」。プログラムの動作には影響しません
print("こんにちは、世界！")
```

**実行結果：**
```
こんにちは、世界！
```

`print()` は「括弧の中のものを画面に表示する」命令です。これだけで動きます。

次は計算：

```python
print(1 + 1)    # → 2
print(10 * 5)   # → 50
print(2 ** 10)  # → 1024（2の10乗）
```

変数に値を保存して使うこともできます：

```python
name = "田中"
print("こんにちは、" + name + "さん")  # → こんにちは、田中さん
```

## エラーが出たときの読み方

プログラミングで一番怖いのが「赤いエラー」ですが、エラーは「どこを直せばいいか」を教えてくれる案内板です。怖がらないでください。

```python
# 間違えて書いてみる
prnt("こんにちは")   # print を間違えた
```

**エラーメッセージ：**
```
NameError: name 'prnt' is not defined
```

読み方：
- **NameError** — エラーの種類（名前が見つからない）
- **'prnt' is not defined** — `prnt` という名前が定義されていない

→ `prnt` を `print` に直せば解決です。

### よく出るエラーの種類

| エラー名 | 意味 | よくある原因 |
|---------|------|------------|
| `NameError` | 名前が見つからない | スペルミス・変数名の間違い |
| `SyntaxError` | 文法が間違っている | カッコが閉じていない・コロンを忘れた |
| `TypeError` | 型が合わない | 文字列と数値を足そうとした |
| `IndexError` | 番号が範囲外 | リストの存在しない番号を指定した |

**大切な考え方：エラーは失敗ではなく情報です。** プログラマーも毎日エラーを見ています。エラーを読んで→修正して→実行する、このサイクルがプログラミングそのものです。

## プログラマーの考え方

### 分解する

大きな問題を小さな問題に分けます。

```
「成績一覧から上位10人を抽出したい」
  ↓ 分解
  1. ファイルを読み込む
  2. 点数で並べ替える
  3. 上位10件を取り出す
  4. 表示する
```

### 繰り返しを見つける

「同じことを何度もやっている」と気づいたら、コードで自動化できます。

```python
# 手動でやると...
print("1番目: Alice")
print("2番目: Bob")
print("3番目: Carol")
# ...100人分書く？

# ループを使うと...
names = ["Alice", "Bob", "Carol", "Dave", "Eve"]
for i, name in enumerate(names, 1):
    print(f"{i}番目: {name}")
# → 5行でどんな人数にも対応できる
```

### 試して確認する

プログラミングは「完璧に書いてから実行」ではなく、**小さく書いてすぐ試す**のが基本です。

```
小さなコードを書く → 実行してみる → 期待通りか確認
    → 期待通りでないなら修正 → また実行
```
