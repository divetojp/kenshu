# 関数型プログラミング

> `.map()` `.filter()` `.reduce()` をすでに使っていますが、その背後にある考え方を学びます。

---

## はじめて読む人へ

関数型プログラミングは、処理を小さな関数の組み合わせとして考える書き方です。状態をむやみに変更せず、入力から出力を作る流れを明確にします。


### 読む前に押さえること

- 純粋関数は、同じ入力なら常に同じ出力を返します。
- 副作用を減らすと、テストやデバッグがしやすくなります。
- map、filter、reduce は、データの流れを表す道具です。

### 読み終えたら説明できること

- 純粋関数と副作用の違いを説明できる。
- 高階関数が何を受け取り何を返すか読める。
- リスト内包表記と関数型の考え方を比較できる。

---

## 関数型プログラミングとは

プログラミングには大きく 2 つの「スタイル」があります。

**命令型（Imperative）：「どうやるか」を書く**

```python
numbers = [1, 2, 3, 4, 5]
doubled = []
for n in numbers:
    doubled.append(n * 2)
```

このコードでは、「空のリストを作る」「1つずつ取り出す」「2倍する」「追加する」という手順を順番に書いています。コンピュータに対して作業手順を細かく指示するので、何が起きているかを追いやすい一方、処理が長くなると「結局このコードは何をしたいのか」が見えにくくなることがあります。

**関数型（Functional）：「何をするか」を書く**

```python
numbers = [1, 2, 3, 4, 5]
doubled = list(map(lambda n: n * 2, numbers))
```

こちらは「`numbers` の各要素に `n * 2` という変換を適用する」と読めます。`map` は元のデータを1つずつ取り出し、関数を適用した結果を返します。Python の `map` はそのままだと遅延評価されるオブジェクトなので、リストとして中身を見たいときに `list(...)` で囲んでいます。

関数型プログラミングの特徴：

1. **純粋関数**（同じ入力には必ず同じ出力）
2. **イミュータブル**（データを変更しない）
3. **副作用を避ける**（関数外の状態を変えない）
4. **高階関数**（関数を引数や戻り値として扱う）

---

## 純粋関数（Pure Function）

純粋関数は、同じ入力に対して常に同じ出力を返し、外側の状態を変えない関数です。この性質があると、関数単体で動作を予測しやすくなります。

たとえば、リストを受け取って新しいリストを返す関数は理解しやすいです。一方で、関数の中でグローバル変数を書き換えたり、ファイルへ保存したりすると、呼び出すたびに外側へ影響が出るため、テストやデバッグが難しくなります。

**純粋関数の条件：**

1. 同じ引数には必ず同じ結果を返す
2. 関数の外の状態を読み書きしない（副作用がない）

```python
# 純粋関数
def add(a, b):
    return a + b

add(3, 5)  # 何度呼んでも 8

# 純粋でない関数（副作用あり）
total = 0

def add_to_total(n):
    global total
    total += n   # 外の変数を変更している → 副作用
    return total

add_to_total(5)  # 5
add_to_total(5)  # 10 ← 同じ引数なのに違う結果
```

上の `add` は、入力 `a` と `b` だけで結果が決まります。関数の外側にある変数やファイル、現在時刻、乱数などに依存していないため、何度呼んでも同じ入力なら同じ結果になります。

一方、`add_to_total` は外側の `total` を変更しています。1回目と2回目で同じ `5` を渡しているのに結果が変わるのは、関数の外に状態を持っているからです。このような外部状態の読み書きを **副作用** と呼びます。副作用がすべて悪いわけではありませんが、どこで状態が変わるのかを意識しないと、バグの原因を追いにくくなります。

**なぜ重要か：**

- テストが簡単（入力と出力だけ確認すればいい）
- 並列処理と相性がいい（共有状態を変えないので競合しない）
- コードの動作が予測しやすい

## イミュータブル（不変性）

データを変更するのではなく、**新しいデータを作る** という考え方です。

```python
# ミュータブル（変更する）
scores = [70, 85, 60]
scores.append(90)     # 元のリストを変更
scores[0] = 75        # 元のリストを変更

# イミュータブル（新しいリストを作る）
scores = [70, 85, 60]
new_scores = scores + [90]         # 新しいリストを返す
updated_scores = [75] + scores[1:] # 新しいリストを返す
# scores は変わっていない
```

ミュータブルな書き方では、`scores` そのものの中身が変わります。短いコードでは問題になりにくいですが、同じリストを別の関数や別の変数も参照している場合、「いつの間にか値が変わっていた」という状況が起きます。

イミュータブルな書き方では、元のデータを残したまま新しいデータを作ります。メモリ使用量は増えることがありますが、処理の前後を比較しやすく、データ分析の前処理やテストでは特に扱いやすくなります。

```python
# タプルはイミュータブルなデータ構造
point = (3, 4)
# point[0] = 5  # TypeError: タプルは変更できない

# 「変更」したい場合は新しいタプルを作る
new_point = (5, point[1])
```

**JavaScript / TypeScript では：**

```ts
// const は再代入を防ぐが中身の変更は防がない
const arr = [1, 2, 3];
arr.push(4);       // これはできてしまう

// Object.freeze で完全に不変にする
const frozen = Object.freeze([1, 2, 3]);
frozen.push(4);    // TypeError

// スプレッド構文で新しい配列を作る（推奨パターン）
const arr = [1, 2, 3];
const newArr = [...arr, 4];  // arr を変えずに新しい配列を作る
```

## 高階関数（Higher-Order Function）

**関数を引数として受け取ったり、関数を戻り値として返したりする関数** です。

```python
# 関数を引数として受け取る
def apply(func, value):
    return func(value)

def double(x): return x * 2
def square(x): return x ** 2

print(apply(double, 5))  # 10
print(apply(square, 5))  # 25
```

`apply` は、具体的な処理を自分の中に固定していません。代わりに、受け取った `func` を `value` に適用します。つまり、「何をするか」を外から差し替えられる関数です。これが高階関数の基本的な使い方です。

```python
# 関数を返す（クロージャ）
def multiplier(n):
    def multiply(x):
        return x * n    # 外側の n を「覚えている」
    return multiply

double = multiplier(2)
triple = multiplier(3)

print(double(5))  # 10
print(triple(5))  # 15
```

`multiplier(2)` を呼ぶと、「受け取った値を2倍する関数」が作られます。`multiplier(3)` なら「3倍する関数」です。関数を返すことで、似た処理を毎回書かずに、条件だけを変えた処理を生成できます。

## クロージャ（Closure）

関数が **定義されたスコープの変数を覚えている** 仕組みです。

```python
def make_counter():
    count = 0                 # この count を関数が覚える
    def increment():
        nonlocal count
        count += 1
        return count
    return increment

counter = make_counter()
print(counter())  # 1
print(counter())  # 2
print(counter())  # 3
```

`make_counter()` の実行が終わっても、内側の `increment` は `count` を覚えています。この「関数が自分の外側にあった変数を保持する」性質がクロージャです。状態を持つので純粋関数ではありませんが、状態を狭い範囲に閉じ込められるため、グローバル変数を使うより安全です。

```js
// JavaScript
function makeCounter() {
    let count = 0;
    return () => {
        count++;
        return count;
    };
}

const counter = makeCounter();
console.log(counter());  // 1
console.log(counter());  // 2
```

## map / filter / reduce

Python の組み込み高階関数です。リストを「変換」するパターンをシンプルに書けます。

3つの役割は、データ処理の基本動作に対応しています。`map` は形を変える、`filter` は必要なものだけ残す、`reduce` は全体を1つの値へまとめる操作です。データサイエンスでは、表データの前処理や特徴量作成でも同じ発想を何度も使います。

### map：各要素を変換する

```python
numbers = [1, 2, 3, 4, 5]

# 命令型
doubled = []
for n in numbers:
    doubled.append(n * 2)

# 関数型（map）
doubled = list(map(lambda n: n * 2, numbers))

# リスト内包表記（Python らしい書き方）
doubled = [n * 2 for n in numbers]
```

`map` の例では、どちらの書き方でも「各要素を2倍した新しいリスト」を作っています。Python ではリスト内包表記の方が読みやすい場面が多いですが、`map` を知っておくと JavaScript や pandas の処理も理解しやすくなります。

### filter：条件に合う要素だけ取り出す

```python
numbers = [1, 2, 3, 4, 5, 6]

evens = list(filter(lambda n: n % 2 == 0, numbers))
# [2, 4, 6]

# リスト内包表記
evens = [n for n in numbers if n % 2 == 0]
```

`n % 2 == 0` は「2で割った余りが0」、つまり偶数であるという条件です。`filter` は条件を満たす要素だけを残します。データ分析でいうと、「売上が0より大きい行だけ」「欠損値がない行だけ」といった抽出に対応します。

### reduce：リストを 1 つの値にまとめる

```python
from functools import reduce

numbers = [1, 2, 3, 4, 5]

total = reduce(lambda acc, n: acc + n, numbers, 0)
# 0+1=1, 1+2=3, 3+3=6, 6+4=10, 10+5=15
# → 15

# 同じことを sum() で書けるが、reduce は「合計以外のまとめ方」にも使える
product = reduce(lambda acc, n: acc * n, numbers, 1)
# → 120（階乗）
```

`reduce` では `acc` が途中結果、`n` が次の要素です。最初に `0` や `1` のような初期値を置き、要素を1つずつ合成していきます。合計なら `sum()` の方が素直ですが、「辞書にまとめる」「最大値と最小値を同時に持つ」など、独自の集約をしたいときに考え方が役立ちます。

## 関数の合成（Function Composition）

複数の関数を組み合わせて新しい関数を作ります。

関数の合成は、「小さな変換を順番につなげて、大きな処理を作る」考え方です。たとえば、`3` に `add1`、`double`、`square` を順に適用すると、`3 → 4 → 8 → 64` と値が変化します。1つの巨大な関数を書くより、小さな関数を組み合わせる方がテストしやすく、途中の処理も確認しやすくなります。

```python
def add1(x): return x + 1
def double(x): return x * 2
def square(x): return x ** 2

# 手動で合成
result = square(double(add1(3)))  # add1(3)=4 → double=8 → square=64

# compose 関数を作る
from functools import reduce

def compose(*fns):
    return reduce(lambda f, g: lambda x: f(g(x)), fns)

transform = compose(square, double, add1)
print(transform(3))  # 64
```

**JavaScript ではパイプライン演算子（提案中）や `compose` ライブラリが使われます：**

```js
// 右から左に適用（compose）
const transform = (x) => square(double(add1(x)));

// lodash / ramda を使う場合
const transform = R.compose(square, double, add1);
```

## 実践：データ処理パイプライン

関数型スタイルは **データを変換する処理の流れ** を書くのに特に向いています。

ここでは「学生データの一覧」から、条件に合う学生だけを取り出し、点数順に並べ、名前だけを取り出します。これはデータ分析でよくある「抽出 → 並べ替え → 列選択」という流れです。先に処理の流れを日本語で書けるようにしてからコードにすると、何を実装すべきかが明確になります。

```python
students = [
    {"name": "田中", "score": 85, "grade": "B"},
    {"name": "山田", "score": 92, "grade": "A"},
    {"name": "佐藤", "score": 67, "grade": "C"},
    {"name": "鈴木", "score": 78, "grade": "B"},
    {"name": "高橋", "score": 55, "grade": "D"},
]
```

概念としては、次のようなパイプラインです。

1. `students` から grade が A または B の学生だけ残す
2. score の高い順に並べる
3. name だけ取り出す

実際の Python では、各段階の結果に名前をつけると読みやすくなります。

```python
eligible = [s for s in students if s["grade"] in ("A", "B")]
sorted_students = sorted(eligible, key=lambda s: s["score"], reverse=True)
names = [s["name"] for s in sorted_students]
print(names)  # ['山田', '田中', '鈴木']
```

`eligible` は条件を満たす学生のリスト、`sorted_students` は点数順に並べたリスト、`names` は名前だけのリストです。途中結果に意味のある名前をつけると、関数型の流れを保ちながら、初学者にも読みやすいコードになります。

## pandas との関係

pandas の操作は関数型スタイルに近いです。

pandas では DataFrame を直接書き換えるより、「条件で絞った DataFrame」「並べ替えた DataFrame」のように、処理結果を次の処理へ渡していく書き方がよく使われます。これは関数型プログラミングの「入力を受け取り、変換して、出力する」という考え方と相性がよいです。

```python
import pandas as pd

df = pd.DataFrame(students)

result = (
    df[df["grade"].isin(["A", "B"])]       # filter
    .sort_values("score", ascending=False)  # sort
    ["name"]                                # map（列選択）
)
print(result.tolist())  # ['山田', '田中', '鈴木']
```

このコードの各行は、先ほどの Python リスト処理と同じ意味を持っています。`df["grade"].isin(["A", "B"])` で条件を作り、`sort_values` で並べ替え、最後に `["name"]` で名前列だけを取り出しています。pandas のメソッドチェーンは、処理の流れを上から下へ読めるようにする書き方です。

`df.pipe()` を使うと関数の合成がより明示的に書けます。

```python
def filter_grade(df, grades):
    return df[df["grade"].isin(grades)]

def sort_by_score(df):
    return df.sort_values("score", ascending=False)

result = df.pipe(filter_grade, ["A", "B"]).pipe(sort_by_score)
```

`pipe` は「DataFrame を関数に渡し、その結果を次の関数へ渡す」ためのメソッドです。前処理が長くなったとき、処理を小さな関数に分けて `pipe` でつなぐと、どの段階で何をしているかが見えやすくなります。

## 命令型 vs 関数型：使い分け

| 場面 | 向いているスタイル |
|------|-----------------|
| データの変換・集計 | 関数型（map/filter/reduce） |
| 状態を持つ UI の更新 | 命令型（またはリアクティブ） |
| 並列処理 | 関数型（副作用がないと安全） |
| パフォーマンスが重要なループ | 命令型（オーバーヘッドが少ない） |
| テストしやすさを優先 | 関数型（純粋関数はテストが簡単） |

> 現実のコードは両方を組み合わせて書きます。「できるだけ関数型で書き、状態管理が必要なところだけ命令型にする」が良いバランスです。

## まとめ

| 概念 | 意味 |
|------|------|
| 純粋関数 | 同じ入力 → 同じ出力。副作用なし |
| イミュータブル | データを変更しない。新しいデータを作る |
| 高階関数 | 関数を引数・戻り値として扱う |
| クロージャ | 定義されたスコープの変数を記憶する関数 |
| map/filter/reduce | リストを変換・絞り込み・集約する高階関数 |
