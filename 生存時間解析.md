# 生存時間解析

「あるイベントが起きるまでの時間」を分析する統計手法です。患者の生存期間・機器の故障までの時間・顧客の解約までの期間——「打ち切り」という特殊なデータ構造を持つ時間データを正しく扱います。統計検定準1級の重要領域です。

---

## はじめて読む人へ

「20 人の患者を追跡したが、5 人は調査期間中にイベントが起きなかった（まだ生存している・転院した）」——このような**打ち切り（censoring）**が生存時間データの最大の特徴です。打ち切りデータを単純に除外したり 0 として扱うと、推定が偏ります。このページではその問題を正しく扱う方法を学びます。

## 生存時間データの特徴

### 基本用語

| 用語 | 意味 | 例（臨床試験） |
|------|------|-------------|
| **イベント（event）** | 分析対象の出来事 | 死亡・再発・解約 |
| **生存時間 $T$** | イベントまでの時間 | 治療開始から死亡まで |
| **打ち切り（censoring）** | 観察終了までイベント未発生 | 追跡中に転院・研究終了時点で生存 |
| **打ち切り時間 $C$** | 打ち切りが発生した時刻 | 最終観察日 |

### 打ち切りの種類

```
個体 1: ──────────●          (イベント発生)
個体 2: ──────────────○      (右打ち切り: 観察終了時に未発生)
個体 3: ────○                (右打ち切り: 途中脱落)
個体 4:      ──────────●    (左打ち切り: イベント前の時間不明)
              ↑
              0       追跡期間

●: イベント発生  ○: 打ち切り
```

**右打ち切り（Right Censoring）** が最も一般的です。「観察している間にイベントが起きなかった」ことは分かるが、その後いつ起きるかは不明です。

### なぜ通常の回帰では不十分か

打ち切り個体を除外すると：
- 長期生存者を除くことで生存時間を**過小推定**
- サンプルサイズが減り検出力が低下

打ち切り時間をイベント時間として使うと：
- イベントが「起きた」と誤判定するため**過小推定**

→ 打ち切りを正しく組み込んだ**尤度関数**が必要です。

## 生存関数とハザード関数

### 生存関数 $S(t)$

時刻 $t$ まで生存している（イベントが起きない）確率：

$$
S(t) = P(T > t), \quad S(0) = 1, \quad S(\infty) = 0
$$

単調減少関数で、グラフは右下がりの「階段状の曲線」になります。

### ハザード関数 $h(t)$

時刻 $t$ まで生存した個体が、次の瞬間にイベントが起きる**瞬間的リスク**：

$$
h(t) = \lim_{\Delta t \to 0} \frac{P(t \leq T < t + \Delta t \mid T \geq t)}{\Delta t}
$$

**生存関数との関係：**

$$
S(t) = \exp\!\left(-\int_0^t h(u)\, du\right) = \exp(-H(t))
$$

$H(t) = \int_0^t h(u)\, du$ は**累積ハザード関数**です。

### 累積ハザード $H(t)$ の直感

$H(t)$ は「時刻 $t$ までの「危険の蓄積量」です。$H(t) = 1$ のとき $S(t) \approx 0.368$（約 37% が生存）。

## カプラン・マイヤー推定量

ノンパラメトリックな生存関数の推定法です。分布の仮定が不要です。

### 計算方法

イベント時刻 $t_1 < t_2 < \cdots < t_k$ に対して：

$$
\hat{S}(t) = \prod_{t_i \leq t} \left(1 - \frac{d_i}{n_i}\right)
$$

- $d_i$：時刻 $t_i$ でのイベント数
- $n_i$：時刻 $t_i$ 直前の生存者数（リスクセット）

### 具体例

| 時刻 $t_i$ | $n_i$（生存者）| $d_i$（イベント）| $\hat{S}(t_i)$ |
|-----------|-------------|----------------|--------------|
| 3 | 10 | 1 | $(1 - 1/10) = 0.900$ |
| 5 | 8 | 2 | $0.900 \times (1 - 2/8) = 0.675$ |
| 8 | 5 | 1 | $0.675 \times (1 - 1/5) = 0.540$ |

（打ち切りがあっても $n_i$ から除外するだけで計算できます）

### KM 曲線の読み方

```
S(t)
1.0 ─┐
     │  ─┐
0.75 │    │  ─┐
     │    │   │  ─┐  ─
0.5  │    │   │   │
     │    └   └   └
0.25 │
     └──────────────── t (月)
     0    3   5   8
     
     縦の垂線（↓）: イベント発生
     +マーク: 打ち切り（その時点では生存）
```

### 信頼区間（Greenwood の公式）

$$
\text{Var}[\hat{S}(t)] \approx \hat{S}(t)^2 \sum_{t_i \leq t} \frac{d_i}{n_i(n_i - d_i)}
$$

## ログランク検定

2 つ（以上）のグループ（例：治療 A vs 治療 B）の生存関数が等しいかを検定します。

### 検定統計量

各イベント時刻で「期待されるイベント数」を計算し、実際の観測数との差を積み上げます。

$$
W = \frac{(O_1 - E_1)^2}{E_1} + \frac{(O_2 - E_2)^2}{E_2} \sim \chi^2(1)
$$

- $O_g$：グループ $g$ の実際のイベント数
- $E_g = n_{g,i} / n_i \times d_i$ の合計（各時点でリスクセットの割合だけ期待される）

**直感：** 2 グループが同じ生存関数なら、各時点のイベントはリスクセットのサイズに比例して配分されるはず → それからの偏差が統計量になります。

### 加重ログランク検定

Gehan-Wilcoxon 検定（初期のイベントを重視）など、時点によって重みを変えた変種があります。ハザードの比が時間によって変化する場合に有用です。

---

## Cox 比例ハザードモデル

共変量（年齢・治療群・血圧など）が生存時間に与える影響を推定する半パラメトリックなモデルです。

### モデルの形

$$
h(t \mid \mathbf{x}) = h_0(t) \cdot \exp(\beta_1 x_1 + \beta_2 x_2 + \cdots + \beta_p x_p)
$$

- $h_0(t)$：**ベースラインハザード**（ノンパラメトリックに推定、$\mathbf{x} = \mathbf{0}$ のとき）
- $\exp(\boldsymbol{\beta}^\top \mathbf{x})$：共変量による乗算的な調整

### 比例ハザード仮定

2 つのグループ（$\mathbf{x}_1, \mathbf{x}_2$）のハザード比が時間に依存しない：

$$
\frac{h(t \mid \mathbf{x}_1)}{h(t \mid \mathbf{x}_2)} = \frac{\exp(\boldsymbol{\beta}^\top \mathbf{x}_1)}{\exp(\boldsymbol{\beta}^\top \mathbf{x}_2)} = \text{定数}
$$

→ KM 曲線で 2 グループが交差する（ハザード比が時間変化する）場合はこの仮定が破れています。

### ハザード比の解釈

$$
\text{HR} = \exp(\beta_j)
$$

$x_j$ が 1 増えたとき、ハザードが $\exp(\beta_j)$ 倍になります。

| $\hat{\beta}_j$ | HR | 解釈 |
|----------------|-----|------|
| $< 0$ | $< 1$ | $x_j$ が増えると生存しやすい（保護因子） |
| $= 0$ | $= 1$ | $x_j$ はハザードと無関係 |
| $> 0$ | $> 1$ | $x_j$ が増えると危険（リスク因子） |

### 部分尤度推定

$h_0(t)$ を推定しなくても $\boldsymbol{\beta}$ を推定できる、Cox の天才的なアイデアです。

$$
L(\boldsymbol{\beta}) = \prod_{i: \text{イベント}} \frac{\exp(\boldsymbol{\beta}^\top \mathbf{x}_i)}{\sum_{j \in \mathcal{R}(t_i)} \exp(\boldsymbol{\beta}^\top \mathbf{x}_j)}
$$

$\mathcal{R}(t_i)$：時刻 $t_i$ でのリスクセット（まだイベントが起きていない個体の集合）。

## lifelines による実装

```python
from lifelines import KaplanMeierFitter, CoxPHFitter
from lifelines.statistics import logrank_test
import pandas as pd

# データ例
df = pd.DataFrame({
    'duration': [5, 8, 12, 3, 15, 7, 10, 4],
    'event':    [1, 1,  0, 1,  1, 0,  1, 1],   # 1: イベント, 0: 打ち切り
    'group':    ['A','A','A','A','B','B','B','B'],
    'age':      [60, 55, 72, 48, 63, 51, 68, 45]
})

# カプラン・マイヤー推定
kmf = KaplanMeierFitter()
for grp in ['A', 'B']:
    mask = df['group'] == grp
    kmf.fit(df[mask]['duration'], df[mask]['event'], label=grp)

# ログランク検定
result = logrank_test(
    df[df.group=='A']['duration'], df[df.group=='B']['duration'],
    event_observed_A=df[df.group=='A']['event'],
    event_observed_B=df[df.group=='B']['event']
)
print(f"p値: {result.p_value:.4f}")

# Cox 比例ハザードモデル
cph = CoxPHFitter()
cph.fit(df[['duration','event','age']], duration_col='duration', event_col='event')
cph.print_summary()  # ハザード比・95%CI・p値を表示
```

## 数学的導出

### 尤度関数の構築

打ち切りがある場合の尤度を正しく書きます。

イベントが時刻 $t_i$ に発生した個体 $i$ の尤度：$f(t_i) = h(t_i) S(t_i)$

打ち切りが時刻 $c_i$ に起きた個体 $i$ の尤度：$S(c_i)$（$c_i$ 時点で生存していることが分かっているだけ）

全体の尤度：

$$
L = \prod_{i: \text{イベント}} f(t_i) \cdot \prod_{i: \text{打ち切り}} S(c_i) = \prod_{i: \text{イベント}} h(t_i) \cdot \prod_i S(t_i \wedge c_i)
$$

$h_0(t)$ に指数分布 $h_0 = \lambda$（定数）を仮定すると：

$$
L = \prod_{i: \delta_i=1} \lambda e^{\boldsymbol{\beta}^\top \mathbf{x}_i} \cdot \exp\!\left(-\lambda e^{\boldsymbol{\beta}^\top \mathbf{x}_i} t_i\right)
$$

Cox モデルの部分尤度は $h_0(t)$ を消去することでこれよりシンプルな形になります。

### Greenwood の分散公式の導出

$\hat{S}(t_k) = \prod_{i=1}^k (1 - d_i/n_i)$ の分散をデルタ法で求めます。

$$
\log \hat{S}(t_k) = \sum_{i=1}^k \log(1 - d_i/n_i)
$$

各項が独立で、$d_i/n_i \ll 1$ の近似のもとで：

$$
\text{Var}[\log \hat{S}(t_k)] \approx \sum_{i=1}^k \frac{d_i}{n_i^2 (1-d_i/n_i)^{-1}} \approx \sum_{i=1}^k \frac{d_i}{n_i(n_i-d_i)}
$$

デルタ法 $\text{Var}[g(X)] \approx [g'(\mu)]^2 \text{Var}[X]$ で $g(u) = e^u$ を使うと Greenwood の公式になります。
