# バンディットアルゴリズム

**探索と活用のトレードオフ**（Exploration-Exploitation Dilemma）を解く逐次意思決定の手法群です。ε-greedy・UCB・Thompson Sampling・Contextual Bandit を扱います。オンライン広告のクリック率最適化・推薦システム・A/Bテストの逐次実施・LLM のハイパーパラメータ調整まで幅広く使われます。

---

## はじめて読む人へ

「3 台のスロットマシンがあり、各マシンの当たる確率は知らない。限られた回数で最大の賞金を稼ぐにはどう試すか」——これがバンディット問題（多腕バンディット）です。ずっと試さないと（探索なし）最良のマシンを見つけられないが、試しすぎると（活用なし）稼げない。このトレードオフが現実の多くの問題に現れます。

### 読む前に押さえること

- [ベイズ理論](ベイズ理論) — 事前/事後分布（Thompson Sampling の理解のため）
- [確率・統計基礎](確率・統計基礎) — 確率分布・信頼区間

### 読み終えたら説明できること

- 後悔（Regret）の概念を説明できる
- UCB が「不確実な選択肢を優先する」理由を信頼区間の観点から説明できる
- Thompson Sampling がベイズ更新を活用する仕組みを説明できる

## 問題設定

### 多腕バンディット（Multi-Armed Bandit, MAB）

```
K 台のスロットマシン（アーム）: a₁, a₂, ..., aₖ
各アームの真の期待報酬（未知）: μ₁, μ₂, ..., μₖ

各ラウンド t で:
  1. アーム aₜ を選択
  2. 報酬 rₜ を観測（確率的）
  3. 次の選択に活かす

目標: T ラウンドの累積報酬を最大化
```

### 後悔（Regret）

最適なアームを常に選び続けた場合と比較した**損失の累積**：

$$
R_T = T \cdot \mu^* - \sum_{t=1}^T \mathbb{E}[r_{t}]
$$

$\mu^* = \max_k \mu_k$：最良アームの期待報酬。良いアルゴリズムほど $R_T$ が小さくなります。

**サブ線形 Regret：** 良いアルゴリズムは $R_T = O(\sqrt{T} \log T)$ 程度（$T$ より遅く増加）。ランダムに選ぶと $R_T = O(T)$（線形 Regret、最悪）。

## ε-greedy

### アルゴリズム

```
確率 ε: ランダムにアームを選ぶ（探索）
確率 1-ε: 現在の最大推定報酬のアームを選ぶ（活用）
```

**パラメータの選択：**
- $\varepsilon = 0.1$（10% 探索）が一般的な初期値
- $\varepsilon$ を時間とともに減衰させる（$\varepsilon_t = 1/\sqrt{t}$ など）と Regret が改善

### 問題点

- **不均一な探索：** 劣ったアームにも同じ確率で探索予算を割く
- **信頼度を無視：** まだ十分に試していないアームと十分試したアームを区別しない

---

## UCB（Upper Confidence Bound）

### 「楽観主義のもとで不確実性を優先する」

**UCB1 の選択則：**

$$
a_t = \arg\max_{a} \left[\hat{\mu}_a + \sqrt{\frac{2 \ln t}{N_a}}\right]
$$

- $\hat{\mu}_a$：アーム $a$ のこれまでの平均報酬（活用）
- $\sqrt{2 \ln t / N_a}$：信頼区間の幅（探索ボーナス；試行回数 $N_a$ が少ないほど大きい）

**直感：**

```
よく試したアーム（N_a 大）: ボーナス小 → 推定が確実 → 活用を優先
あまり試していないアーム（N_a 小）: ボーナス大 → 推定が不確実 → 探索を優先
```

### 理論的保証

UCB1 の累積 Regret の上界（Auer et al., 2002）：

$$
R_T \leq \sum_{a: \mu_a < \mu^*} \frac{8 \ln T}{\Delta_a} + \left(1 + \frac{\pi^2}{3}\right)\sum_{a: \mu_a < \mu^*} \Delta_a
$$

$\Delta_a = \mu^* - \mu_a$：アーム $a$ の劣化量。劣っているアームほど少ない試行で見切りをつける（$\Delta_a$ が大きいほど早く収束）。

## Thompson Sampling（ベイズバンディット）

### アイデア

各アームの期待報酬についての**事後分布からサンプリング**し、最大値のアームを選びます。

**Beta-Bernoulli バンディット（クリック率の最適化）：**

```python
import numpy as np

# 各アームの成功回数・失敗回数を初期化
alpha = np.ones(K)   # 事前分布: Beta(1, 1) = 均一
beta_  = np.ones(K)

for t in range(T):
    # 各アームの報酬をサンプリング
    samples = np.random.beta(alpha, beta_)

    # 最大サンプルのアームを選ぶ
    arm = np.argmax(samples)

    # 報酬を観測してベイズ更新
    reward = observe_reward(arm)
    if reward == 1:
        alpha[arm] += 1   # 成功 → alpha を増やす
    else:
        beta_[arm] += 1   # 失敗 → beta を増やす
```

**ベイズ更新の役割：** 試行回数が増えるほど事後分布が狭まり（確信が高まり）、サンプル間のばらつきが小さくなります → 自然に探索から活用へ移行。

### UCB vs Thompson Sampling

| 観点 | UCB | Thompson Sampling |
|------|-----|-------------------|
| 理論的保証 | 厳密な Regret 界 | ベイズ的保証 |
| 実装のシンプルさ | シンプル | やや複雑（分布の選択） |
| 実務性能 | 良い | しばしば UCB を上回る |
| 計算コスト | 低い | サンプリングが必要 |

## Contextual Bandit

各ラウンドに**文脈情報**（コンテキスト $x_t$）が与えられます。ユーザーの特徴量・ページの属性などを考慮した上で最良の行動を選びます。

```
MAB（文脈なし）:
  各ラウンド → アームを選択 → 報酬観測

Contextual Bandit（文脈あり）:
  ユーザーの特徴 xₜ → モデルがアームを選択 → 報酬観測 → モデル更新
```

### LinUCB

線形報酬モデル $r(x, a) = \theta_a^\top x + \text{noise}$ を仮定した UCB：

$$
a_t = \arg\max_a \left[\theta_a^\top x_t + \alpha \sqrt{x_t^\top A_a^{-1} x_t}\right]
$$

$A_a = X_a^\top X_a + I$：アーム $a$ の特徴量行列の共分散。$\sqrt{x_t^\top A_a^{-1} x_t}$ は文脈 $x_t$ での予測の不確実性（信頼楕円体の幅）です。

## 応用

### オンライン広告（CTR 最適化）

- アーム：各広告クリエイティブ
- 報酬：クリック（1）/ 非クリック（0）
- Thompson Sampling で配信割合を逐次最適化

### A/Bテストの改善（Multi-Armed Bandit Testing）

通常の A/B テストはデータ収集終了まで結果を使わないが、MAB は途中から劣った variant の配信を自動的に減らします。

```
通常の A/B テスト:  [A: 50%] [B: 50%] ... (固定)
MAB A/B テスト:    [A: 50%] [B: 50%] → [A: 40%] [B: 60%] → ...
                   B の結果が良ければ自動的に配信を増やす
```

**注意：** 統計的有意性を重視する場合は通常の A/B テストの方が適切（MAB は最終的な「勝者」を明確に主張しにくい）。

### 推薦システム

ユーザーへの記事・商品推薦。ユーザーの過去行動をコンテキストとした Contextual Bandit として定式化できます。

## 数学的導出

### UCB1 の信頼区間の根拠（Hoeffding の不等式）

アーム $a$ を $N_a$ 回引いたときの平均報酬 $\hat{\mu}_a$ について、Hoeffding の不等式から：

$$
P\!\left[\mu_a > \hat{\mu}_a + \varepsilon\right] \leq e^{-2N_a \varepsilon^2}
$$

$e^{-2N_a \varepsilon^2} = t^{-4}$（$\varepsilon = \sqrt{2\ln t / N_a}$）とおくと：

$$
P\!\left[\mu_a > \hat{\mu}_a + \sqrt{\frac{2\ln t}{N_a}}\right] \leq t^{-4}
$$

信頼上界が高確率で成立します。UCB1 はこの「高確率で真の平均より大きい上界」を最大化することで、楽観的に探索します。

## 確認問題

1. 後悔（Regret）を最小化することが「最大の累積報酬を得ること」と等価な理由を説明してください。
2. UCB で試行回数 $N_a$ が少ないアームほど探索ボーナスが大きくなる理由を「不確実性の測り方」で説明してください。
3. Thompson Sampling で「試行を重ねると自然に活用が増える」理由をベイズ更新と事後分布の関係から説明してください。
4. 通常の A/B テストより MAB Testing が優れる場面と劣る場面をそれぞれ挙げてください。
