# 時系列分析

時間の順序を持つデータ（株価・気温・売上・センサー値）を分析・予測する手法です。「過去のパターンから未来を予測する」という目標は機械学習と同じですが、**データが時間的に依存し合っている**という特性を明示的に扱います。

---

## はじめて読む人へ

通常の機械学習では「データの順序は関係ない」と仮定します。しかし時系列データでは「昨日の売上」が「今日の売上」に影響します。この依存関係を無視してランダムフォレストに突っ込んでも、よい予測はできません。

このページでは**統計的アプローチ**（ARIMA・Prophet）を扱います。ディープラーニングによる時系列予測（LSTM・Transformer）は [RNN・時系列](RNN-時系列.md) を参照してください。

### 読む前に押さえること

- [確率・統計基礎](確率・統計基礎.md) の平均・分散・相関の概念
- [NumPy / pandas](pandas-sklearn.md) の DataFrame 操作

### 読み終えたら説明できること

- トレンド・季節性・残差への分解を説明できる
- ARIMA の p・d・q パラメータの意味を説明できる
- Prophet でビジネスデータを予測する実装ができる

## 時系列データの構成要素

どんな時系列データも次の 3 要素に分解できます。

!!! info ""
    売上データ =  トレンド   +  季節性     +  残差
                （長期傾向）  （周期的変動）  （説明できない変動）
    
    例：アイスクリームの月別売上
      トレンド：年々少し増えている
      季節性 ：夏に跳ね上がり、冬に落ちる
      残差   ：猛暑・台風など予測外の変動

```python
import pandas as pd
import matplotlib.pyplot as plt
from statsmodels.tsa.seasonal import seasonal_decompose

# サンプルデータ（月別売上）
dates = pd.date_range("2020-01", periods=48, freq="ME")
import numpy as np
rng = np.random.default_rng(42)
trend = np.linspace(100, 160, 48)
seasonal = 30 * np.sin(np.linspace(0, 8 * np.pi, 48))
noise = rng.normal(0, 5, 48)
sales = pd.Series(trend + seasonal + noise, index=dates)

# 分解
result = seasonal_decompose(sales, model="additive", period=12)
result.plot()
plt.tight_layout()
plt.savefig("decompose.png", dpi=150)
```

## 定常性

時系列分析の多くの手法は「データが定常（平均・分散が時間によらず一定）」という仮定に基づきます。

!!! info ""
    非定常（使えない）: 株価、売上の右肩上がりデータ
    定常（使える）  : 前日との差分（リターン）、季節調整済みデータ

**単位根検定（ADF 検定）** で定常かどうかを確認できます。

```python
from statsmodels.tsa.stattools import adfuller

result = adfuller(sales)
print(f"ADF 統計量: {result[0]:.4f}")
print(f"p 値:      {result[1]:.4f}")
# p値 < 0.05 → 定常（差分不要）
# p値 >= 0.05 → 非定常 → 差分を取る
```

差分を取ることで非定常データを定常化できます。

```python
sales_diff = sales.diff().dropna()  # 1階差分
sales_diff2 = sales.diff().diff().dropna()  # 2階差分（まれに必要）
```

## 移動平均（Moving Average）

直近 N 期間の平均を取ることで、ノイズを除去してトレンドを見やすくします。

```python
# 単純移動平均（SMA）
sma_3  = sales.rolling(window=3).mean()   # 3期移動平均
sma_12 = sales.rolling(window=12).mean()  # 年間移動平均

# 指数移動平均（EMA）: 直近を重視
ema = sales.ewm(span=6, adjust=False).mean()

plt.figure(figsize=(12, 4))
plt.plot(sales, label="実績", alpha=0.5)
plt.plot(sma_12, label="12期SMA", linewidth=2)
plt.plot(ema, label="EMA(span=6)", linewidth=2)
plt.legend()
plt.savefig("moving_average.png", dpi=150)
```

## ARIMA

**AR（自己回帰）+ I（差分）+ MA（移動平均誤差）** を組み合わせた古典的な時系列モデルです。

| パラメータ | 意味 |
|-----------|------|
| **p** | 何期前までの値を使うか（AR の次数） |
| **d** | 何回差分を取るか（定常化に必要な回数） |
| **q** | 何期前までの誤差を使うか（MA の次数） |

```python
from statsmodels.tsa.arima.model import ARIMA
from sklearn.metrics import mean_squared_error

# 訓練・テスト分割（時系列は必ず時間順）
train = sales[:-12]
test  = sales[-12:]

# ARIMA(1,1,1) → p=1, d=1, q=1
model = ARIMA(train, order=(1, 1, 1))
result = model.fit()

# 予測
forecast = result.forecast(steps=12)

rmse = mean_squared_error(test, forecast, squared=False)
print(f"RMSE: {rmse:.2f}")

plt.figure(figsize=(12, 4))
plt.plot(train, label="訓練")
plt.plot(test,  label="実績", color="green")
plt.plot(forecast, label="予測", color="red", linestyle="--")
plt.legend()
plt.savefig("arima_forecast.png", dpi=150)
```

### 最適な p, d, q の選び方

手動で探すか `auto_arima` を使います。

```python
# pip install pmdarima
from pmdarima import auto_arima

model = auto_arima(
    train,
    seasonal=True,
    m=12,                  # 季節周期（月次データなら12）
    stepwise=True,
    information_criterion="aic",
    trace=True             # 探索過程を表示
)
print(model.summary())
```

`AIC`（赤池情報量基準）が最小になる組み合わせを自動選択します。

## SARIMA（季節性 ARIMA）

月次・週次など明確な季節性があるデータには SARIMA を使います。

```python
from statsmodels.tsa.statespace.sarimax import SARIMAX

# SARIMA(p,d,q)(P,D,Q,m) → 季節パラメータ付き
model = SARIMAX(train, order=(1, 1, 1), seasonal_order=(1, 1, 1, 12))
result = model.fit(disp=False)

forecast = result.forecast(steps=12)
print(f"RMSE: {mean_squared_error(test, forecast, squared=False):.2f}")
```

## Prophet

Facebook（Meta）が開発した時系列予測ライブラリです。ARIMA に比べて**設定が少なく、ビジネスデータに強い**です。祝日・イベントの効果も組み込めます。

```bash
pip install prophet
```

```python
from prophet import Prophet
import pandas as pd

# Prophet は ds（日付）と y（値）の列を持つ DataFrame を入力とする
df = sales.reset_index()
df.columns = ["ds", "y"]

train_df = df[:-12]
test_df  = df[-12:]

# モデル構築
m = Prophet(
    yearly_seasonality=True,
    weekly_seasonality=False,
    daily_seasonality=False
)
m.fit(train_df)

# 予測（12期先）
future = m.make_future_dataframe(periods=12, freq="ME")
forecast = m.predict(future)

# 可視化
fig = m.plot(forecast)
fig.savefig("prophet_forecast.png", dpi=150)

# トレンド・季節性の分解
fig2 = m.plot_components(forecast)
fig2.savefig("prophet_components.png", dpi=150)
```

### 祝日・イベントの追加

```python
from prophet import Prophet

# 日本の祝日を追加
m = Prophet(yearly_seasonality=True)
m.add_country_holidays(country_name="JP")

# カスタムイベント（セール期間）
sale_events = pd.DataFrame({
    "holiday": "summer_sale",
    "ds": pd.to_datetime(["2021-07-15", "2022-07-20", "2023-07-18"]),
    "lower_window": 0,
    "upper_window": 7,   # 前後7日間影響
})
m = Prophet(holidays=sale_events)
m.fit(train_df)
```

## 評価指標

```python
import numpy as np
from sklearn.metrics import mean_squared_error, mean_absolute_error

y_true = test.values
y_pred = forecast["yhat"].values[-12:]

rmse = mean_squared_error(y_true, y_pred, squared=False)
mae  = mean_absolute_error(y_true, y_pred)
mape = np.mean(np.abs((y_true - y_pred) / y_true)) * 100  # 平均絶対誤差率(%)

print(f"RMSE: {rmse:.2f}")
print(f"MAE:  {mae:.2f}")
print(f"MAPE: {mape:.1f}%")
```

| 指標 | 意味 | 特徴 |
|------|------|------|
| RMSE | 二乗平均平方根誤差 | 大きな誤差を重視 |
| MAE | 平均絶対誤差 | 外れ値に強い |
| MAPE | 平均絶対誤差率（%） | スケールに依存しない・解釈しやすい |

## 手法の使い分け

| 状況 | 推奨 |
|------|------|
| 季節性・トレンドが明確なビジネスデータ | Prophet |
| 統計的な厳密さが必要・学術用途 | SARIMA |
| データが少ない（< 50 点） | ARIMA / SARIMA |
| 長期・複雑なパターン | LSTM / Transformer（[RNN・時系列](RNN-時系列.md)参照）|
