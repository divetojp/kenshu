モデル評価・チューニング¶

「精度が高い」とはどういう意味でしょうか。精度（Accuracy）が 95% と言っても、それが「クラス不均衡なデータで多数派を予測し続けた結果」である可能性があります。モデルを正しく評価するには、何を達成したいかに応じた評価指標を選ぶことが重要です。

はじめて読む人へ¶

モデル評価は、作ったモデルが未知データでも役に立つかを確認する作業です。チューニングは、モデルの設定を変えて性能を改善する作業です。

読む前に押さえること¶

  • 評価指標は、問題の目的に合わせて選びます。
  • 交差検証は、データ分割による偶然を減らすために使います。
  • ハイパーパラメータ探索は、やみくもに試すのではなく検証データで比較します。

読み終えたら説明できること¶

  • 分類・回帰の代表的な評価指標を説明できる。
  • 交差検証の目的を理解できる。
  • GridSearch、RandomizedSearch、Optuna の違いを説明できる。

評価指標¶

評価指標は、モデルの良さをどの観点で測るかを決めるものです。分類では正解率、適合率、再現率、F1、AUC などがあり、それぞれ重視する失敗の種類が違います。

たとえば病気検出では、病気の人を見逃すことが大きな問題になるため、再現率が重要になることがあります。一方、誤検知が多すぎると運用できない場面では、適合率も見ます。

混同行列（Confusion Matrix）¶

分類モデルの予測結果は、以下の 4 種類の組み合わせで整理できます。

混同行列は、分類モデルの「当たり方」と「外し方」を分けて見る表です。単なる正解率ではなく、見逃しと誤検知を分けて考えるための出発点になります。

予測: 陽性 予測: 陰性 実際: 陽性 TP（真陽性） FN（偽陰性） 実際: 陰性 FP（偽陽性） TN（真陰性） - TP（True Positive）：陽性と予測し、実際に陽性だった - FP（False Positive）：陽性と予測したが、実際は陰性だった（誤検知） - FN（False Negative）：陰性と予測したが、実際は陽性だった（見逃し） - TN（True Negative）：陰性と予測し、実際に陰性だった

この 4 つの値から、目的に応じたさまざまな指標が計算できます。

分類の評価指標¶

次のコードでは、分類モデルの代表的な評価指標をまとめて計算します。y_true は正解ラベル、y_pred はモデルの予測ラベルです。

from sklearn.metrics import (
    accuracy_score, precision_score, recall_score, f1_score,
    classification_report, confusion_matrix, roc_auc_score
)

y_true = [0, 1, 1, 0, 1, 0, 1, 1]
y_pred = [0, 1, 0, 0, 1, 1, 1, 1]

print(f"Accuracy : {accuracy_score(y_true, y_pred):.3f}")
print(f"Precision: {precision_score(y_true, y_pred):.3f}")
print(f"Recall   : {recall_score(y_true, y_pred):.3f}")
print(f"F1       : {f1_score(y_true, y_pred):.3f}")

# 全クラスをまとめたレポート
print(classification_report(y_true, y_pred))

# 混同行列の可視化
from sklearn.metrics import ConfusionMatrixDisplay
import matplotlib.pyplot as plt
ConfusionMatrixDisplay.from_predictions(y_true, y_pred)
plt.savefig("confusion_matrix.png")

classification_report は、precision、recall、F1 をクラスごとにまとめて表示します。混同行列の可視化を見ると、どのクラスをどのクラスに間違えやすいかが分かります。

指標 計算式 意味 重視する場面
Accuracy (TP + TN) / 全体 全体の正解率 クラスが均等なとき
Precision TP / (TP + FP) 「陽性と予測」の中で本当に陽性の割合 誤検知を減らしたい（スパムフィルタ）
Recall TP / (TP + FN) 「本当の陽性」のうち正しく検出できた割合 見逃しを減らしたい（がん検診）
F1 2 × P × R / (P + R) Precision と Recall の調和平均 クラス不均衡データ
ROC-AUC 曲線下面積 閾値によらない総合的な判別能力 スコアのランキング評価

Accuracy は不均衡データに使えません。 たとえば「陽性 5%・陰性 95%」のデータで「常に陰性を返すモデル」は Accuracy 95% を達成しますが、陽性を一件も検出できていません。こうした場合は F1 や ROC-AUC を使います。

ROC 曲線と AUC¶

ROC 曲線は「分類の閾値を変化させたとき、FPR（偽陽性率）と TPR（真陽性率）がどう変化するか」をプロットしたものです。AUC（Area Under Curve）はその面積で、1 に近いほど良いモデルです。0.5 はランダム（斜め線）、0.7 以上で実用的な水準とされます。

確率を出すモデルでは、しきい値を 0.5 に固定せず、目的に応じて変えることがあります。ROC 曲線は、そのしきい値を動かしたときの性能変化を見るための図です。

from sklearn.metrics import roc_curve, auc
import matplotlib.pyplot as plt

y_proba = model.predict_proba(X_test)[:, 1]  # 陽性クラスの確率
fpr, tpr, thresholds = roc_curve(y_test, y_proba)
roc_auc = auc(fpr, tpr)

plt.plot(fpr, tpr, label=f"ROC AUC = {roc_auc:.3f}")
plt.plot([0, 1], [0, 1], "k--", label="ランダム")
plt.xlabel("偽陽性率（FPR）")
plt.ylabel("真陽性率（TPR）")
plt.legend()
plt.savefig("roc.png", dpi=150)

y_proba は陽性クラスである確率です。roc_curve はさまざまなしきい値での FPR と TPR を計算し、auc はその曲線の面積を計算します。

回帰の評価指標¶

回帰では、予測値と実測値のズレを評価します。ズレをどのように集計するかによって、外れ値への敏感さや解釈のしやすさが変わります。

from sklearn.metrics import mean_squared_error, mean_absolute_error, r2_score
import numpy as np

rmse = np.sqrt(mean_squared_error(y_test, y_pred))
mae = mean_absolute_error(y_test, y_pred)
r2 = r2_score(y_test, y_pred)

print(f"RMSE: {rmse:.3f}  MAE: {mae:.3f}  R²: {r2:.3f}")

RMSE と MAE は、目的変数と同じ単位で誤差を読めます。R² は、平均値だけで予測するモデルと比べてどれくらい説明できているかを見る指標です。

  • RMSE：誤差を二乗するため、大きな外れ値に敏感
  • MAE：外れ値に頑健。「平均的に何単位ずれているか」を直感的に表す
  • R²：0〜1（1 が完全予測、0 は平均値モデルと同等、負は平均より悪い）

交差検証（クロスバリデーション）¶

単純にデータをランダムに分割して評価すると、「たまたまテストデータが簡単だった」という運による誤差が生じます。交差検証 はデータを複数のブロック（fold）に分割し、「各 fold をテストデータにして残りで学習する」ことを繰り返し、複数の評価値の平均と分散を計算します。これにより評価の信頼性が上がります。

分割 データの割り当て 結果
Fold 1 テスト / 訓練 / 訓練 / 訓練 / 訓練 スコア 1
Fold 2 訓練 / テスト / 訓練 / 訓練 / 訓練 スコア 2
… … …
Fold 5 訓練 / 訓練 / 訓練 / 訓練 / テスト スコア 5

5 回分のスコアを平均 ± 標準偏差でモデルの性能を評価します。 分類では StratifiedKFold を使うことで、各 fold のクラス比率を元のデータと同じに保ちます（不均衡データで特に重要）。

次のコードでは、5 分割の交差検証で F1 スコアを計算しています。1 回の分割だけで判断せず、複数回の評価の平均とばらつきを見ます。

from sklearn.model_selection import cross_val_score, StratifiedKFold

cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)

scores = cross_val_score(
    model, X, y,
    cv=cv,
    scoring="f1",
    n_jobs=-1
)
print(f"F1: {scores.mean():.3f} ± {scores.std():.3f}")
# 標準偏差が大きい → fold によってスコアがバラバラ → 不安定なモデル

平均スコアが高くても標準偏差が大きい場合、データの分け方によって性能が大きく変わる不安定なモデルかもしれません。平均だけでなくばらつきも見ます。

ハイパーパラメータチューニング¶

モデルの「学習で決まるパラメータ（重みなど）」とは別に、「人が事前に設定するパラメータ」をハイパーパラメータと呼びます（例：学習率、木の深さ、正則化の強さ）。ハイパーパラメータの値はデータや問題によって最適値が異なるため、探索が必要です。

GridSearchCV（全探索）¶

指定した組み合わせをすべて試します。組み合わせ数が多いと時間がかかりますが、見落としがありません。

GridSearchCV は、指定したパラメータ候補の全組み合わせを試します。候補が少ないときや、重要なパラメータの範囲が分かっているときに向いています。

from sklearn.model_selection import GridSearchCV
from sklearn.ensemble import RandomForestClassifier

param_grid = {
    "n_estimators": [100, 200, 300],
    "max_depth": [None, 5, 10],
    "min_samples_split": [2, 5],
}
# 3 × 3 × 2 = 18 通り × 5 fold = 90 回の学習

gs = GridSearchCV(
    RandomForestClassifier(random_state=42),
    param_grid,
    cv=5,
    scoring="f1",
    n_jobs=-1,
    verbose=1
)
gs.fit(X_train, y_train)

print(f"最良パラメータ: {gs.best_params_}")
print(f"最良 CV スコア: {gs.best_score_:.3f}")
best_model = gs.best_estimator_

この例では 18 通りのパラメータを 5 fold で評価するため、合計 90 回の学習が走ります。best_estimator_ には、最良パラメータで学習されたモデルが入ります。

RandomizedSearchCV（ランダム探索）¶

パラメータをランダムにサンプリングして探索します。GridSearchCV より少ない試行回数で広い探索空間をカバーできるため、パラメータが多いモデルに向いています。

ランダム探索は、すべてを試す代わりに、指定した分布から一定回数だけ候補を選んで試します。候補が多すぎるときに現実的な探索方法です。

from sklearn.model_selection import RandomizedSearchCV
from scipy.stats import randint, uniform

param_dist = {
    "n_estimators": randint(100, 500),
    "max_depth": [None, 3, 5, 7, 10],
    "min_samples_split": randint(2, 10),
    "max_features": uniform(0.1, 0.9),
}

rs = RandomizedSearchCV(
    RandomForestClassifier(random_state=42),
    param_dist,
    n_iter=50,          # 50 通りをランダムに試す
    cv=5,
    scoring="f1",
    random_state=42,
    n_jobs=-1
)
rs.fit(X_train, y_train)
print(f"最良パラメータ: {rs.best_params_}")

n_iter=50 は 50 通りだけ試すという意味です。randint や uniform を使うと、整数や連続値の範囲からランダムに候補を選べます。

Optuna（ベイズ最適化）¶

GridSearch はパラメータ空間を均等に探索しますが、実際には「精度が上がりやすい領域」が存在します。ベイズ最適化 は過去の試行結果から「次に試すべき領域」を統計的に推定し、効率的に最適値を探します。

Optuna はこのベイズ最適化を Python で簡単に使えるライブラリです。trial.suggest_* でパラメータの範囲を指定し、objective 関数が返す値を最大化（または最小化）するパラメータを自動的に見つけます。

Optuna では、1 回の試行を trial、試行全体の管理を study と呼びます。objective 関数は、パラメータを受け取り、モデルを評価し、最終的なスコアを返す関数です。

import optuna
from sklearn.model_selection import cross_val_score
import xgboost as xgb

optuna.logging.set_verbosity(optuna.logging.WARNING)

def objective(trial):
    # trial.suggest_* でパラメータ空間を定義
    params = {
        "n_estimators": trial.suggest_int("n_estimators", 100, 1000),
        "max_depth": trial.suggest_int("max_depth", 3, 10),
        "learning_rate": trial.suggest_float("learning_rate", 1e-3, 0.3, log=True),
        "subsample": trial.suggest_float("subsample", 0.5, 1.0),
        "colsample_bytree": trial.suggest_float("colsample_bytree", 0.5, 1.0),
        "min_child_weight": trial.suggest_int("min_child_weight", 1, 10),
    }
    model = xgb.XGBClassifier(**params, random_state=42, eval_metric="logloss", n_jobs=-1)
    scores = cross_val_score(model, X_train, y_train, cv=3, scoring="roc_auc")
    return scores.mean()

# Study を作成してチューニング開始
study = optuna.create_study(
    direction="maximize",
    sampler=optuna.samplers.TPESampler(seed=42),  # TPE（Tree-structured Parzen Estimator）
)
study.optimize(objective, n_trials=100, show_progress_bar=True)

print(f"最良スコア: {study.best_value:.4f}")
print(f"最良パラメータ:")
for key, value in study.best_params.items():
    print(f"  {key}: {value}")

direction="maximize" は、返ってくるスコアを最大化するという意味です。AUC や F1 は最大化、損失や RMSE は最小化として設定します。

Optuna の Pruning（早期打ち切り）¶

promising でない試行を途中で打ち切ることで、探索を大幅に高速化できます。MedianPruner は「中間スコアが過去の試行の中央値より低い trial を打ち切る」ルールです。

Pruning は、途中経過を見て見込みが低い試行を早めに止める仕組みです。時間のかかるモデルでは、良くなりそうにない試行に最後まで計算資源を使わないことが重要です。

import optuna
from optuna.integration import XGBoostPruningCallback
import xgboost as xgb

def objective_with_pruning(trial):
    params = {
        "n_estimators": trial.suggest_int("n_estimators", 100, 1000),
        "max_depth": trial.suggest_int("max_depth", 3, 10),
        "learning_rate": trial.suggest_float("learning_rate", 1e-3, 0.3, log=True),
        "eval_metric": "logloss",
    }

    model = xgb.XGBClassifier(
        **params,
        random_state=42,
        callbacks=[XGBoostPruningCallback(trial, "validation_0-logloss")]
    )
    model.fit(
        X_train, y_train,
        eval_set=[(X_val, y_val)],
        early_stopping_rounds=20,
        verbose=False
    )

    return model.score(X_val, y_val)

study = optuna.create_study(
    direction="maximize",
    pruner=optuna.pruners.MedianPruner(n_startup_trials=10, n_warmup_steps=20)
)
study.optimize(objective_with_pruning, n_trials=100)

n_startup_trials は、最初の何試行は打ち切らずに観察するかを表します。最初から打ち切りすぎると、探索の判断材料が不足します。

Optuna の結果を可視化する¶

チューニング結果は、最良パラメータだけでなく「どのパラメータが効いたか」「探索が改善しているか」を見ることが大切です。

import optuna.visualization as vis

# パラメータの重要度（どのパラメータが結果に影響を与えているか）
fig = vis.plot_param_importances(study)
fig.write_html("param_importances.html")

# 探索の履歴（試行回数と最良スコアの推移）
fig = vis.plot_optimization_history(study)
fig.write_html("optimization_history.html")

# パラメータ間の相関（どの組み合わせが良いか）
fig = vis.plot_contour(study, params=["learning_rate", "max_depth"])
fig.write_html("contour.html")

可視化によって、重要なパラメータや、良いスコアが出やすい範囲を確認できます。次の探索範囲を狭める判断にも使えます。

学習曲線とバリデーション曲線¶

モデルが過学習（訓練精度は高いが検証精度が低い） か 未学習（両方低い） かを診断するツールです。

バリデーション曲線は、1 つのハイパーパラメータを変えたときに、訓練スコアと検証スコアがどう変わるかを見る図です。モデルを複雑にすべきか、単純にすべきかの判断材料になります。

from sklearn.model_selection import validation_curve
import matplotlib.pyplot as plt
import numpy as np

# パラメータの変化に対する訓練/検証スコアの推移
param_range = [1, 5, 10, 20, 50, 100]
train_scores, val_scores = validation_curve(
    RandomForestClassifier(random_state=42),
    X, y,
    param_name="n_estimators",
    param_range=param_range,
    cv=5,
    scoring="f1",
    n_jobs=-1
)

train_mean = train_scores.mean(axis=1)
val_mean = val_scores.mean(axis=1)

plt.plot(param_range, train_mean, label="訓練スコア")
plt.plot(param_range, val_mean, label="検証スコア")
plt.fill_between(param_range,
                 train_mean - train_scores.std(axis=1),
                 train_mean + train_scores.std(axis=1), alpha=0.1)
plt.xlabel("n_estimators")
plt.ylabel("F1 スコア")
plt.legend()
plt.savefig("validation_curve.png")

# 解釈の目安：
# 訓練スコア高い・検証スコア低い → 過学習（モデルを単純化 or 正則化強化）
# 訓練スコア低い・検証スコア低い → 未学習（モデルを複雑化 or 特徴量を追加）
# 両方高い → 良好

訓練スコアだけが高い場合は、訓練データを覚えすぎている可能性があります。訓練・検証の両方が低い場合は、モデルの表現力不足や特徴量不足を疑います。

まとめ：チューニングの進め方¶

機械学習のチューニングは「闇雲に試す」のではなく、以下の順で進めると効率的です。

  1. ベースラインを作る：ロジスティック回帰や決定木でまず精度を確認
  2. 適切な評価指標を選ぶ：目的に合った指標（F1・ROC-AUC など）を決める
  3. 交差検証で安定した評価をする：単一のテスト分割より信頼性が高い
  4. 学習曲線で過学習/未学習を診断する：方向性を決めてからチューニング
  5. Optuna でハイパーパラメータを最適化する：GridSearch より効率的
  6. 最終評価はテストデータで 1 回だけ：チューニング中にテストデータを見ない
