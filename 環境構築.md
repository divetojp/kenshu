# 環境構築

## 環境構築とは

**開発に必要なツールをPCにインストールして、作業できる状態にすること** です。

初めてプロジェクトに参加するときに一度だけ行います。

---

## はじめて読む人へ

環境構築は、開発を始める前に必要な道具をそろえる作業です。料理でいうと、材料を切る前に包丁・まな板・コンロを準備する段階にあたります。

コードやコマンドが出てきたら、最初から全部を覚えようとしなくて大丈夫です。まずは「何を入力し、何が処理され、何が出力されるのか」を文章で説明できるように読むと、手を動かす前の理解が安定します。

### 読む前に押さえること

- 環境構築の目的は、自分のPCで同じ手順を再現できるようにすることです。
- インストール後は、必ずバージョン確認コマンドで成功を確認します。
- うまくいかないときは、OS、CPU、パス、権限のどれが原因かを切り分けます。

### 読み終えたら説明できること

- Git、Docker、VS Code がなぜ必要か説明できる。
- インストール後の確認コマンドを実行できる。
- 環境構築の失敗を落ち着いて切り分けられる。

---


> **情報工学メモ：PATH 環境変数とは（コマンドがどうやって見つかるか）**  
> `git --version` とタイプしたとき、ターミナルはどこにある `git` を実行すればよいかを知る必要があります。これを解決するのが **PATH 環境変数** です。PATH には「コマンドを探すフォルダのリスト」が `:` 区切りで格納されており（例：`/usr/bin:/usr/local/bin`）、ターミナルはこのリストを順番に検索して一致するコマンドを実行します。新しいツールをインストールしたのに「command not found」と出る場合、そのツールが置かれたフォルダが PATH に含まれていないことが原因です。インストーラーが自動的に PATH を設定してくれることが多いですが、手動で追加が必要な場合もあります。

## インストールするもの一覧

ここで入れる道具は、それぞれ役割が違います。Git は変更履歴を管理し、Docker は実行環境をそろえ、VS Code はコードを書く場所になり、GitHub はチームやクラウドと変更を共有する場所になります。

環境構築で大切なのは、「入れたつもり」ではなく「使える状態か」を確認することです。そのため、インストール後には必ずバージョン確認コマンドを実行します。

| ツール | 用途 | 確認コマンド |
|--------|------|------------|
| Git | バージョン管理 | `git --version` |
| Docker Desktop | ローカル開発環境（※） | `docker --version` |
| VS Code | コードエディタ | — |
| Node.js（任意） | Docker を使わない場合 | `node --version` |

> **※ Docker とコンテナについて：** Docker は「コンテナ」という仮想的な実行環境を作るツールです。コンテナとは、アプリとその動作に必要な設定・ライブラリをまとめてパッケージ化したものです。「自分のPCでは動いたのに別のPCでは動かない」という問題を解決します。詳しくは [Docker](Docker) ページで解説します。

> Docker を使う場合は Node.js の個別インストールは不要です。

## ① Git のインストール

### Windows

[git-scm.com](https://git-scm.com) からインストーラーをダウンロードして実行してください。  
設定はすべてデフォルトのまま「Next」で進めて OK です。

インストール後にターミナル（PowerShell）を**再起動**してから確認してください。

インストールしただけでは、今開いているターミナルが新しいコマンドを認識していないことがあります。確認前にターミナルを開き直すと、PATH の更新が反映されます。

```bash
git --version
```

`git version 2.xx.x` と表示されればインストール済みです。

### Mac

Mac には最初から Git が入っていることが多いです。まず確認してみましょう。

Mac でも、まずはコマンドで Git が使えるか確認します。バージョンが表示されれば、そのまま次へ進めます。

```bash
git --version
```

`git version 2.xx.x` と表示されればインストール済みです。インストールされていない場合は、コマンドを実行すると自動でインストールの案内が出ます。または [git-scm.com](https://git-scm.com) からダウンロードしてください。

### インストール後の初期設定

インストールしたら、名前とメールアドレスを登録します（一度だけ）。

この名前とメールアドレスは、Git のコミットに記録されます。チーム開発では、誰が変更したかを追えるようにするための基本設定です。

```bash
git config --global user.name "山田 太郎"
git config --global user.email "taro@example.com"
```

## ② Docker Desktop のインストール

### ダウンロード

[Docker Desktop 公式サイト](https://www.docker.com/products/docker-desktop/) にアクセスして、自分のOS（Windows / Mac）に合ったものをダウンロードします。

### Windows の注意点

**WSL 2（Windows Subsystem for Linux）** が必要です。インストールされていない場合、Docker Desktop のインストール中に案内が表示されます。

PowerShell を管理者として開き、以下を実行してください（事前に済んでいれば不要）：

WSL 2 は、Windows 上で Linux 環境を動かすための仕組みです。Docker Desktop は内部で Linux コンテナを動かすため、Windows ではこの準備が必要になります。

```powershell
wsl --install
```

再起動後に Docker Desktop のインストールを進めてください。

### Mac の注意点

MacBook のチップによってダウンロードするファイルが違います。

Apple Silicon と Intel では CPU の種類が違うため、Docker Desktop のインストーラーも対応するものを選ぶ必要があります。次のコマンドで確認できます。

```bash
# チップの種類を確認するコマンド
uname -m
```

| 表示 | 選ぶもの |
|------|---------|
| `arm64` | Apple Silicon（M1/M2/M3/M4）用 |
| `x86_64` | Intel チップ用 |

### インストール後の確認

Docker Desktop を起動した状態で、ターミナルで以下を実行します。

Docker はアプリをインストールするだけでなく、Docker Desktop が起動している必要があります。バージョン確認では、Docker コマンドと Compose コマンドの両方を確認します。

```bash
docker --version
docker compose version
```

どちらもバージョンが表示されれば OK です。

> **Docker Desktop は起動していないと使えません。**  
> 毎回PCを起動したら Docker Desktop を立ち上げる習慣をつけましょう。  
> メニューバー（Mac）やタスクバー（Windows）にクジラのアイコンが表示されていれば起動中です。🐳

## ③ VS Code のインストール

VS Code（Visual Studio Code）は、Microsoft が開発した無料のコードエディタです。

[code.visualstudio.com](https://code.visualstudio.com) からダウンロードしてインストールします。

### おすすめの拡張機能

VS Code を開いて、左のアイコン（拡張機能タブ）から検索してインストールできます。

| 拡張機能名 | 用途 |
|-----------|------|
| **Astro** | Astro ファイルのシンタックスハイライト |
| **Prettier** | コードの自動整形 |
| **Japanese Language Pack** | VS Code の日本語化 |

### ターミナルを VS Code で開く

`Ctrl + `` ` （バッククォート）で VS Code 内蔵のターミナルが開きます。  
エディタとターミナルを切り替えなくていいのでとても便利です。

## プロジェクトを手元で動かす

環境構築が完了したら、実際にプロジェクトを起動してみましょう。

最後に、実際のプロジェクトをクローンして Docker で起動します。ここまで成功すると、開発に必要な道具が一通りつながっていることを確認できます。

```bash
# 1. リポジトリをクローン
git clone https://github.com/ohmitechacademy/ota_hp.git
cd ota_hp

# 2. Docker で開発サーバーを起動
docker compose up

# 3. ブラウザで http://localhost:4321 を開く
```

サイトが表示されれば成功です！

> **`docker compose up` の仕組み（簡単に）**  
> プロジェクトのフォルダに `docker-compose.yml` というファイルがあります。これがコンテナの設定書で、「どの Node.js バージョンを使うか」「どのポートで起動するか」などが書かれています。  
> `docker compose up` はこのファイルを読み込んでコンテナを起動し、自動的に `npm run dev`（開発サーバー）を実行してくれます。詳しくは [Docker](Docker) ページで解説しています。
