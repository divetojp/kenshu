# 数値計算

コンピュータで数学的な計算を「正確に・効率よく・安定して」行うための技術です。浮動小数点数の限界・数値安定性・数値微分・数値積分・線形方程式の解法を扱います。機械学習では「なぜ損失が NaN になるのか」「なぜ混合精度学習が使われるのか」の答えがここにあります。

---

## はじめて読む人へ

コンピュータは実数を有限のビット数で表現するため、計算誤差が避けられません。「$0.1 + 0.2 = 0.3$ のはずが `0.30000000000000004` になる」——これが浮動小数点の現実です。この誤差が積み重なると、深層学習の学習が不安定になります。

### 読む前に押さえること

- [微分・最適化基礎](微分・最適化基礎.md) の微分・積分の概念
- Python の基本的な数値演算

### 読み終えたら説明できること

- 浮動小数点の丸め誤差がなぜ起きるかを説明できる
- 数値的に不安定な計算パターンを識別できる
- 機械学習の混合精度（float16/float32）の使い分けを説明できる

## 浮動小数点数（IEEE 754）

コンピュータの実数表現は「すべての実数を表せない」という根本的な制限があります。

### 64ビット浮動小数点（float64 / double）

$$
x = (-1)^s \times 1.m \times 2^{e-1023}
$$

| 部分 | ビット数 | 意味 |
|------|---------|------|
| 符号 $s$ | 1 | $+$ か $-$ |
| 指数 $e$ | 11 | $2^{e-1023}$ のスケール |
| 仮数 $m$ | 52 | 精度（有効数字 ~15 桁）|

```python
import numpy as np
import sys

# float64 の限界値
print(f"最大値: {sys.float_info.max:.2e}")       # 1.80e+308
print(f"最小正規化数: {sys.float_info.min:.2e}") # 2.23e-308
print(f"機械イプシロン: {sys.float_info.epsilon:.2e}")  # 2.22e-16

# 有名な浮動小数点の罠
print(f"0.1 + 0.2 = {0.1 + 0.2}")         # 0.30000000000000004
print(f"0.1 + 0.2 == 0.3: {0.1 + 0.2 == 0.3}")  # False!

# 正しい比較方法
print(f"np.isclose(0.1+0.2, 0.3): {np.isclose(0.1 + 0.2, 0.3)}")  # True
```

### 桁落ち（Catastrophic Cancellation）

ほぼ同じ値の引き算は有効桁数が激減します。

```python
# 桁落ちの例
x = 1.0000001
y = 1.0000000
diff = x - y   # 本来は 0.0000001 = 1e-7

# float64 で計算
print(f"差: {diff:.10e}")       # 9.9999999e-08 (誤差あり)

# 悪い例：x²- y² を直接計算
x, y = 1000000.1, 1000000.0
bad  = x**2 - y**2              # (x+y)(x-y) を展開してから引く → 精度損失
good = (x + y) * (x - y)        # 数式を変形してから計算 → 精度保持
print(f"直接: {bad:.6f}, 変形: {good:.6f}")  # good の方が正確
```

## 数値微分

解析的な微分式を使わずに、数値的に微分を近似します。

```python
import numpy as np

# 前進差分（1次精度）
def forward_diff(f, x, h=1e-5):
    return (f(x + h) - f(x)) / h

# 中心差分（2次精度、より正確）
def central_diff(f, x, h=1e-5):
    return (f(x + h) - f(x - h)) / (2 * h)

# 例：f(x) = sin(x) の微分（真値は cos(x)）
f = np.sin
x = 1.0
true_deriv = np.cos(x)

print(f"真の微分: {true_deriv:.10f}")
print(f"前進差分: {forward_diff(f, x):.10f}  (誤差: {abs(forward_diff(f, x) - true_deriv):.2e})")
print(f"中心差分: {central_diff(f, x):.10f}  (誤差: {abs(central_diff(f, x) - true_deriv):.2e})")
# 中心差分の方が2桁精度が高い
```

### h の選択トレードオフ

```python
import matplotlib.pyplot as plt

f = np.sin
x = 1.0
true_deriv = np.cos(x)

h_values = np.logspace(-16, 0, 100)
errors = [abs(central_diff(f, x, h) - true_deriv) for h in h_values]

plt.loglog(h_values, errors)
plt.xlabel("刻み幅 h")
plt.ylabel("絶対誤差")
plt.title("h が小さすぎると丸め誤差で精度が悪化する")
plt.axvline(1e-5, color='red', linestyle='--', label="最適な h ≈ 1e-5")
plt.legend()
plt.savefig("numerical_diff.png", dpi=150)
# → h が大きい: 打ち切り誤差 大
# → h が小さい: 丸め誤差 大
# → 最適な h が存在する
```

## 数値積分

解析解がない積分を数値的に近似します。

```python
import numpy as np
from scipy import integrate

# 台形則（Trapezoidal Rule）
def trapezoid(f, a, b, n=1000):
    x = np.linspace(a, b, n+1)
    y = f(x)
    h = (b - a) / n
    return h * (y[0]/2 + np.sum(y[1:-1]) + y[-1]/2)

# シンプソン則（より高精度）
def simpsons(f, a, b, n=1000):
    if n % 2 != 0:
        n += 1
    x = np.linspace(a, b, n+1)
    y = f(x)
    h = (b - a) / n
    return h/3 * (y[0] + 4*np.sum(y[1::2]) + 2*np.sum(y[2:-1:2]) + y[-1])

# ∫₀¹ x² dx = 1/3 ≈ 0.3333...
f = lambda x: x**2
print(f"台形則:       {trapezoid(f, 0, 1):.8f}")
print(f"シンプソン則: {simpsons(f, 0, 1):.8f}")
print(f"scipy.quad:   {integrate.quad(f, 0, 1)[0]:.8f}")
print(f"真値:         {1/3:.8f}")

# モンテカルロ積分（高次元に強い）
def monte_carlo_integral(f, a, b, n_samples=100000):
    x = np.random.uniform(a, b, n_samples)
    return (b - a) * np.mean(f(x))

print(f"モンテカルロ: {monte_carlo_integral(f, 0, 1):.4f}")
```

## 線形方程式の解法

$Ax = b$ を解く数値的な方法です。

```python
import numpy as np
from scipy import linalg

n = 500
A = np.random.randn(n, n)
A = A @ A.T + n * np.eye(n)  # 正定値行列
b = np.random.randn(n)

# 直接法（LU 分解）
x_lu = linalg.solve(A, b)

# 反復法（共役勾配法）—— 大規模疎行列に有効
from scipy.sparse.linalg import cg
x_cg, info = cg(A, b, tol=1e-6)

residual_lu = np.linalg.norm(A @ x_lu - b)
residual_cg = np.linalg.norm(A @ x_cg - b)
print(f"LU 解法の残差: {residual_lu:.2e}")
print(f"CG 解法の残差: {residual_cg:.2e}")
```

### 条件数と数値安定性

```python
# 条件数 κ(A) = ||A|| × ||A⁻¹|| ≈ 最大固有値/最小固有値
A_good = np.array([[4, 1], [1, 3]], dtype=float)  # 条件数 小
A_bad  = np.array([[1, 1], [1, 1+1e-8]], dtype=float)  # 条件数 大（ほぼ特異）

print(f"良い行列の条件数: {np.linalg.cond(A_good):.2f}")
print(f"悪い行列の条件数: {np.linalg.cond(A_bad):.2e}")
# 条件数が大きい → 入力の小さな変化が解に大きく影響 → 数値的に不安定
```

## 機械学習への応用：混合精度学習

```python
import torch

# float32（標準）vs float16（半精度）
x32 = torch.tensor(1e-5, dtype=torch.float32)
x16 = torch.tensor(1e-5, dtype=torch.float16)

print(f"float32: {x32.item():.6e}")   # 正確
print(f"float16: {x16.item():.6e}")   # 表現できず 0 になる可能性

# float16 のオーバーフロー
large = torch.tensor(70000, dtype=torch.float32)
print(f"float32 70000: {large.item()}")
large16 = large.to(torch.float16)
print(f"float16 70000: {large16.item()}")  # inf（オーバーフロー）

# PyTorch の自動混合精度（AMP）
# 順伝播: float16（高速）、逆伝播: float32（安定）
from torch.cuda.amp import autocast, GradScaler

# with autocast():
#     output = model(input)         # float16 で計算
# scaler.scale(loss).backward()     # 勾配をスケーリングして安定化
```

| 型 | 精度（有効数字）| 最大値 | メモリ | 用途 |
|----|--------------|-------|-------|------|
| float64 | ~15 桁 | 1.8×10³⁰⁸ | 8 byte | 科学計算 |
| float32 | ~7 桁 | 3.4×10³⁸ | 4 byte | ML 標準 |
| float16 | ~3 桁 | 65504 | 2 byte | ML 高速化 |

## ODE の数値解法

微分方程式 $\frac{dy}{dt} = f(t, y)$ を数値的に解きます。

```python
from scipy.integrate import solve_ivp
import numpy as np
import matplotlib.pyplot as plt

# ロジスティック方程式: dy/dt = r*y*(1 - y/K)
r, K = 0.5, 100.0

def logistic(t, y):
    return r * y[0] * (1 - y[0] / K)

# 解く
sol = solve_ivp(logistic, t_span=[0, 30], y0=[1.0], dense_output=True)

t = np.linspace(0, 30, 300)
y = sol.sol(t)[0]

plt.plot(t, y, label="数値解（RK45）")
# 解析解
y_analytic = K / (1 + (K - 1) * np.exp(-r * t))
plt.plot(t, y_analytic, '--', label="解析解")
plt.xlabel("時間"); plt.ylabel("個体数")
plt.title("ロジスティック方程式の数値解")
plt.legend()
plt.savefig("ode.png", dpi=150)
```
