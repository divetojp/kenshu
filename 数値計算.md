# 数値計算

コンピュータで数学的な計算を「正確に・効率よく・安定して」行うための技術です。浮動小数点数の限界・数値安定性・数値微分・数値積分・線形方程式の解法を扱います。機械学習では「なぜ損失が NaN になるのか」「なぜ混合精度学習が使われるのか」の答えがここにあります。

---

## はじめて読む人へ

コンピュータは実数を有限のビット数で表現するため、計算誤差が避けられません。「$0.1 + 0.2 = 0.3$ のはずが `0.30000000000000004` になる」——これが浮動小数点の現実です。この誤差が積み重なると、深層学習の学習が不安定になります。

### 読む前に押さえること

- [微分・最適化基礎](微分・最適化基礎.md) の微分・積分の概念
- Python の基本的な数値演算

### 読み終えたら説明できること

- 浮動小数点の丸め誤差がなぜ起きるかを説明できる
- 数値的に不安定な計算パターンを識別できる
- 機械学習の混合精度（float16/float32）の使い分けを説明できる

## 浮動小数点数（IEEE 754）

コンピュータの実数表現は「すべての実数を表せない」という根本的な制限があります。

### 64ビット浮動小数点（float64 / double）

$$
x = (-1)^s \times 1.m \times 2^{e-1023}
$$

| 部分 | ビット数 | 意味 |
|------|---------|------|
| 符号 $s$ | 1 | $+$ か $-$ |
| 指数 $e$ | 11 | $2^{e-1023}$ のスケール |
| 仮数 $m$ | 52 | 精度（有効数字 ~15 桁）|

```python
import numpy as np
import sys

print(f"最大値: {sys.float_info.max:.2e}")       # 1.80e+308
print(f"機械イプシロン: {sys.float_info.epsilon:.2e}")  # 2.22e-16

# 有名な浮動小数点の罠
print(f"0.1 + 0.2 = {0.1 + 0.2}")         # 0.30000000000000004
print(f"0.1 + 0.2 == 0.3: {0.1 + 0.2 == 0.3}")  # False!
print(f"np.isclose(0.1+0.2, 0.3): {np.isclose(0.1 + 0.2, 0.3)}")  # True
```

### 桁落ち（Catastrophic Cancellation）

ほぼ同じ値の引き算は有効桁数が激減します。

```python
# x²-y² を直接計算 vs 数式変形
x, y = 1000000.1, 1000000.0
bad  = x**2 - y**2       # 精度損失
good = (x + y) * (x - y) # 数式を変形してから計算 → 精度保持
```

## 数値微分

解析的な微分式を使わずに、数値的に微分を近似します。

| 手法 | 式 | 誤差 |
|------|-----|------|
| 前進差分 | $(f(x+h) - f(x)) / h$ | $O(h)$ |
| 中心差分 | $(f(x+h) - f(x-h)) / (2h)$ | $O(h^2)$ |

中心差分は $h$ を 1/10 にすると誤差が 1/100 になり、前進差分より 1 桁精度が良いです。ただし $h$ を小さくしすぎると浮動小数点の丸め誤差が増大するため、$h \approx 10^{-5}$ が実用的な最適値です。

---

## 数値積分

解析解がない積分を数値的に近似します。

| 手法 | 精度 | 特徴 |
|------|------|------|
| 台形則 | $O(h^2)$ | 実装が簡単 |
| シンプソン則 | $O(h^4)$ | 台形則より高精度 |
| モンテカルロ積分 | $O(1/\sqrt{n})$ | 高次元に強い |

`scipy.integrate.quad` を使えば適応的な数値積分が可能です。モンテカルロ積分は次元の呪いを受けにくく、高次元積分で力を発揮します。

## 線形方程式の解法

$Ax = b$ を解く数値的な方法です。

```python
import numpy as np
from scipy import linalg

n = 500
A = np.random.randn(n, n)
A = A @ A.T + n * np.eye(n)  # 正定値行列
b = np.random.randn(n)

x_lu = linalg.solve(A, b)   # 直接法（LU 分解）
```

**反復法（共役勾配法）** は大規模疎行列に有効です（`scipy.sparse.linalg.cg`）。

### 条件数と数値安定性

条件数 $\kappa(A) = \|A\| \cdot \|A^{-1}\|$ が大きい（ほぼ特異な）行列は、入力の小さな変化が解に大きく影響し数値的に不安定です。`np.linalg.cond(A)` で確認できます。

## 機械学習への応用：混合精度学習

| 型 | 精度（有効数字）| 最大値 | メモリ | 用途 |
|----|--------------|-------|-------|------|
| float64 | ~15 桁 | 1.8×10³⁰⁸ | 8 byte | 科学計算 |
| float32 | ~7 桁 | 3.4×10³⁸ | 4 byte | ML 標準 |
| float16 | ~3 桁 | 65504 | 2 byte | ML 高速化 |
| bfloat16 | ~2 桁（範囲広）| 3.4×10³⁸ | 2 byte | TPU 向け |

float16 は最大値が 65504 のため、大きな値（例：70000）はオーバーフローして `inf` になります。PyTorch の自動混合精度（AMP）は、順伝播を float16（高速）で行い、逆伝播を float32（安定）で行うことで両方の長所を活かします。

## ODE の数値解法

微分方程式 $\frac{dy}{dt} = f(t, y)$ を数値的に解きます。

| 手法 | 精度 | 特徴 |
|------|------|------|
| オイラー法 | $O(h)$ | 最もシンプル。大きな誤差 |
| RK4（ルンゲ・クッタ 4 次）| $O(h^4)$ | 標準的な選択 |
| RK45（適応的）| 高精度 | `scipy.integrate.solve_ivp` の既定手法 |

ロジスティック方程式 $\frac{dy}{dt} = r y (1 - y/K)$ のように解析解がある場合でも、ODE ソルバーで数値解との比較を確認できます。Diffusion モデルや Neural ODE はこのフレームワークを活用しています。

## 数学的導出

### 中心差分公式の誤差がテイラー展開で $O(h^2)$ になる理由

前進差分 $(f(x+h) - f(x))/h$ は $O(h)$ の誤差を持ちますが、中心差分はより精度が高い理由を示します。

テイラー展開から：

$$
f(x+h) = f(x) + h f'(x) + \frac{h^2}{2}f''(x) + \frac{h^3}{6}f'''(x) + O(h^4)
$$

$$
f(x-h) = f(x) - h f'(x) + \frac{h^2}{2}f''(x) - \frac{h^3}{6}f'''(x) + O(h^4)
$$

2 式の差を取ると：

$$
f(x+h) - f(x-h) = 2h f'(x) + \frac{h^3}{3}f'''(x) + O(h^5)
$$

両辺を $2h$ で割ると：

$$
\frac{f(x+h) - f(x-h)}{2h} = f'(x) + \frac{h^2}{6}f'''(x) + O(h^4)
$$

$$
\boxed{\text{誤差} = \frac{h^2}{6}f'''(x) + O(h^4) = O(h^2)}
$$

前進差分の誤差 $O(h)$ に対して **1 桁精度が良い**理由はここにあります。$h$ を 10 分の 1 にすると、前進差分の誤差は $1/10$ になりますが、中心差分の誤差は $1/100$ になります。

---

### 浮動小数点表現の丸め誤差の構造

IEEE 754 float64 の表現：

$$
x = (-1)^s \times (1 + \text{仮数部}) \times 2^{\text{指数部}}
$$

**機械イプシロン：** 1 より大きい最小の float64 と 1 との差：

$$
\epsilon_{\text{mach}} = 2^{-52} \approx 2.22 \times 10^{-16}
$$

各演算で **相対誤差は最大** $\epsilon_{\text{mach}}/2$ 生じます。`0.1 + 0.2 ≠ 0.3` の理由は 0.1, 0.2, 0.3 がいずれも 2 進数で有限小数にならず、それぞれ丸め誤差を持つためです。

**数値安定性の指針：**

| 危険なコード | 安全なコード |
|------------|-------------|
| `log(1 + x)` ($x \approx 0$) | `np.log1p(x)` |
| `exp(x) - 1` ($x \approx 0$) | `np.expm1(x)` |
| `sqrt(x^2 + y^2)` | `np.hypot(x, y)` |
| softmax をそのまま実装 | `x -= x.max()` して計算 |

## 確認問題

1. 中心差分 $(f(x+h)-f(x-h))/(2h)$ の誤差が $O(h^2)$ になることをテイラー展開で示してください。
2. `0.1 + 0.2 != 0.3` がなぜ起きるかを浮動小数点表現の観点から説明してください。
3. 機械学習で float16 を使うと学習が不安定になることがある理由と、AMP（自動混合精度）がどう対処するかを説明してください。
